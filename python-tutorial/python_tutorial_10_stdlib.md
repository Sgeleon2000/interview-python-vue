# Глава 10. Краткий обзор стандартной библиотеки (Brief Tour of the Standard Library) — конспект и вопросы

## О чём глава

Python поставляется с «батарейками в комплекте» (batteries included) — огромной стандартной библиотекой, которая покрывает большинство повседневных задач: работу с файлами и ОС, регулярные выражения, математику, доступ в интернет, даты, сжатие данных, тестирование. Эта глава — обзорная экскурсия по самым полезным модулям. Здесь не нужно знать каждый аргумент каждой функции наизусть, но важно понимать, **какой модуль решает какую задачу**, чтобы не изобретать велосипед.

Главная мысль для собеседования: «Прежде чем писать свой код — посмотри, нет ли готового решения в стандартной библиотеке».

## Ключевые концепции (с примерами и выводом)

### os — интерфейс операционной системы

```python
import os

# Текущая рабочая директория
print(os.getcwd())            # => /Users/user/project (текущий путь)

# Сменить директорию
os.chdir('/tmp')              # перейти в /tmp

# Выполнить команду оболочки (устаревший способ, лучше subprocess)
os.system('mkdir today')      # => 0 (код возврата; создаёт папку today)

# Переменные окружения
print(os.environ.get('HOME')) # => /Users/user (значение переменной окружения)

# Список файлов в директории
print(os.listdir('.'))        # => ['file1.py', 'file2.txt']
```

ВАЖНО: для работы с путями используйте `os.path` (или современный `pathlib`), а не склейку строк:

```python
import os.path

path = os.path.join('folder', 'sub', 'file.txt')  # кроссплатформенный путь
print(path)                   # => folder/sub/file.txt (на Linux/macOS)

print(os.path.exists(path))   # => False (существует ли файл)
print(os.path.basename(path)) # => file.txt (имя файла)
print(os.path.dirname(path))  # => folder/sub (директория)
print(os.path.splitext(path)) # => ('folder/sub/file', '.txt')
```

Совет из туториала: вместо `from os import *` используйте `import os`, чтобы встроенная функция `open()` не была перекрыта `os.open()`.

### shutil — высокоуровневые операции с файлами

`os` даёт низкоуровневые операции, а `shutil` — удобные «человеческие»: копирование, перемещение, удаление целых деревьев.

```python
import shutil

shutil.copyfile('data.db', 'archive.db')   # копировать содержимое файла
shutil.copy('report.txt', '/tmp/')         # копировать файл в директорию
shutil.move('/build/exe', 'installdir')    # переместить (как mv)
shutil.rmtree('old_folder')                # удалить директорию целиком (рекурсивно!)

# Создать zip-архив из директории одной строкой
shutil.make_archive('backup', 'zip', 'src_dir')  # => backup.zip
```

### glob — поиск файлов по шаблону

Модуль `glob` ищет файлы по маске в стиле shell (`*`, `?`, `[...]`).

```python
import glob

print(glob.glob('*.py'))        # => ['main.py', 'utils.py'] (все .py в текущей папке)
print(glob.glob('data/*.csv'))  # => ['data/a.csv', 'data/b.csv']

# Рекурсивный поиск во всех подпапках
print(glob.glob('**/*.py', recursive=True))  # => все .py во всём дереве
```

### sys — аргументы командной строки, потоки, выход

```python
import sys

# sys.argv — список аргументов командной строки
# Запуск: python script.py one two
print(sys.argv)               # => ['script.py', 'one', 'two']

# Поток ошибок (отдельно от обычного вывода)
sys.stderr.write('Внимание: предупреждение!\n')  # пишет в stderr

# Завершить программу с кодом выхода
# sys.exit(0)  — успех; sys.exit(1) — ошибка
```

Здесь `argv[0]` — имя самого скрипта, остальное — переданные аргументы. `sys.stderr` отделяет диагностику от полезного вывода (`sys.stdout`), это важно для пайплайнов.

### argparse — нормальный разбор аргументов CLI

`sys.argv` хорош для простых случаев, но для настоящих CLI-утилит используют `argparse`: он сам генерирует справку (`--help`), проверяет типы, обрабатывает флаги.

```python
import argparse

parser = argparse.ArgumentParser(
    prog='top',
    description='Показать N первых строк файлов')
parser.add_argument('filenames', nargs='+')          # 1+ позиционных аргументов
parser.add_argument('-l', '--lines', type=int, default=10)  # опция с числом

args = parser.parse_args(['--lines=5', 'a.txt', 'b.txt'])
print(args)
# => Namespace(filenames=['a.txt', 'b.txt'], lines=5)
print(args.lines)      # => 5
print(args.filenames)  # => ['a.txt', 'b.txt']
```

### re — регулярные выражения

Модуль `re` даёт мощные инструменты для обработки строк по шаблону. Для простых случаев — обычные методы строк (`.replace()`, `.split()`), для сложных — регулярки.

```python
import re

# Найти все слова, начинающиеся на 'f'
print(re.findall(r'\bf[a-z]*', 'which foot or hand fell fastest'))
# => ['foot', 'fell', 'fastest']

# Замена с группами и обратными ссылками
print(re.sub(r'(\b[a-z]+) \1', r'\1', 'cat in the the hat'))
# => cat in the hat  (убрали дублирующееся слово)

# Поиск с группами
m = re.search(r'(\d{4})-(\d{2})-(\d{2})', 'Дата: 2026-06-20')
print(m.group(1), m.group(2), m.group(3))  # => 2026 06 20
```

Используйте «сырые» строки `r'...'` для шаблонов, чтобы не экранировать обратные слэши дважды.

### math, random, statistics — математика и случайность

```python
import math

print(math.cos(math.pi / 4))  # => 0.707106... (косинус 45 градусов)
print(math.log(1024, 2))      # => 10.0 (логарифм по основанию 2)
print(math.sqrt(16))          # => 4.0
```

```python
import random

print(random.choice(['яблоко', 'груша', 'банан']))  # => случайный элемент
print(random.sample(range(100), 5))   # => [30, 83, 16, 4, 8] (5 без повторов)
print(random.random())                # => 0.176... (float в [0.0, 1.0))
print(random.randrange(6))            # => 4 (целое из range(6): 0..5)
```

```python
import statistics

data = [2.75, 1.75, 1.25, 0.25, 0.5, 1.25, 3.5]
print(statistics.mean(data))    # => 1.6071428... (среднее)
print(statistics.median(data))  # => 1.25 (медиана)
print(statistics.variance(data))# => 1.3720... (дисперсия)
```

### urllib.request и smtplib — доступ в интернет

```python
from urllib.request import urlopen

# Скачать содержимое страницы
with urlopen('http://example.com') as response:
    html = response.read().decode('utf-8')
    print(html[:50])  # => первые 50 символов HTML-страницы
```

```python
import smtplib

# Отправить письмо через локальный SMTP-сервер
server = smtplib.SMTP('localhost')
server.sendmail('from@example.org', 'to@example.org',
                'From: from@example.org\nTo: to@example.org\n\nПривет!')
server.quit()
```

(smtplib требует доступного SMTP-сервера; на собеседовании важно знать, что модуль существует.)

### datetime — даты и время

```python
from datetime import date, datetime, timedelta

today = date.today()
print(today)                    # => 2026-06-20

# Форматирование
print(today.strftime('%d.%m.%Y'))  # => 20.06.2026

# Арифметика дат
birthday = date(1990, 5, 15)
age_days = today - birthday
print(age_days.days)            # => 13185 (разница в днях, объект timedelta)

# Завтра
print(today + timedelta(days=1))  # => 2026-06-21
```

`datetime` поддерживает часовые пояса, арифметику и парсинг (`strptime`).

### zlib, gzip — сжатие данных

```python
import zlib

s = b'witch which has which witches wrist watch'
print(len(s))                   # => 41 (исходный размер)
t = zlib.compress(s)
print(len(t))                   # => 37 (сжатый размер)
print(zlib.decompress(t))       # => b'witch which has...' (распаковка)
print(zlib.crc32(s))            # => 226805979 (контрольная сумма)
```

Также есть `gzip`, `bz2`, `lzma`, `zipfile`, `tarfile` для разных форматов архивов.

### timeit, doctest, unittest — измерение и тесты

`timeit` — точное измерение времени выполнения маленьких фрагментов:

```python
from timeit import Timer

# Что быстрее: распаковка кортежа или временная переменная?
print(Timer('t=a; a=b; b=t', 'a=1; b=2').timeit())      # => 0.057... сек
print(Timer('a,b = b,a', 'a=1; b=2').timeit())          # => 0.034... сек (быстрее)
```

`doctest` — тесты прямо в докстрингах:

```python
def average(values):
    """Возвращает среднее арифметическое списка чисел.

    >>> average([20, 30, 70])
    40.0
    """
    return sum(values) / len(values)

import doctest
doctest.testmod()  # автоматически проверяет примеры в докстрингах
```

`unittest` — полноценный фреймворк для тестов:

```python
import unittest

class TestAverage(unittest.TestCase):
    def test_average(self):
        self.assertEqual(average([20, 30, 70]), 40.0)
    def test_empty(self):
        with self.assertRaises(ZeroDivisionError):
            average([])

# unittest.main()  # запускает все тесты
```

## Подробный разбор по темам

### os vs shutil vs pathlib — кто за что отвечает

- **os / os.path** — низкоуровневый интерфейс ОС: переменные окружения, процессы, базовые операции с путями (строковые).
- **shutil** — высокоуровневые файловые операции: копировать, перемещать, удалять деревья, создавать архивы.
- **pathlib** (введён в 3.4, в туториале упомянут косвенно) — объектно-ориентированный современный способ работы с путями: `Path('folder') / 'file.txt'`. На реальной работе предпочитают его вместо `os.path`.

### Почему argparse, а не ручной разбор sys.argv

`argparse` бесплатно даёт: автоматический `--help`, проверку обязательных аргументов, преобразование типов (`type=int`), значения по умолчанию, понятные сообщения об ошибках, поддержку подкоманд. Ручной разбор `sys.argv` оправдан только для совсем тривиальных скриптов.

### stdout vs stderr

Два независимых потока вывода. `stdout` — для полезных данных (их можно перенаправить в файл или пайп `| grep`), `stderr` — для ошибок и диагностики (видны на экране, даже если stdout перенаправлен). Не пишите ошибки в `print()` (по умолчанию это stdout) — используйте `print(..., file=sys.stderr)` или `logging`.

### Производительность измеряют, а не угадывают

Интуиция о скорости часто врёт. `timeit` запускает код миллион раз и даёт честную цифру. Это любимый аргумент на собеседованиях: «как ты проверишь, что оптимизация реально помогла?» — ответ: измерить через `timeit` или профилировщик (`cProfile`).

### Три уровня тестирования

- **doctest** — лёгкие примеры в документации, заодно проверяют, что докстринги не устарели.
- **unittest** — встроенный xUnit-фреймворк для серьёзных наборов тестов с setUp/tearDown, ассертами, моками.
- (вне туториала) **pytest** — сторонний, но де-факто стандарт в индустрии; проще синтаксис.

## Частые вопросы на собеседовании (Q/A)

**В: Чем отличаются `os.system()` и `subprocess`?**
О: `os.system()` — простой запуск команды через оболочку, возвращает только код выхода, не даёт доступа к выводу и опасен из-за shell-инъекций. Современный рекомендованный способ — модуль `subprocess` (`subprocess.run()`), который даёт контроль над stdin/stdout/stderr и безопаснее.

**В: Как правильно собирать пути к файлам, чтобы код работал на Windows и Linux?**
О: Через `os.path.join()` или `pathlib.Path`, а не конкатенацию строк с `/`. Разделители путей различаются между ОС, эти инструменты подставляют правильный.

**В: В чём разница между `glob` и `os.listdir`?**
О: `os.listdir` возвращает все имена в директории, `glob` фильтрует по shell-шаблону (`*.txt`) и умеет рекурсивный поиск (`**`). `glob` удобнее для выборки по маске.

**В: Что такое `sys.argv[0]`?**
О: Имя запущенного скрипта. Реальные переданные аргументы начинаются с `sys.argv[1]`.

**В: Зачем нужен `argparse`, если есть `sys.argv`?**
О: `argparse` автоматически генерирует справку, валидирует и преобразует аргументы, обрабатывает опции и флаги, выдаёт понятные ошибки. Это стандарт для CLI-утилит.

**В: Когда использовать регулярные выражения, а когда методы строк?**
О: Для простых задач (поиск подстроки, замена, split по разделителю) — методы строк (`in`, `.replace()`, `.split()`): они читабельнее и быстрее. Регулярки — когда нужен сложный шаблон (даты, email, группы захвата).

**В: Почему `random.random()` не подходит для криптографии?**
О: Это псевдослучайный генератор (Mersenne Twister), предсказуемый при известном состоянии. Для безопасности (токены, пароли) используют модуль `secrets`.

**В: Чем `stderr` отличается от `stdout`?**
О: Это разные потоки. `stdout` — для данных программы, `stderr` — для ошибок/логов. Они независимо перенаправляются, поэтому ошибки видны даже когда вывод уходит в файл.

**В: Как измерить, какой из двух вариантов кода быстрее?**
О: Модулем `timeit` для микробенчмарков или `cProfile`/`profile` для профилирования целой программы. Не доверять «на глаз».

**В: Что делает `shutil.rmtree()` и чем он опасен?**
О: Рекурсивно удаляет директорию со всем содержимым. Опасен тем, что необратим — легко удалить лишнее, если ошибиться с путём.

**В: Какие три уровня тестов даёт стандартная библиотека?**
О: `doctest` (примеры в докстрингах), `unittest` (xUnit-фреймворк). Сторонний `pytest` популярен, но не входит в stdlib.

**В: Как скачать страницу из интернета средствами стандартной библиотеки?**
О: `urllib.request.urlopen(url)`. На практике чаще берут стороннюю `requests`, но в stdlib есть `urllib`.

## Подводные камни (gotchas)

- **`from os import *`** перекрывает встроенную `open()` функцией `os.open()` (у них разный смысл). Всегда `import os`.
- **`shutil.rmtree()`** удаляет рекурсивно и без подтверждения — проверяйте путь дважды.
- **Склейка путей строками** (`'a' + '/' + 'b'`) ломается на Windows. Используйте `os.path.join`/`pathlib`.
- **Регулярки без `r''`**: `'\d'` может дать предупреждение или неверное поведение из-за двойного экранирования. Пишите `r'\d'`.
- **`random` для безопасности** — нельзя; нужен `secrets`.
- **`urllib` для серьёзных HTTP-запросов** неудобен (заголовки, сессии); индустрия использует `requests`/`httpx`, но на собеседовании знать про `urllib` полезно.
- **`os.system()`** уязвим к shell-инъекциям и не даёт вывод — предпочитайте `subprocess.run()`.
- **`datetime.now()` без часового пояса** возвращает «наивное» время без tz — при работе с разными зонами используйте `timezone`-aware объекты.

## Шпаргалка

| Задача | Модуль / функция |
|---|---|
| Текущая папка, env, список файлов | `os.getcwd()`, `os.environ`, `os.listdir()` |
| Сборка путей кроссплатформенно | `os.path.join()`, `pathlib.Path` |
| Копировать/перемещать/удалять файлы | `shutil.copy/move/rmtree`, `shutil.make_archive` |
| Поиск файлов по маске | `glob.glob('*.py')`, `glob.glob('**/*', recursive=True)` |
| Аргументы CLI (просто) | `sys.argv` |
| Аргументы CLI (по-взрослому) | `argparse.ArgumentParser` |
| Вывод ошибок / выход | `sys.stderr`, `sys.exit(code)` |
| Регулярные выражения | `re.findall / search / sub / match` |
| Математика / случайность / статистика | `math`, `random`, `statistics` |
| Скачать URL / отправить почту | `urllib.request.urlopen`, `smtplib.SMTP` |
| Даты и время | `datetime`, `date`, `timedelta`, `strftime` |
| Сжатие / архивы | `zlib`, `gzip`, `zipfile`, `tarfile` |
| Замер времени | `timeit.Timer(...).timeit()` |
| Тесты в докстрингах / фреймворк | `doctest`, `unittest` |

Главное правило главы: **«Не пиши то, что уже написано в стандартной библиотеке»**.
