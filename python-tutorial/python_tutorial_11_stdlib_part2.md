# Глава 11. Краткий обзор стандартной библиотеки — Часть II — конспект и вопросы

## О чём глава

Вторая экскурсия по стандартной библиотеке — про более «продвинутые» модули, которые нужны в реальных приложениях: красивое форматирование вывода, шаблоны строк, упаковка/распаковка бинарных данных, многопоточность, логирование, управление памятью через слабые ссылки, эффективные структуры данных (deque, heapq, bisect, array) и точная десятичная арифметика. Это инструменты «второго эшелона» — не для каждого скрипта, но критичные для production-кода.

## Ключевые концепции (с примерами и выводом)

### Форматирование вывода: reprlib, pprint, textwrap, locale

**reprlib** — компактный `repr` для больших структур (обрезает длинные коллекции):

```python
import reprlib

print(reprlib.repr(set('supercalifragilisticexpialidocious')))
# => {'a', 'c', 'd', 'e', 'f', 'g', ...}  (показывает не всё, с многоточием)
```

**pprint** — «красивая печать» вложенных структур с отступами и переносами:

```python
import pprint

t = [[[['black', 'cyan'], 'white', ['green', 'red']], [['magenta', 'yellow'], 'blue']]]
pprint.pprint(t, width=30)
# =>
# [[[['black', 'cyan'],
#     'white',
#     ['green', 'red']],
#    [['magenta', 'yellow'],
#     'blue']]]
```

**textwrap** — форматирование абзацев под заданную ширину:

```python
import textwrap

doc = """Метод wrap похож на fill, но возвращает список строк
вместо одной большой строки с переносами."""
print(textwrap.fill(doc, width=40))
# =>
# Метод wrap похож на fill, но возвращает
# список строк вместо одной большой строки
# с переносами.
```

**locale** — форматирование чисел согласно региональным настройкам (разделители групп, валюта):

```python
import locale

locale.setlocale(locale.LC_ALL, 'en_US.UTF-8')
conv = locale.localeconv()
x = 1234567.8
print(locale.format_string("%d", x, grouping=True))  # => 1,234,567
print(locale.format_string("%s%.2f", (conv['currency_symbol'], x), grouping=True))
# => $1,234,567.80
```

### string.Template — шаблоны строк

`Template` даёт упрощённый синтаксис подстановок `$name`, удобный когда шаблоны редактируют пользователи (например, локализация). Безопаснее, чем `format`, при работе с пользовательским вводом.

```python
from string import Template

t = Template('$who любит $what')
print(t.substitute(who='Маша', what='кофе'))
# => Маша любит кофе

# safe_substitute не падает на отсутствующих ключах
t2 = Template('$who любит $what')
print(t2.safe_substitute(who='Петя'))
# => Петя любит $what
```

### struct — упаковка бинарных данных

Модуль `struct` конвертирует между Python-значениями и бинарными C-структурами. Нужен для чтения двоичных форматов файлов, сетевых протоколов.

```python
import struct

# Упаковать: два int (4 байта) и строку
packed = struct.pack('<II', 10, 20)   # '<' = little-endian, 'I' = unsigned int
print(packed)                          # => b'\n\x00\x00\x00\x14\x00\x00\x00'

# Распаковать обратно
a, b = struct.unpack('<II', packed)
print(a, b)                            # => 10 20
```

### threading — многопоточность (кратко)

`threading` позволяет выполнять задачи параллельно, например фоновую работу, не блокируя основной поток. Координация — через события, замки, очереди.

```python
import threading, time

class AsyncWork(threading.Thread):
    def __init__(self, name):
        super().__init__()
        self.name = name
    def run(self):
        print(f'{self.name}: старт фоновой работы')
        time.sleep(1)
        print(f'{self.name}: готово')

worker = AsyncWork('задача-1')
worker.start()              # запустить поток
print('Основной поток продолжает работать')
worker.join()               # дождаться завершения
# =>
# задача-1: старт фоновой работы
# Основной поток продолжает работать
# задача-1: готово
```

Совет из туториала: для координации потоков предпочтительнее использовать `queue.Queue`, чем низкоуровневые замки (`Lock`), — это снижает риск ошибок синхронизации.

### logging — логирование

Гибкая система логов с уровнями важности. Заменяет россыпь `print()` в коде.

```python
import logging

logging.basicConfig(level=logging.INFO,
                    format='%(asctime)s %(levelname)s: %(message)s')

logging.debug('Отладочное сообщение')      # не покажется (уровень ниже INFO)
logging.info('Информационное сообщение')   # => 2026-06-20 ... INFO: Информационное...
logging.warning('Предупреждение!')         # => ... WARNING: Предупреждение!
logging.error('Произошла ошибка')          # => ... ERROR: Произошла ошибка
```

Уровни по возрастанию: DEBUG < INFO < WARNING < ERROR < CRITICAL. По умолчанию выводятся WARNING и выше. Логи можно слать в файл, по сети, в email — без изменения кода приложения.

### weakref — слабые ссылки

Обычная ссылка не даёт объекту удалиться сборщиком мусора. `weakref` позволяет ссылаться на объект, **не** мешая его удалению — нужно для кэшей, где не хотим удерживать объекты в памяти.

```python
import weakref, gc

class A:
    def __init__(self, value):
        self.value = value
    def __repr__(self):
        return str(self.value)

a = A(10)
d = weakref.WeakValueDictionary()
d['primary'] = a            # слабая ссылка, объект не удерживается
print(d['primary'])         # => 10 (объект ещё жив, на него есть ссылка a)

del a                       # удаляем единственную сильную ссылку
gc.collect()                # запускаем сборку мусора
# d['primary'] теперь вызовет KeyError — объект удалён
print('primary' in d)       # => False
```

### Структуры для списков: array, collections.deque, bisect, heapq

**array** — массив одного типа, компактнее списка по памяти:

```python
from array import array

a = array('H', [4000, 10, 700, 22222])  # 'H' = unsigned short (2 байта)
print(sum(a))                            # => 26932
print(a[1:3])                            # => array('H', [10, 700])
```

**collections.deque** — двусторонняя очередь, быстрые добавление/удаление с обоих концов (O(1)):

```python
from collections import deque

d = deque(['task1', 'task2', 'task3'])
d.append('task4')           # добавить справа
print(d.popleft())          # => task1 (быстро взять слева, в отличие от list.pop(0))
d.appendleft('task0')       # добавить слева
print(d)                    # => deque(['task0', 'task2', 'task3', 'task4'])
```

**bisect** — бинарный поиск и вставка в отсортированный список:

```python
import bisect

scores = [(100, 'perl'), (200, 'tcl'), (400, 'lua'), (500, 'python')]
bisect.insort(scores, (300, 'ruby'))  # вставить, сохранив сортировку
print(scores)
# => [(100, 'perl'), (200, 'tcl'), (300, 'ruby'), (400, 'lua'), (500, 'python')]
```

**heapq** — куча (heap), всегда даёт наименьший элемент за O(log n):

```python
import heapq

data = [1, 3, 5, 7, 9, 2, 4, 6, 8, 0]
heapq.heapify(data)         # превратить список в кучу на месте
heapq.heappush(data, -5)    # добавить элемент
print(heapq.heappop(data))  # => -5 (всегда наименьший)
print([heapq.heappop(data) for _ in range(3)])  # => [0, 1, 2] (три наименьших)
```

### decimal — точная десятичная арифметика

`float` хранит числа в двоичной системе и даёт ошибки округления. `Decimal` считает в десятичной — критично для денег, налогов, финансов.

```python
from decimal import Decimal

# Проблема float:
print(0.1 + 0.1 + 0.1 - 0.3)              # => 5.551115123125783e-17 (не ноль!)
print(round(0.70 * 1.05, 2))              # => 0.73 (а должно быть 0.74)

# Decimal считает точно:
print(Decimal('0.1') + Decimal('0.1') + Decimal('0.1') - Decimal('0.3'))
# => 0.0  (ровно ноль)
print(round(Decimal('0.70') * Decimal('1.05'), 2))
# => 0.74  (правильно, как ожидает бухгалтерия)

# Контроль точности
print(Decimal('1') / Decimal('7'))        # => 0.1428571428571428571428571429
```

## Подробный разбор по темам

### Почему deque, а не list для очередей

У `list` операция `pop(0)` / `insert(0, x)` имеет сложность O(n) — нужно сдвигать все элементы. `deque` даёт O(1) на обоих концах, поэтому для очередей (FIFO) и стеков всегда используют `deque`.

### heapq для «top-N» задач

Классический вопрос: «найти N наибольших элементов в большом потоке данных». Решение через полную сортировку — O(n log n) и вся память. Через `heapq.nlargest(N, data)` или кучу размера N — эффективнее по памяти. `heapq` — это min-heap; для max-heap хранят отрицательные значения или используют `heapq.nlargest`.

### bisect: когда список уже отсортирован

Если данные поддерживаются в отсортированном виде, поиск позиции — `bisect.bisect()` за O(log n), вставка с сохранением порядка — `insort()`. Альтернатива линейному поиску O(n).

### float vs Decimal — главный «денежный» вопрос

`float` (двоичный IEEE 754) не может точно представить десятичные дроби вроде 0.1. Для расчётов с деньгами это приводит к копеечным расхождениям, которые накапливаются. `Decimal` хранит десятичные цифры точно и позволяет управлять округлением и точностью. Правило: **деньги — всегда Decimal (или целые числа копеек)**, никогда float.

### Слабые ссылки и кэши

`weakref` решает проблему кэшей: мы хотим помнить объект, пока он используется в программе, но не хотим держать его в памяти только из-за кэша. `WeakValueDictionary`/`WeakKeyDictionary` автоматически удаляют записи, когда объект больше нигде не нужен.

### logging vs print

`print` нельзя выключить без правки кода, он не различает уровни важности и всегда идёт в stdout. `logging` даёт уровни (DEBUG/INFO/WARNING/ERROR), форматирование, метки времени, направление в файлы/сеть и включение/выключение через конфиг. В production-коде используют `logging`, а не `print`.

## Частые вопросы на собеседовании (Q/A)

**В: Почему `0.1 + 0.2 != 0.3` в Python?**
О: `float` хранится в двоичной системе, где 0.1 и 0.2 — бесконечные дроби, поэтому есть ошибка округления. Результат — `0.30000000000000004`. Для точных расчётов используют `decimal.Decimal` или целые копейки.

**В: Когда использовать `Decimal` вместо `float`?**
О: Везде, где важна десятичная точность: деньги, финансы, налоги, проценты. `Decimal` точен в десятичной системе и контролирует округление.

**В: Чем `collections.deque` лучше `list` для очереди?**
О: `deque.popleft()` и `appendleft()` работают за O(1), а у списка `pop(0)`/`insert(0,...)` — O(n) из-за сдвига элементов. Для FIFO-очередей нужен deque.

**В: Что такое `heapq` и для чего он?**
О: Реализация двоичной кучи (min-heap). Даёт наименьший элемент и вставку за O(log n). Применяется для приоритетных очередей и задач «N наибольших/наименьших».

**В: Что делает `bisect`?**
О: Бинарный поиск позиции в отсортированном списке (`bisect`) и вставку с сохранением сортировки (`insort`), оба за O(log n) на поиск.

**В: В чём разница между `array` и `list`?**
О: `array` хранит элементы одного примитивного типа компактно (как C-массив), экономит память. `list` хранит произвольные объекты (ссылки) и универсальнее, но тяжелее.

**В: Зачем нужны слабые ссылки (`weakref`)?**
О: Чтобы ссылаться на объект, не мешая сборщику мусора его удалить. Используются в кэшах и для разрыва циклических ссылок.

**В: Почему `logging` лучше `print` в реальных проектах?**
О: Уровни важности, форматирование с метками времени, направление вывода в файлы/сеть, включение через конфиг без правки кода. `print` всего этого не умеет.

**В: Что такое GIL и как он связан с `threading`?**
О: Global Interpreter Lock — в CPython только один поток исполняет байткод одновременно. Поэтому `threading` ускоряет I/O-bound задачи (ожидание сети/диска), но не CPU-bound. Для CPU-нагрузки — `multiprocessing`.

**В: Зачем нужен `struct`?**
О: Для упаковки/распаковки бинарных данных — чтения двоичных форматов файлов и сетевых протоколов, где важны порядок байт и размеры полей.

**В: Чем `pprint` отличается от обычного `print`?**
О: `pprint` красиво форматирует вложенные структуры с отступами и переносами по ширине, что делает большие словари/списки читаемыми.

**В: В чём смысл `string.Template` против `str.format`/f-строк?**
О: `Template` использует простой синтаксис `$name`, безопаснее при работе с пользовательскими шаблонами (не исполняет произвольный код), удобен для локализации.

## Подводные камни (gotchas)

- **`float` для денег** — копеечные ошибки округления накапливаются. Используйте `Decimal`.
- **`Decimal(0.1)` из float** даёт неточное число; передавайте строку: `Decimal('0.1')`.
- **`list.pop(0)` в цикле** — O(n²) на больших данных; берите `deque`.
- **`heapq` — это min-heap**; для max-heap кладите отрицательные числа или используйте `nlargest`.
- **`threading` и CPU-bound** — из-за GIL не ускорит вычисления; для них нужен `multiprocessing`.
- **`print` вместо `logging`** в библиотеке/сервисе — нельзя отключить и нет уровней.
- **`array` хранит один тип** — попытка положить несовместимый тип вызовет ошибку.
- **`weakref` на встроенные типы** (`list`, `dict`, `int`) не работает напрямую — только на пользовательские классы и некоторые типы.
- **`struct`: не указали порядок байт** (`<`, `>`) — получите платформо-зависимый результат, что ломает переносимость.
