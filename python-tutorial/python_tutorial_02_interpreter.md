# Глава 2. Using the Python Interpreter (Использование интерпретатора) — конспект и вопросы

## О чём глава

Глава объясняет, как запускать Python: интерактивный режим (REPL), запуск скриптов, передача кода и модулей через флаги командной строки, доступ к аргументам через `sys.argv`, кодировка исходных файлов и оформление исполняемых скриптов (shebang).

## Ключевые концепции (с примерами и выводом)

### Запуск интерпретатора и REPL

```text
$ python3
Python 3.12.0 (...)
>>> 2 + 2
4
>>> name = "Аня"
>>> name
'Аня'
>>> exit()        # или quit(), или Ctrl-D (Unix) / Ctrl-Z+Enter (Windows)
```

- `>>>` — основное приглашение (primary prompt).
- `...` — продолжение многострочной конструкции (secondary prompt).

```python
>>> if True:          # после двоеточия REPL ждёт продолжения
...     print("да")   # отступ обязателен
...                   # пустая строка завершает блок
да                    # => да
```

### Запуск скрипта, флаги -c и -m

```bash
python script.py            # запустить файл script.py
python script.py arg1 arg2  # с аргументами командной строки
python -c "print(1+1)"      # выполнить команду-строку  => 2
python -m json.tool data.json  # запустить модуль как скрипт (-m)
python -i script.py         # выполнить скрипт и остаться в интерактивном режиме
```

### Аргументы командной строки: sys.argv

```python
# файл args.py
import sys
print(sys.argv)        # список аргументов; sys.argv[0] — имя скрипта

# Запуск:  python args.py hello 42
# =>  ['args.py', 'hello', '42']
print(sys.argv[0])     # => args.py        (имя самого скрипта)
print(sys.argv[1:])    # => ['hello', '42'] (только переданные аргументы)
```

Особые случаи `sys.argv[0]`:
- `python script.py` → `sys.argv[0] == 'script.py'`
- `python -c "..."` → `sys.argv[0] == '-c'`
- `python -m module` → `sys.argv[0]` — полный путь к модулю
- интерактивный режим / чтение из stdin → `sys.argv[0] == ''`

### Кодировка исходников (UTF-8 и coding declaration)

```python
# По умолчанию исходные файлы Python 3 — в кодировке UTF-8.
# Кириллица и эмодзи в строках и комментариях работают без объявлений:
сообщение = "Привет, 🌍"
print(сообщение)        # => Привет, 🌍

# Если нужна ДРУГАЯ кодировка файла, объявляют её в первой или второй строке:
# -*- coding: cp1251 -*-
# или короткой формой, которую распознаёт Python:
# coding: latin-1
```

### Shebang — исполняемый скрипт на Unix

```python
#!/usr/bin/env python3
# Первая строка-«шебанг» позволяет запускать файл напрямую: ./script.py
# env находит python3 в PATH (переносимее, чем жёсткий путь /usr/bin/python3)
print("Запущено напрямую")
```

```bash
chmod +x script.py   # сделать файл исполняемым
./script.py          # запуск без явного вызова python
```

## Подробный разбор по темам

### Интерактивный режим (REPL)
- REPL = Read-Eval-Print Loop: читает выражение, вычисляет, печатает результат, повторяет.
- В интерактивном режиме результат выражения **автоматически печатается** (через `repr`), в скрипте — нет.

```python
>>> "строка"        # repr — с кавычками
'строка'
>>> print("строка") # print — без кавычек
строка
```

- Переменная `_` хранит результат последнего выражения (только в REPL):

```python
>>> 10 * 5
50
>>> _ + 2
52
```

### Передача программы интерпретатору — три способа
1. **Файл-скрипт**: `python script.py` — самый частый способ.
2. **`-c command`**: выполнить строку кода. Удобно для однострочников; строку берут в кавычки.
3. **`-m module`**: найти модуль в `sys.path` и запустить как скрипт. Запускается `__main__`-часть модуля. Примеры: `python -m http.server`, `python -m pip install`, `python -m venv env`, `python -m json.tool`.

Преимущество `-m`: не нужно знать путь к файлу модуля, и каталог запуска добавляется в `sys.path` корректно.

### Флаги, полезные на практике
- `-i` — после выполнения скрипта войти в интерактивный режим (удобно для отладки, доступны все переменные).
- `-O` — оптимизация: убирает `assert` и код под `if __debug__`.
- `-V` / `--version` — версия Python.
- `-B` — не писать `.pyc` файлы.

### Кодировка: детали
- Python 3: исходники по умолчанию **UTF-8**. Идентификаторы могут содержать Unicode (например, кириллицу), хотя это не рекомендуется для общего кода.
- Объявление кодировки нужно лишь когда файл не в UTF-8. Формат строки: `# -*- coding: <encoding> -*-` либо `# coding: <encoding>` в первой/второй строке.
- Важно не путать кодировку **файла** (как байты на диске интерпретируются как текст) и кодировку **ввода/вывода** (как `print` пишет в терминал).

### Shebang и переносимость
- `#!/usr/bin/env python3` предпочтительнее `#!/usr/bin/python3`, т.к. ищет интерпретатор в `PATH` (работает в виртуальных окружениях и нестандартных установках).
- На Windows shebang самим ОС не используется, но его учитывает Python Launcher `py`.

## Частые вопросы на собеседовании (Q/A)

**Q1. Что такое REPL?**
A: Read-Eval-Print Loop — интерактивная оболочка Python: читает введённое выражение, вычисляет, печатает результат и ждёт следующего ввода. Запускается командой `python` без аргументов.

**Q2. Чем `>>>` отличается от `...`?**
A: `>>>` — основное приглашение для новой команды. `...` — приглашение продолжения, появляется внутри многострочных конструкций (блоки `if`, `for`, `def` и т.п.). Пустая строка завершает блок.

**Q3. Что хранится в `sys.argv` и что такое `sys.argv[0]`?**
A: `sys.argv` — список аргументов командной строки (строки). `sys.argv[0]` — имя/путь самого скрипта, а `sys.argv[1:]` — переданные пользователем аргументы.

**Q4. В чём разница между `python -c` и `python -m`?**
A: `-c "code"` выполняет переданную строку кода. `-m module` находит модуль в `sys.path` и запускает его как скрипт (как `__main__`), не требуя пути к файлу. Пример `-m`: `python -m http.server`.

**Q5. Какая кодировка у исходников Python по умолчанию и как её изменить?**
A: По умолчанию UTF-8 (в Python 3). Изменить можно объявлением в первой или второй строке файла: `# -*- coding: cp1251 -*-`.

**Q6. Что такое shebang и зачем `#!/usr/bin/env python3`?**
A: Shebang — первая строка `#!...`, указывающая ОС (Unix), каким интерпретатором запускать файл. Форма с `env` ищет `python3` в `PATH`, что переносимее жёсткого пути и работает с виртуальными окружениями.

**Q7. Почему в REPL результат печатается сам, а в скрипте — нет?**
A: REPL для каждого введённого выражения вызывает `repr()` и печатает результат (кроме `None`). В скрипте выражения вычисляются, но не печатаются — для вывода нужен явный `print()`.

**Q8. Что делает переменная `_` в интерактивном режиме?**
A: Хранит результат последнего вычисленного выражения, позволяя переиспользовать его (`_ + 2`). Это поведение только REPL; в обычном коде `_` — просто имя.

**Q9. Как выполнить скрипт и остаться в интерактивном режиме?**
A: Флагом `-i`: `python -i script.py`. После выполнения скрипта откроется REPL со всеми его переменными — удобно для отладки.

**Q10. Как корректно выйти из интерпретатора?**
A: `exit()`, `quit()`, либо Ctrl-D (Unix) / Ctrl-Z затем Enter (Windows).

**Q11. Почему shebang не «срабатывает» при `python script.py`?**
A: Shebang — инструкция для загрузчика ОС при прямом запуске `./script.py`. Когда вы пишете `python script.py`, интерпретатор уже выбран явно, и первая строка трактуется просто как комментарий.

## Подводные камни (gotchas)

- **`sys.argv[0]` — это не первый аргумент пользователя.** Первый пользовательский аргумент — `sys.argv[1]`. Частая ошибка off-by-one.
- **Все элементы `sys.argv` — строки.** Число `42` придёт как `'42'`; нужно явно приводить (`int(sys.argv[1])`).
- **Объявление кодировки работает только в первой/второй строке.** Если поставить его ниже — Python проигнорирует.
- **Shebang не работает без `chmod +x`** и не учитывается при запуске через `python file.py`.
- **`-c` и кавычки в shell.** В однострочниках следите за экранированием кавычек; внутренние кавычки часто конфликтуют с оболочкой.
- **`-O` убирает `assert`.** Нельзя полагаться на `assert` для проверок, влияющих на логику в продакшене.
- **Переменная `_` существует только в REPL.** В скриптах после `import gettext` `_` может означать функцию перевода — не путать.

## Шпаргалка

- Запуск REPL: `python` (без аргументов). Выход: `exit()` / Ctrl-D.
- Приглашения: `>>>` основное, `...` продолжение блока.
- Запуск кода: `python file.py` | `python -c "code"` | `python -m module`.
- Отладка: `python -i file.py` (остаться в REPL после скрипта).
- Аргументы: `import sys; sys.argv` — список строк; `[0]` — имя скрипта, `[1:]` — аргументы пользователя.
- Кодировка по умолчанию: UTF-8. Сменить: `# -*- coding: cp1251 -*-` в строке 1-2.
- Shebang (Unix): `#!/usr/bin/env python3` + `chmod +x` → `./script.py`.
- `_` в REPL — последний результат.
- Полезные `-m`: `http.server`, `pip`, `venv`, `json.tool`, `pytest`.
