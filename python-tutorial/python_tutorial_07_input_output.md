# Глава 7. Ввод и вывод (Input and Output) — конспект и вопросы

## О чём глава

Глава 7 туториала Python рассказывает, как программа форматирует вывод и
обменивается данными с внешним миром — прежде всего с файлами. Темы:

- форматирование строк: **f-strings** (подробно), `str.format()`, старый стиль `%`;
- разница между `str()` и `repr()`;
- чтение и запись файлов: `open`, режимы, менеджер контекста `with`, методы
  `read`/`readline`/`readlines`/`write`, итерация по файлу, `seek`/`tell`;
- сохранение структурированных данных через модуль `json` (`dump`/`load`);
- кратко о кодировках.

---

## Ключевые концепции (с примерами и выводом)

### f-strings — основной способ форматирования (Python 3.6+)

```python
name, age = "Иван", 30
print(f"{name} — {age} лет")            # => Иван — 30 лет
print(f"2 + 2 = {2 + 2}")               # выражение внутри => 2 + 2 = 4
pi = 3.14159
print(f"pi = {pi:.2f}")                 # спецификатор => pi = 3.14
print(f"{name=}")                        # отладка (3.8+) => name='Иван'
```

### str.format() и старый % — альтернативы

```python
print("{} — {} лет".format("Иван", 30)) # => Иван — 30 лет
print("{0} {1} {0}".format("a", "b"))   # по индексу => a b a
print("pi = %.2f" % 3.14159)            # старый стиль => pi = 3.14
```

### str() vs repr()

```python
import datetime
d = datetime.date(2026, 6, 20)
print(str(d))    # => 2026-06-20         (человекочитаемо)
print(repr(d))   # => datetime.date(2026, 6, 20)  (для разработчика)
print(str("hi")) # => hi
print(repr("hi"))# => 'hi'  (с кавычками)
```

### Чтение/запись файлов через with

```python
with open("file.txt", "w", encoding="utf-8") as f:
    f.write("первая строка\n")
    f.write("вторая строка\n")

with open("file.txt", "r", encoding="utf-8") as f:
    for line in f:                       # ленивое чтение по строкам
        print(line.rstrip())             # => первая строка / вторая строка
```

### JSON — сериализация структур

```python
import json
data = {"name": "Иван", "ages": [30, 31], "active": True}
text = json.dumps(data, ensure_ascii=False)   # объект -> строка
print(text)        # => {"name": "Иван", "ages": [30, 31], "active": true}
back = json.loads(text)                        # строка -> объект
print(back["name"])# => Иван
```

## Подробный разбор по темам

### 1. f-strings (форматированные строковые литералы) — подробно

f-string — это строковый литерал с префиксом `f` (или `F`), внутри которого в
фигурных скобках `{}` можно писать **произвольные выражения Python**. Они
вычисляются во время выполнения и подставляются в строку.

**Выражения внутри**:

```python
print(f"{2 ** 10}")                     # => 1024
items = [1, 2, 3]
print(f"Сумма = {sum(items)}")          # => Сумма = 6
user = {"name": "Анна"}
print(f"Привет, {user['name']}!")       # => Привет, Анна!  (внутри ' если снаружи ")
print(f"{'верх'.upper()}")              # вызов метода => ВЕРХ
```

**Отладочный `=`** (Python 3.8+) — печатает и выражение, и значение:

```python
x = 42
print(f"{x=}")              # => x=42
print(f"{x * 2=}")          # => x * 2=84
print(f"{x = }")            # пробелы сохраняются => x = 42
```

Очень удобно для быстрой отладки вместо `print("x =", x)`.

**Спецификаторы формата** (после двоеточия) — мини-язык форматирования:

```python
pi = 3.14159265
print(f"{pi:.2f}")          # 2 знака после запятой => 3.14
print(f"{pi:10.3f}")        # ширина 10, 3 знака => '     3.142'
print(f"{pi:<10.3f}")       # выравнивание влево  => '3.142     '
print(f"{pi:^10.3f}")       # по центру           => '  3.142   '
print(f"{1000000:,}")       # разделитель тысяч   => 1,000,000
print(f"{1000000:_}")       # подчёркивание       => 1_000_000
print(f"{255:#x}")          # hex с префиксом      => 0xff
print(f"{255:08b}")         # двоичное, ширина 8   => 11111111
print(f"{0.1234:.1%}")      # проценты             => 12.3%
print(f"{42:+}")            # всегда со знаком      => +42
print(f"{'hi':>8}")         # строка, вправо       => '      hi'
print(f"{'hi':*^8}")        # заполнитель *         => '***hi***'
```

Ширину/точность можно задавать динамически через вложенные `{}`:

```python
width, prec = 10, 3
print(f"{pi:{width}.{prec}f}")          # => '     3.142'
```

**Конверсии `!r`, `!s`, `!a`** — применяют `repr`, `str`, `ascii` к значению:

```python
name = "Иван"
print(f"{name!r}")          # repr => 'Иван' (с кавычками)
print(f"{name!s}")          # str  => Иван
print(f"{name!a}")          # ascii => 'Иван'
```

`!r` очень полезен в логах/сообщениях об ошибках, чтобы видеть кавычки и тип:

```python
value = None
print(f"Получено значение: {value!r}")  # => Получено значение: None
```

Чтобы вставить **буквальную фигурную скобку**, её удваивают:

```python
print(f"{{не выражение}} и {2+2}")      # => {не выражение} и 4
```

### 2. str.format()

Более старый, но всё ещё используемый способ. Работает по позиции, по индексу
или по имени.

```python
print("{} и {}".format("a", "b"))               # позиционно => a и b
print("{1} и {0}".format("a", "b"))             # по индексу => b и a
print("{name}: {val}".format(name="x", val=1))  # по имени   => x: 1

# Спецификаторы те же, что в f-strings:
print("{:.2f}".format(3.14159))                 # => 3.14
print("{:>10}".format("hi"))                    # => '        hi'

# Распаковка словаря:
d = {"name": "Иван", "age": 30}
print("{name}, {age}".format(**d))              # => Иван, 30
```

f-strings — это, по сути, более удобный синтаксис над тем же мини-языком
форматирования, поэтому все спецификаторы (`:.2f`, `:>10` и т.д.) общие.

### 3. Старый стиль % (printf-style)

Унаследованный из C способ. Встречается в старом коде и в некоторых API
(например, в стандартном `logging`).

```python
print("%s — %d лет" % ("Иван", 30))     # => Иван — 30 лет
print("%.2f" % 3.14159)                 # => 3.14
print("%05d" % 42)                      # => 00042
print("%(name)s" % {"name": "Иван"})    # по имени => Иван
```

Минусы: легко ошибиться с типом (`%d` для строки → ошибка), хуже читаемость при
многих аргументах. Для нового кода предпочитают f-strings.

### 4. str() vs repr()

Два способа получить строковое представление объекта:

- `str(obj)` — **читаемое** представление для пользователя (вызывает `__str__`).
- `repr(obj)` — **однозначное**, «для разработчика», в идеале такое, что
  `eval(repr(obj)) == obj` (вызывает `__repr__`).

```python
s = "строка\nс переносом"
print(str(s))    # печатает с реальным переносом строки
print(repr(s))   # => 'строка\nс переносом'  (видны кавычки и спецсимволы)

print(str(42))   # => 42
print(repr(42))  # => 42 (для int совпадают)
```

Если у класса определён только `__repr__`, он используется и как fallback для
`str`. В контейнерах элементы всегда показываются через `repr`:

```python
print(["a", "b"])         # => ['a', 'b']  (repr элементов — с кавычками)
print(str(["a", "b"]))    # => ['a', 'b']  (то же)
```

```python
class Point:
    def __init__(self, x, y):
        self.x, self.y = x, y
    def __repr__(self):
        return f"Point({self.x!r}, {self.y!r})"   # для разработчика
    def __str__(self):
        return f"({self.x}, {self.y})"            # для пользователя

p = Point(1, 2)
print(str(p))    # => (1, 2)
print(repr(p))   # => Point(1, 2)
print(p)         # print использует str => (1, 2)
print([p])       # контейнер использует repr => [Point(1, 2)]
```

Рекомендация: всегда определять `__repr__` (полезно в логах и отладке), а
`__str__` — по необходимости.

`str.rjust`/`ljust`/`center`/`zfill` — старые методы выравнивания строк, которые
тоже упоминаются в главе:

```python
print("42".zfill(5))        # => 00042
print("hi".rjust(6))        # => '    hi'
print("hi".center(6, "*"))  # => '**hi**'
```

### 5. Открытие файлов: open и режимы

```python
f = open(filename, mode="r", encoding=None)
```

Основные режимы:

| Режим | Значение |
|-------|----------|
| `"r"` | чтение (по умолчанию); ошибка, если файла нет |
| `"w"` | запись; **создаёт/обнуляет** файл (truncate!) |
| `"a"` | дозапись в конец; создаёт, если нет |
| `"x"` | эксклюзивное создание; ошибка, если файл уже есть |
| `"r+"` | чтение и запись (файл должен существовать) |
| `"b"` | бинарный режим (добавляется к другим: `"rb"`, `"wb"`) |
| `"t"` | текстовый режим (по умолчанию) |

Текстовый режим работает со строками `str` и выполняет декодирование/кодирование
по `encoding`; бинарный — с `bytes` и не трогает байты.

> Всегда указывайте `encoding="utf-8"` явно в текстовом режиме: по умолчанию
> берётся локаль-зависимая кодировка, что приводит к разному поведению на разных
> ОС. Бинарный режим — для не-текстовых данных (картинки, архивы).

### 6. Менеджер контекста with

`with` гарантирует закрытие файла даже при исключении — это идиоматичный способ
работы с файлами.

```python
with open("file.txt", encoding="utf-8") as f:
    data = f.read()
# здесь файл уже автоматически закрыт, даже если внутри было исключение
print(f.closed)     # => True
```

Без `with` пришлось бы вручную писать `try/finally` с `f.close()`. Если не
закрыть файл, можно потерять данные (буфер не сброшен) или исчерпать дескрипторы.

Можно открыть несколько файлов в одном `with`:

```python
with open("in.txt", encoding="utf-8") as src, \
     open("out.txt", "w", encoding="utf-8") as dst:
    dst.write(src.read())
```

### 7. Методы чтения и записи

```python
with open("file.txt", encoding="utf-8") as f:
    f.read()          # весь файл как одна строка (или read(n) — n символов)
    f.readline()      # одна строка (включая \n; '' в конце файла)
    f.readlines()     # список ВСЕХ строк (загружает всё в память)
```

**Итерация по файлу** — самый эффективный и идиоматичный способ читать построчно
(ленивый, не грузит весь файл):

```python
with open("file.txt", encoding="utf-8") as f:
    for line in f:                  # построчно, экономно по памяти
        process(line.rstrip("\n"))  # обычно срезают перевод строки
```

> `read()` и `readlines()` грузят весь файл в память — плохо для больших файлов.
> Для них используйте итерацию `for line in f:`.

**Запись**:

```python
with open("out.txt", "w", encoding="utf-8") as f:
    f.write("текст\n")              # write принимает строку, возвращает число символов
    n = f.write("ещё\n")
    print(n)                        # => 4 (число записанных символов)
    f.writelines(["a\n", "b\n"])    # записать список строк (без добавления \n!)
```

`write` не добавляет перевод строки автоматически — его нужно ставить вручную.
Чтобы записать число, его сначала конвертируют в строку: `f.write(str(42))`.

### 8. seek и tell — позиция в файле

```python
with open("data.bin", "wb") as f:
    f.write(b"0123456789abcdef")

with open("data.bin", "rb") as f:
    print(f.tell())     # => 0  (текущая позиция в байтах)
    print(f.read(4))    # => b'0123'
    print(f.tell())     # => 4
    f.seek(2)           # перейти на байт 2 от начала
    print(f.read(1))    # => b'2'
    f.seek(-3, 2)       # 3 байта от КОНЦА (whence=2)
    print(f.read(1))    # => b'd'
```

`seek(offset, whence)`:
- `whence=0` (по умолчанию) — от начала;
- `whence=1` — от текущей позиции;
- `whence=2` — от конца.

В **текстовом** режиме `seek` ограничен: разрешён только `seek(0)` и переходы на
ранее полученные через `tell()` смещения (из-за переменной длины символов в
кодировках). Полноценная навигация по смещениям — в бинарном режиме.

### 9. Сохранение структурированных данных через json

Файлы хранят строки/байты, поэтому чтобы сохранить вложенные структуры (списки,
словари, числа), их сериализуют. Модуль `json` — стандартный текстовый формат,
понятный многим языкам.

```python
import json

data = {"name": "Иван", "ages": [30, 31], "active": True, "note": None}

# Сериализация в строку:
s = json.dumps(data)                 # компактно, не-ASCII экранируется
print(s)  # => {"name": "Иван", ... "active": true, "note": null}

# Читаемо и с кириллицей:
s = json.dumps(data, ensure_ascii=False, indent=2, sort_keys=True)
print(s)
# {
#   "active": true,
#   "ages": [ 30, 31 ],
#   "name": "Иван",
#   "note": null
# }

# Десериализация из строки:
obj = json.loads(s)
print(obj["ages"][0])                # => 30
```

`dump`/`load` — то же, но напрямую с файловым объектом:

```python
with open("data.json", "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False, indent=2)   # объект -> файл

with open("data.json", encoding="utf-8") as f:
    data = json.load(f)                                  # файл -> объект
```

Соответствие типов Python ↔ JSON:

| Python | JSON |
|--------|------|
| `dict` | object |
| `list`, `tuple` | array |
| `str` | string |
| `int`, `float` | number |
| `True` / `False` | `true` / `false` |
| `None` | `null` |

Важно: `tuple` сериализуется как массив и при загрузке станет `list`. Ключи
словаря в JSON всегда строки (числовые ключи становятся строками). Произвольные
объекты (например, `datetime`, `set`, пользовательские классы) по умолчанию **не
сериализуются** — нужен параметр `default=` или собственный энкодер.

> Не используйте `pickle` для данных из недоверенных источников — десериализация
> pickle может выполнить произвольный код. JSON безопаснее и переносимее для
> обмена данными; pickle — только для доверенного внутреннего хранения
> Python-объектов.

### 10. Кодировки (кратко)

Текст в файлах хранится как байты; кодировка задаёт соответствие символ ↔ байты.
UTF-8 — стандарт де-факто, поддерживает все символы Unicode.

```python
# Явное кодирование/декодирование строк:
b = "Привет".encode("utf-8")     # str -> bytes
print(b)                          # => b'\xd0\x9f\xd1\x80\xd0\xb8...'
s = b.decode("utf-8")             # bytes -> str
print(s)                          # => Привет

# Всегда указывайте encoding при открытии текстовых файлов:
open("f.txt", "w", encoding="utf-8")
```

Неверная кодировка приводит к `UnicodeDecodeError` или «кракозябрам». Параметр
`errors=` (`"strict"` по умолчанию, `"ignore"`, `"replace"`) управляет
обработкой ошибок декодирования. С Python 3.15 UTF-8 становится кодировкой по
умолчанию для текстовых файлов, но указывать `encoding` явно — хорошая привычка
для совместимости.

## Частые вопросы на собеседовании (Q/A)

**Q1. Чем f-strings лучше `str.format()` и `%`?**
A. f-strings читаемее (выражение видно прямо в месте вставки), быстрее
(вычисляются в момент компиляции литерала), поддерживают любые выражения и
отладочный `=`. `format()` и `%` многословнее и легче ошибиться с порядком/типом
аргументов. Все используют один мини-язык спецификаторов.

**Q2. В чём разница между `str()` и `repr()`?**
A. `str()` даёт читаемое представление для пользователя (`__str__`), `repr()` —
однозначное «для разработчика» (`__repr__`), в идеале воспроизводимое через
`eval`. У строк `repr` добавляет кавычки и показывает спецсимволы. Контейнеры
печатают элементы через `repr`. Если `__str__` не задан, используется `__repr__`.

**Q3. Что делает конверсия `!r` в f-string?**
A. Применяет `repr()` к значению вместо `str()`. Полезно в логах: `f"{x!r}"`
покажет кавычки у строк и тип, помогая отличить `"5"` от `5`.

**Q4. Зачем нужен `with` при работе с файлами?**
A. `with` — менеджер контекста: он гарантированно закрывает файл при выходе из
блока, в том числе при исключении. Это заменяет ручной `try/finally` с
`close()`, предотвращает утечку дескрипторов и потерю данных из буфера.

**Q5. В чём разница между режимами `"w"` и `"a"`?**
A. `"w"` создаёт новый файл или **обнуляет** существующий (truncate), запись
идёт с начала. `"a"` дозаписывает в конец, не стирая содержимое; создаёт файл,
если его нет.

**Q6. Чем отличаются `read()`, `readline()`, `readlines()` и `for line in f`?**
A. `read()` — весь файл одной строкой (или `read(n)` символов). `readline()` —
одна строка. `readlines()` — список всех строк (всё в памяти). `for line in f` —
ленивая построчная итерация, самый экономный по памяти способ для больших файлов.

**Q7. Почему для больших файлов нельзя `read()`/`readlines()`?**
A. Они загружают весь файл в память целиком. Для гигабайтных файлов это вызовет
перерасход памяти. Нужно итерироваться построчно `for line in f`.

**Q8. Что делают `seek` и `tell`?**
A. `tell()` возвращает текущую позицию (в байтах для бинарного режима).
`seek(offset, whence)` перемещает позицию: `whence` 0 — от начала, 1 — от
текущей, 2 — от конца. В текстовом режиме навигация ограничена.

**Q9. В чём разница `json.dumps`/`loads` и `json.dump`/`load`?**
A. `dumps`/`loads` работают со **строкой** (s — string): объект↔строка.
`dump`/`load` — напрямую с **файловым объектом**: объект↔файл.

**Q10. Что произойдёт с кортежем и множеством при сериализации в JSON?**
A. `tuple` станет JSON-массивом и при загрузке вернётся как `list`. `set` по
умолчанию не сериализуется (`TypeError`) — нужно конвертировать в список или
задать `default=`.

**Q11. Зачем `ensure_ascii=False` в `json.dumps`?**
A. По умолчанию не-ASCII символы экранируются (`\uXXXX`). `ensure_ascii=False`
сохраняет кириллицу и другие символы как есть — читаемее и компактнее (с
`encoding="utf-8"` при записи).

**Q12. Почему JSON безопаснее pickle для обмена данными?**
A. JSON — текстовый формат данных и не исполняет код. `pickle.load` из
недоверенного источника может выполнить произвольный код (уязвимость). JSON
также переносим между языками.

**Q13. Зачем явно указывать `encoding="utf-8"`?**
A. Без него берётся кодировка по умолчанию, зависящая от ОС/локали, что ломает
переносимость и приводит к `UnicodeDecodeError`/«кракозябрам». Явный UTF-8
делает поведение одинаковым везде.

**Q14. Добавляет ли `write` перевод строки? А `writelines`?**
A. Нет, оба не добавляют `\n` — его нужно ставить вручную. `write` принимает
одну строку, `writelines` — итерируемое строк, но также без добавления
разделителей.

## Подводные камни (gotchas)

1. **Режим `"w"` стирает файл.** Открытие существующего файла в `"w"` обнуляет
   его содержимое ещё до первой записи. Для дозаписи — `"a"`.

2. **Файл не закрыт без `with`.** Забытый `close()` → данные могут остаться в
   буфере и не записаться, а дескрипторы — утечь. Всегда `with`.

3. **`read()`/`readlines()` на большом файле** загружают всё в память.
   Используйте `for line in f`.

4. **`write` не добавляет `\n`.** Строки склеятся, если не ставить перевод
   вручную.

5. **`write` принимает только строки** (в текстовом режиме). `f.write(42)` →
   `TypeError`; нужно `f.write(str(42))`.

6. **Отсутствие `encoding`** ведёт к платформозависимому поведению и ошибкам с
   не-ASCII текстом.

7. **`json` теряет типы.** `tuple` → `list`, числовые ключи словаря → строки,
   `set`/`datetime`/классы не сериализуются без `default`.

8. **`seek` в текстовом режиме ограничен** — нельзя произвольно прыгать по
   байтовым смещениям; для этого нужен бинарный режим.

9. **Двойные `{{ }}` в f-strings** — чтобы напечатать литеральную фигурную
   скобку, её надо удвоить.

10. **`repr` vs `str` в логах.** Логирование значения без `!r` может скрыть, что
    это строка с пробелами/спецсимволами или `None`. Используйте `{x!r}`.

11. **Не сравнивайте `readline()` с `None`** для конца файла — в конце он
    возвращает пустую строку `''`, а не `None`.

## Шпаргалка

```python
# --- Форматирование ---
f"{value}"                      # подстановка
f"{expr + 1}"                   # выражение
f"{x=}"                         # отладка: x=<значение>  (3.8+)
f"{pi:.2f}"                     # 2 знака после запятой
f"{n:>10}", f"{n:<10}", f"{n:^10}"   # выравнивание: вправо/влево/центр
f"{n:,}", f"{n:_}"              # разделители тысяч
f"{255:#x}", f"{n:08b}"         # hex / двоичное с дополнением
f"{val!r}"                      # repr;  !s — str;  !a — ascii
"{} {}".format(a, b)            # str.format()
"%.2f" % x                      # старый стиль

# --- str vs repr ---
str(obj)    # читаемо (__str__)
repr(obj)   # однозначно, для разработчика (__repr__)

# --- Файлы ---
with open("f.txt", "r", encoding="utf-8") as f:
    f.read()        # весь файл  | f.read(n) — n символов
    f.readline()    # одна строка
    f.readlines()   # список строк (всё в память)
    for line in f:  # ленивая построчная итерация (рекомендуется)
        ...

with open("f.txt", "w", encoding="utf-8") as f:  # "w" обнуляет, "a" дозапись, "x" эксклюзивно
    f.write("текст\n")          # \n вручную!
    f.writelines(["a\n", "b\n"])

f.tell()                        # позиция
f.seek(0)                       # в начало;  seek(off, whence) whence 0/1/2

# --- JSON ---
json.dumps(obj, ensure_ascii=False, indent=2)   # объект -> строка
json.loads(s)                                    # строка -> объект
json.dump(obj, f, ensure_ascii=False)            # объект -> файл
json.load(f)                                      # файл -> объект
# tuple->array(list при load), True->true, None->null, ключи -> строки

# --- Кодировки ---
"текст".encode("utf-8")         # str -> bytes
b"...".decode("utf-8")          # bytes -> str
open(..., encoding="utf-8")     # всегда указывайте явно
```
