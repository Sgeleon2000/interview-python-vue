## О чём глава

Первое знакомство с языком на практике: числа и арифметика, строки (литералы, экранирование, raw- и многострочные строки, индексация, срезы, неизменяемость, методы), списки (операции, срезы, изменяемость) и первые шаги в программировании — множественное присваивание, цикл `while`, последовательность Фибоначчи.

## Ключевые концепции (с примерами и выводом)

### Числа: int, float, операторы

```python
2 + 2          # => 4
50 - 5*6       # => 20      (приоритет операций как в математике)
(50 - 5*6) / 4 # => 5.0     (деление / всегда даёт float)
17 / 3         # => 5.666666666666667   (истинное деление)
17 // 3        # => 5       (целочисленное деление, отбрасывает дробь "вниз")
17 % 3         # => 2       (остаток от деления)
5 ** 2         # => 25      (возведение в степень)
2 ** 10        # => 1024
```

```python
# Типы и смешанная арифметика
type(3)        # => <class 'int'>
type(3.0)      # => <class 'float'>
4 * 3.75       # => 15.0    (int * float -> float, float "заразен")
3 * 3.75 / 1.5 # => 7.5

# int в Python имеет неограниченную точность (длинная арифметика):
2 ** 100       # => 1267650600228229401496703205376
```

### Округление целочисленного деления вниз (важный нюанс)

```python
7 // 2     # => 3
-7 // 2    # => -4    (НЕ -3! Округление к минус бесконечности, floor)
7 % -3     # => -2    (знак остатка совпадает со знаком делителя)
# Инвариант:  (a // b) * b + (a % b) == a
```

### Строки: литералы и кавычки

```python
'spam eggs'          # => 'spam eggs'      (одинарные кавычки)
"spam eggs"          # => 'spam eggs'      (двойные — эквивалентны)
'doesn\'t'           # => "doesn't"        (экранирование апострофа)
"doesn't"            # => "doesn't"        (или просто другие кавычки)
'"Yes," they said.'  # => '"Yes," they said.'
```

```python
text = """Первая строка

# Вторая строка
```

### Конкатенация и повторение

```python
'Py' + 'thon'        # => 'Python'   (склейка)
'ab' * 3             # => 'ababab'    (повторение)

# Соседние СТРОКОВЫЕ ЛИТЕРАЛЫ склеиваются автоматически:
'Py' 'thon'          # => 'Python'
text = ('Очень длинная '   # удобно для разбивки длинных строк
        'строка в коде')
# => 'Очень длинная строка в коде'

# НО переменные так склеить нельзя — только литералы:
prefix = 'Py'
# prefix 'thon'   -> SyntaxError
prefix + 'thon'      # => 'Python'   (правильно)
```

### Индексация и срезы строк

```python
word = 'Python'
word[0]     # => 'P'    (первый символ, индексы с 0)
word[5]     # => 'n'    (последний)
word[-1]    # => 'n'    (последний через отрицательный индекс)
word[-2]    # => 'o'
word[-6]    # => 'P'

word[0:2]   # => 'Py'   (символы с 0 до 2, НЕ включая 2)
word[2:5]   # => 'tho'
word[:2]    # => 'Py'   (с начала)
word[4:]    # => 'on'   (до конца)
word[-2:]   # => 'on'   (последние два)

# Срез всегда возвращает корректную подстроку, инвариант:
# word[:i] + word[i:] == word
word[:2] + word[2:]   # => 'Python'
```

```python
# Выход за границы: индекс — ошибка, срез — безопасен
# word[42]      -> IndexError
word[42:]       # => ''   (срез не падает)
word[4:42]      # => 'on'
```

### Неизменяемость строк

```python
word = 'Python'
# word[0] = 'J'    -> TypeError: строки immutable
new = 'J' + word[1:]   # => 'Jython'   (создаём новую строку)
len('Python')          # => 6
```

### Списки: базовые операции

```python
squares = [1, 4, 9, 16, 25]
squares[0]        # => 1
squares[-1]       # => 25
squares[-3:]      # => [9, 16, 25]   (срез возвращает НОВЫЙ список)
squares[:]        # => [1, 4, 9, 16, 25]   (поверхностная копия)
squares + [36, 49]  # => [1, 4, 9, 16, 25, 36, 49]   (конкатенация)
len(squares)      # => 5

# Вложенные списки
matrix = [[1, 2], [3, 4]]
matrix[0][1]      # => 2
```

### Изменяемость списков (в отличие от строк)

```python
cubes = [1, 8, 27, 65, 125]
cubes[3] = 64          # списки изменяемы — присваивание по индексу работает
print(cubes)           # => [1, 8, 27, 64, 125]

cubes.append(216)      # добавить в конец
cubes.append(7 ** 3)   # => [..., 216, 343]

letters = ['a', 'b', 'c', 'd']
letters[1:3] = ['x', 'y']   # замена среза
print(letters)              # => ['a', 'x', 'y', 'd']
letters[1:3] = []           # удаление среза
print(letters)              # => ['a', 'd']
letters[:] = []             # очистить весь список
print(letters)              # => []
```

### Первые шаги: множественное присваивание и Фибоначчи

```python
# Множественное (параллельное) присваивание: правая часть вычисляется ПОЛНОСТЬЮ,
# потом распаковывается. Поэтому обмен значений — в одну строку:
a, b = 0, 1
a, b = b, a + b   # сначала считается (b, a+b), затем присваивается a и b

# Ряд Фибоначчи
a, b = 0, 1
while a < 10:
    print(a, end=' ')   # end=' ' — печать в одну строку с пробелом
    a, b = b, a + b
print()
# => 0 1 1 2 3 5 8
```

## Подробный разбор по темам

### Числа
- `int` — целые произвольной точности (нет переполнения, как в C/Java).
- `float` — числа с плавающей точкой (double, ~15-17 значащих цифр).
- `/` — истинное деление, **всегда** даёт `float` (даже `4 / 2 == 2.0`).
- `//` — целочисленное деление с округлением **вниз** (floor), к минус бесконечности.
- `%` — остаток; в Python знак результата совпадает со знаком **делителя**.
- `**` — степень; правоассоциативна: `2 ** 3 ** 2 == 2 ** 9 == 512`.
- `complex` — есть и комплексные числа: `3 + 4j`.

### Строки
- Кавычки `'...'` и `"..."` равнозначны; выбор обычно по тому, какие кавычки внутри текста.
- Экранирование: `\n` (перевод строки), `\t` (таб), `\\` (бэкслеш), `\'`, `\"`.
- **Raw-строки** `r'...'` — обратный слеш не спецсимвол; незаменимы для путей Windows и регулярных выражений.
- **Тройные кавычки** `'''...'''` / `"""..."""` — многострочные литералы; сохраняют переносы. Чтобы не было пустой первой строки, используют `\` в конце первой строки.
- Строки **неизменяемы** (immutable): любой «изменяющий» метод возвращает новую строку.
- Индексация с 0; отрицательные индексы — с конца (`-1` последний).
- **Срез** `s[start:stop]`: включает `start`, **не включает** `stop`; полезный мысленный образ — индексы стоят «между» символами.
- Полезные методы: `.upper()`, `.lower()`, `.strip()`, `.split()`, `.replace()`, `.startswith()`, `.find()`, `.join()`.

### Списки
- Упорядоченные, изменяемые, могут хранить элементы разных типов и вложенные списки.
- Поддерживают индексацию, срезы, `+`, `*`, `len()`.
- **Срез списка возвращает новый список** (поверхностная копия), а присваивание срезу изменяет исходный.
- Методы: `.append()`, `.extend()`, `.insert()`, `.remove()`, `.pop()`, `.sort()`, `.reverse()`.

### Особенности синтаксиса первых программ
- Блоки задаются **отступами** (обычно 4 пробела), а не скобками.
- `while условие:` — двоеточие и отступленный блок.
- `print(a, b)` разделяет аргументы пробелом и добавляет `\n`; параметры `sep=` и `end=` это меняют.

## Частые вопросы на собеседовании (Q/A)

**Q1. В чём разница между `/` и `//`?**
A: `/` — истинное деление, всегда возвращает `float` (`7/2 == 3.5`, `4/2 == 2.0`). `//` — целочисленное деление с округлением вниз (floor), тип зависит от операндов (`7//2 == 3`).

**Q2. Чему равно `-7 // 2` и почему?**
A: `-4`, а не `-3`. `//` округляет к минус бесконечности (floor), а не отбрасывает дробную часть в сторону нуля.

**Q3. Какой знак у результата `%` в Python?**
A: Знак остатка совпадает со знаком **делителя**: `7 % -3 == -2`, `-7 % 3 == 2`. Действует инвариант `(a // b) * b + (a % b) == a`.

**Q4. Чем строка отличается от списка?**
A: Строка неизменяема (immutable) и хранит только символы; список изменяем (mutable) и хранит любые объекты. У обоих есть индексация и срезы, но у строки нельзя `s[0] = 'x'`.

**Q5. Что такое срез и работает ли он за границами?**
A: `s[start:stop]` возвращает подпоследовательность: включает `start`, исключает `stop`. Срез не падает при выходе за границы (вернёт меньше элементов или пустую последовательность), в отличие от индексации, которая бросает `IndexError`.

**Q6. Что такое raw-строка и зачем она?**
A: Литерал `r'...'`, в котором обратный слеш не считается началом escape-последовательности. Нужен для путей Windows (`r'C:\new'`) и регулярных выражений, чтобы не дублировать `\\`.

**Q7. Как поменять местами две переменные в Python?**
A: Через множественное присваивание: `a, b = b, a`. Правая часть вычисляется в кортеж целиком, затем распаковывается — временная переменная не нужна.

**Q8. Что выведет `'ab' * 3` и `'Py' 'thon'`?**
A: `'ababab'` (повторение строки). `'Python'` — соседние строковые **литералы** склеиваются автоматически (но переменные так не склеить — нужен `+`).

**Q9. Чему равно `word[:i] + word[i:]`?**
A: Исходной строке `word` — это инвариант срезов для любого `i`.

**Q10. В чём разница между индексацией и срезом списка по результату?**
A: Индексация (`lst[0]`) возвращает сам элемент. Срез (`lst[0:2]`) возвращает новый список (поверхностную копию). `lst[:]` — частый способ скопировать список.

**Q11. Почему `0.1 + 0.2 != 0.3`?**
A: `float` хранится в двоичном виде с конечной точностью, поэтому `0.1 + 0.2 == 0.30000000000000004`. Для точных вычислений (деньги) используют `decimal.Decimal` или сравнивают через `math.isclose`.

**Q12. Что делает `letters[1:3] = []`?**
A: Удаляет элементы с индексами 1 и 2 из списка (присваивание пустого списка срезу). Списки изменяемы, поэтому срез-присваивание меняет их «на месте».

## Подводные камни (gotchas)

- **`/` всегда float.** Даже `6 / 3 == 2.0`, а не `2`. Если нужно целое — `//` или `int(...)`.
- **`//` округляет вниз, не «к нулю».** `-7 // 2 == -4`. Частая ошибка с отрицательными числами.
- **Срез не падает, индексация падает.** `s[100]` → `IndexError`, но `s[100:]` → `''`.
- **Строки неизменяемы.** `s[0] = 'X'` → `TypeError`; все методы возвращают новую строку, а не меняют исходную.
- **`\` в обычных строках — escape.** `'C:\new'` содержит `\n` (перевод строки). Используйте raw-строки или `\\`.
- **Автосклейка только литералов.** `var 'text'` — SyntaxError; нужно `var + 'text'`.
- **Срез копирует поверхностно.** `b = a[:]` — новый список, но вложенные объекты общие (shallow copy). Для глубокой копии — `copy.deepcopy`.
- **`float` неточен.** Не сравнивайте `float` через `==`; используйте `math.isclose`.
- **`a, b = b, a + b` — порядок важен.** Правая часть вычисляется до присваивания, поэтому Фибоначчи работает корректно; разбив на две строки `a = b; b = a + b`, вы получите неверный результат.

## Шпаргалка

Числа:
- `+ - * /` обычная арифметика; `/` → float.
- `//` целочисленное (floor), `%` остаток (знак как у делителя), `**` степень.
- `int` — без переполнения; `float` — неточен; `complex` — `3+4j`.

Строки (immutable):
- `'...'` `"..."` равнозначны; `r'...'` raw; `'''...'''` многострочные.
- `s[i]` индекс (с 0; `-1` с конца), `s[a:b]` срез (включая a, исключая b).
- `+` склейка, `*` повтор, `len(s)` длина.
- Методы: `.upper() .lower() .strip() .split() .replace() .join()`.

Списки (mutable):
- `lst[i]`, `lst[a:b]` (срез → новый список), `lst[:]` копия.
- `+` `*` `len()`; `lst[i]=x`, `lst[a:b]=[...]`, `lst[a:b]=[]` (удаление).
- Методы: `.append() .extend() .insert() .pop() .remove() .sort() .reverse()`.

Первые программы:
- Блоки — отступами (4 пробела); `while cond:` + двоеточие.
- Множественное присваивание: `a, b = b, a + b` (обмен/Фибоначчи).
- `print(x, end=' ')` — без переноса строки; `sep=` — разделитель.
