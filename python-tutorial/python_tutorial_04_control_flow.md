# Глава 4. More Control Flow Tools — конспект и вопросы

## О чём глава

Глава 4 официального туториала Python — самая «плотная» из вводных. Помимо уже знакомого `if`, она вводит почти весь инструментарий управления потоком выполнения и, главное, **определение функций** со всеми их тонкостями. Именно отсюда растёт большинство классических вопросов на собеседовании по Python:

- условия `if / elif / else`;
- циклы `for` и `while`, функция `range`;
- операторы `break`, `continue` и **необычный блок `else` у циклов**;
- заглушка `pass`;
- оператор `match` (structural pattern matching, появился в Python 3.10);
- определение функций `def`, возврат значения, `None`;
- **область видимости и правило LEGB**;
- аргументы по умолчанию и **ловушка изменяемого значения по умолчанию**;
- позиционные и именованные (keyword) аргументы;
- произвольные списки аргументов `*args` и `**kwargs`;
- позиционно-только `/` и keyword-only `*`;
- распаковка аргументов при вызове (`*list`, `**dict`);
- анонимные функции `lambda` (и **ловушка замыкания в цикле**);
- строки документации (docstrings);
- аннотации функций (function annotations);
- кратко про стиль кода PEP 8.

## Ключевые концепции (с примерами и выводом)

### if / elif / else

```python
x = int(input("Введите число: "))  # допустим, ввели 42

if x < 0:
    x = 0
    print("Отрицательное обнулили")
elif x == 0:
    print("Ноль")
elif x == 1:
    print("Единица")
else:
    print("Больше одного")   # => Больше одного
```

В Python нет конструкции `switch` в классическом виде (до 3.10) — цепочка `if/elif/else` её заменяет. `elif` — это сокращение от `else if`, веток `elif` может быть сколько угодно, `else` — необязателен.

### Цикл for

`for` в Python — это **итерация по элементам последовательности** (foreach), а не счётчик с условием, как в C.

```python
words = ["кот", "окно", "защита"]
for w in words:
    print(w, len(w))
# => кот 3
# => окно 4
# => защита 6
```

### range

```python
list(range(5))        # => [0, 1, 2, 3, 4]
list(range(2, 5))     # => [2, 3, 4]
list(range(0, 10, 3)) # => [0, 3, 6, 9]
list(range(10, 0, -2))# => [10, 8, 6, 4, 2]
```

`range` — это не список, а «ленивый» объект-последовательность; он не хранит все числа в памяти, а вычисляет их по требованию:

```python
r = range(1_000_000)
print(r)            # => range(0, 1000000)
print(sum(r))       # => 499999500000  (память почти не тратится)
print(r[500])       # => 500 (поддерживает индексацию)
```

Чтобы получить и индекс, и значение, используют `enumerate`:

```python
for i, w in enumerate(["a", "b", "c"], start=1):
    print(i, w)
# => 1 a
# => 2 b
# => 3 c
```

### while

```python
a, b = 0, 1
while a < 10:
    print(a, end=" ")
    a, b = b, a + b
print()
# => 0 1 1 2 3 5 8
```

### break, continue

```python
for n in range(2, 10):
    for x in range(2, n):
        if n % x == 0:
            print(f"{n} = {x} * {n // x}")
            break          # выходим из ВНУТРЕННЕГО цикла
    else:
        print(n, "— простое число")
# => 2 — простое число
# => 3 — простое число
# => 4 = 2 * 2
# => 5 — простое число
# => 6 = 2 * 3
# => 7 — простое число
# => 8 = 2 * 4
# => 9 = 3 * 3
```

```python
for num in range(2, 10):
    if num % 2 == 0:
        print("Чётное:", num)
        continue           # пропустить остаток тела, перейти к след. итерации
    print("Нечётное:", num)
```

### else у циклов (частый вопрос!)

Блок `else` у `for` и `while` выполняется, **если цикл завершился естественно (без `break`)**. Удобно запомнить как «no-break»:

```python
def find(seq, target):
    for item in seq:
        if item == target:
            print("Нашли:", item)
            break
    else:
        # выполнится, только если break НЕ сработал
        print("Не найдено")

find([1, 2, 3], 2)   # => Нашли: 2
find([1, 2, 3], 9)   # => Не найдено
```

Подробнее — в разделе «Подробный разбор».

### pass

`pass` — это пустая операция (no-op). Нужна там, где синтаксис требует тело, а делать ничего не надо:

```python
def todo():
    pass            # реализуем позже

class Empty:
    pass

while True:
    pass            # бесконечное ожидание (например, заглушка)
```

### match (structural pattern matching, 3.10+)

```python
def http_status(code):
    match code:
        case 400:
            return "Bad request"
        case 401 | 403 | 404:        # несколько значений через |
            return "Not allowed / Not found"
        case 500:
            return "Server error"
        case _:                      # _ — «всё остальное» (wildcard)
            return "Unknown"

print(http_status(404))  # => Not allowed / Not found
print(http_status(418))  # => Unknown
```

Это не просто «switch». `match` умеет раскладывать (деструктурировать) структуры:

```python
def describe(point):
    match point:
        case (0, 0):
            return "Начало координат"
        case (0, y):
            return f"На оси Y, y={y}"
        case (x, 0):
            return f"На оси X, x={x}"
        case (x, y):
            return f"Точка ({x}, {y})"
        case _:
            return "Не точка"

print(describe((0, 5)))   # => На оси Y, y=5
print(describe((3, 4)))   # => Точка (3, 4)
```

### Определение функций (def)

```python
def fib(n):
    """Вернуть список чисел Фибоначчи меньше n."""
    result = []
    a, b = 0, 1
    while a < n:
        result.append(a)
        a, b = b, a + b
    return result

print(fib(20))  # => [0, 1, 1, 2, 3, 5, 8, 13]
```

Функция без `return` (или с пустым `return`) возвращает `None`:

```python
def greet(name):
    print("Привет,", name)

result = greet("Аня")   # => Привет, Аня
print(result)           # => None
```

## Подробный разбор по темам

### 1. Цикл for — не модифицируй коллекцию, по которой идёшь

Изменять список во время итерации по нему — частый источник багов. Итерируйтесь по копии (`for x in lst[:]`) или собирайте новый список:

```python
users = [{"name": "a", "active": True}, {"name": "b", "active": False}]

# ПЛОХО: удаление из списка во время прохода пропускает элементы
# for u in users:
#     if not u["active"]:
#         users.remove(u)   # сдвигает индексы — баг

# ХОРОШО: новый список
active = [u for u in users if u["active"]]
```

`range` стоит брать, когда нужны именно числа; для перебора элементов — прямой `for x in seq`. Антипаттерн `for i in range(len(seq)): seq[i]` почти всегда заменяется на `enumerate`.

### 2. else у циклов — почему «no-break» и где это полезно

Семантика: `else` после цикла = «цикл дошёл до конца, ни разу не вызвав `break`». Мнемоника от Раймонда Хеттингера: читать не как `if/else`, а как `for/no_break`.

Типичная задача — «поиск с обработкой случая "не найдено"»:

```python
def is_prime(n):
    if n < 2:
        return False
    for d in range(2, int(n ** 0.5) + 1):
        if n % d == 0:
            break          # нашли делитель — не простое
    else:
        return True        # делителей не нашли -> простое
    return False

print([n for n in range(2, 20) if is_prime(n)])
# => [2, 3, 5, 7, 11, 13, 17, 19]
```

Важная деталь: если цикл **вообще не выполнил ни одной итерации** (например, пустая последовательность), `else` всё равно сработает — break-то не было:

```python
for x in []:
    print(x)
else:
    print("else сработал даже для пустого цикла")  # => else сработал...
```

С `while` аналогично: `else` выполнится, когда условие стало ложным «само», но не выполнится при выходе через `break`.

### 3. Определение функций: объекты, ссылки, возврат

- `def` создаёт объект-функцию и связывает его с именем. Имя можно переприсвоить:

```python
def f(): return 1
g = f            # g ссылается на тот же объект-функцию
print(g())       # => 1
print(f is g)    # => True
```

- Функции — объекты первого класса: их можно передавать в другие функции, хранить в списках/словарях, возвращать.

- Аргументы передаются по присваиванию («call by object reference»): внутри функции имя параметра — новая ссылка на тот же объект. Переприсвоение параметра наружу не видно, **мутация изменяемого объекта — видна**:

```python
def rebind(x):
    x = 99            # переприсваивание — локально

def mutate(lst):
    lst.append(99)    # мутация — видна снаружи

a = 1
rebind(a)
print(a)              # => 1  (не изменилось)

b = [1, 2]
mutate(b)
print(b)              # => [1, 2, 99]  (изменилось!)
```

### 4. Область видимости и LEGB

Python ищет имя по правилу **LEGB**: **L**ocal → **E**nclosing → **G**lobal → **B**uilt-in.

```python
x = "global"

def outer():
    x = "enclosing"
    def inner():
        # x не присваивается внутри inner -> ищется выше:
        # Local? нет -> Enclosing? есть -> "enclosing"
        print(x)
    inner()

outer()              # => enclosing
print(len)           # len найдётся в Built-in
```

- По умолчанию присваивание внутри функции создаёт **локальную** переменную. Чтобы менять глобальную — `global`; чтобы менять переменную охватывающей функции — `nonlocal`:

```python
counter = 0
def inc():
    global counter
    counter += 1

inc(); inc()
print(counter)       # => 2

def make_counter():
    n = 0
    def step():
        nonlocal n
        n += 1
        return n
    return step

c = make_counter()
print(c(), c(), c()) # => 1 2 3
```

- Частая ошибка `UnboundLocalError`: если в функции есть присваивание имени, оно считается локальным во **всём** теле, даже до присваивания:

```python
y = 10
def bad():
    print(y)         # UnboundLocalError: y используется до присваивания
    y = 20           # из-за этой строки y локальная во всей функции
```

### 5. Аргументы по умолчанию: вычисляются один раз

Значение по умолчанию вычисляется **один раз — в момент определения функции**, а не при каждом вызове. Для неизменяемых значений (числа, строки, `None`) это незаметно. Для изменяемых (`list`, `dict`, `set`) — это **самая известная ловушка Python**.

```python
# ОПАСНО:
def append_to(item, target=[]):   # один и тот же список на все вызовы!
    target.append(item)
    return target

print(append_to(1))   # => [1]
print(append_to(2))   # => [1, 2]   <-- сюрприз! список «помнит» прошлое
print(append_to(3))   # => [1, 2, 3]

# ПРАВИЛЬНО: sentinel None
def append_to_ok(item, target=None):
    if target is None:
        target = []
    target.append(item)
    return target

print(append_to_ok(1))  # => [1]
print(append_to_ok(2))  # => [2]
```

Доказательство, что объект один:

```python
def f(x=[]):
    return x
print(f() is f())       # => True  (один и тот же объект)
print(f.__defaults__)   # => ([1, ...],)  значения по умолчанию хранятся в функции
```

Иногда это поведение используют намеренно — например, как простой кэш/мемоизацию, но делать так стоит осознанно и с комментарием.

### 6. Позиционные vs keyword-аргументы; / и *

Правила вызова:
- позиционные передаются по порядку;
- именованные — по имени `name=value`;
- после первого keyword-аргумента в вызове нельзя ставить позиционный.

В сигнатуре `/` отделяет **позиционно-только** параметры, `*` — начало **keyword-only** параметров:

```python
def f(pos_only, /, standard, *, kw_only):
    return pos_only, standard, kw_only

print(f(1, 2, kw_only=3))           # => (1, 2, 3)
print(f(1, standard=2, kw_only=3))  # => (1, 2, 3)
# f(pos_only=1, ...)  -> TypeError: pos_only только позиционный
# f(1, 2, 3)          -> TypeError: kw_only нельзя передать позиционно
```

Зачем нужно:
- `/` (positional-only) — даёт свободу переименовывать параметры в будущем без поломки API; так сделано во многих встроенных функциях (`pow`, `len`, `abs`).
- `*` (keyword-only) — заставляет вызывать булевы/режимные флаги по имени, что улучшает читаемость: `sorted(data, reverse=True)`.

### 7. *args и **kwargs подробно

```python
def make_pizza(size, *toppings, **options):
    print("Размер:", size)
    print("Начинка:", toppings)   # кортеж лишних позиционных
    print("Опции:", options)      # словарь лишних именованных

make_pizza("L", "сыр", "грибы", thin_crust=True, extra_cheese=False)
# => Размер: L
# => Начинка: ('сыр', 'грибы')
# => Опции: {'thin_crust': True, 'extra_cheese': False}
```

Порядок параметров в сигнатуре строго такой:
`positional / *args / keyword-only / **kwargs`.

```python
def full(a, b=2, *args, kw1, kw2="x", **kwargs):
    ...
```

Параметры после `*args` автоматически становятся keyword-only.

Имена `args`/`kwargs` — лишь соглашение; важны звёздочки `*` и `**`.

### 8. Распаковка аргументов при вызове

`*` распаковывает последовательность в позиционные аргументы, `**` — словарь в именованные:

```python
def point(x, y, z):
    return f"({x}, {y}, {z})"

coords = [1, 2, 3]
print(point(*coords))            # => (1, 2, 3)

kw = {"x": 10, "y": 20, "z": 30}
print(point(**kw))               # => (10, 20, 30)

# range тоже принимает распаковку:
args = [2, 10, 2]
print(list(range(*args)))        # => [2, 4, 6, 8]
```

### 9. lambda — анонимные функции

`lambda параметры: выражение` — компактная функция из одного выражения (без `return`, без операторов). Эквивалент:

```python
add = lambda a, b: a + b
# то же самое, что:
def add(a, b):
    return a + b
```

Где уместна: как короткий `key`/`callback` (`sorted`, `map`, `filter`, `min`, `max`):

```python
data = ["bbb", "a", "cc"]
print(sorted(data, key=lambda s: len(s)))   # => ['a', 'cc', 'bbb']
print(max([3, -7, 5], key=lambda n: abs(n)))# => -7
```

Lambda — это полноценное замыкание: она «захватывает» переменные окружения.

#### Ловушка замыкания в цикле (классика!)

Лямбды (и вложенные функции) захватывают **переменную, а не её значение** на момент создания. В цикле все они смотрят на одну и ту же переменную, и к моменту вызова она равна последнему значению:

```python
funcs = [lambda: i for i in range(3)]
print([f() for f in funcs])    # => [2, 2, 2]   <-- не [0, 1, 2]!
```

Почему: все три лямбды ссылаются на `i`; цикл завершился с `i == 2`. Решения:

```python
# 1) Зафиксировать значение через аргумент по умолчанию (вычисляется при создании):
funcs = [lambda i=i: i for i in range(3)]
print([f() for f in funcs])    # => [0, 1, 2]

# 2) Через функцию-фабрику (новая область видимости на каждой итерации):
def make(i):
    return lambda: i
funcs = [make(i) for i in range(3)]
print([f() for f in funcs])    # => [0, 1, 2]

# 3) functools.partial:
from functools import partial
funcs = [partial(lambda x: x, i) for i in range(3)]
print([f() for f in funcs])    # => [0, 1, 2]
```

То же касается обычных `def` внутри цикла — проблема не в `lambda`, а в позднем связывании (late binding) замыканий.

### 10. Docstrings

Строка-литерал первой инструкцией в модуле/функции/классе становится её документацией и попадает в `__doc__`. Используется `help()`, IDE, генераторами документации.

```python
def divide(a, b):
    """Поделить a на b.

    Возбуждает ZeroDivisionError, если b == 0.
    Возвращает float.
    """
    return a / b

print(divide.__doc__.splitlines()[0])   # => Поделить a на b.
help(divide)   # покажет сигнатуру и docstring
```

Соглашение (PEP 257): первая строка — краткое описание с большой буквы и точкой; для многострочных — пустая строка, затем подробности; закрывающие `"""` на отдельной строке.

### 11. Аннотации функций

Аннотации типов параметров и возвращаемого значения. **Не проверяются интерпретатором** во время выполнения — это подсказки для людей и инструментов (mypy, pyright, IDE). Хранятся в `__annotations__`.

```python
def greet(name: str, times: int = 1) -> str:
    return (f"Привет, {name}! " * times).strip()

print(greet("Ник", 2))            # => Привет, Ник! Привет, Ник!
print(greet.__annotations__)
# => {'name': <class 'str'>, 'times': <class 'int'>, 'return': <class 'str'>}

# Аннотация НЕ мешает передать «неправильный» тип:
print(greet(123))                 # сработает на этапе str.* — но не из-за аннотации
```

### 12. PEP 8 (кратко)

Главный стиль-гайд Python. Самое ходовое:
- отступ — 4 пробела, не табы;
- длина строки ~79 символов (часто 88/100 в проектах);
- имена: `snake_case` для функций/переменных, `CapWords` для классов, `UPPER_CASE` для констант;
- пробелы вокруг операторов (`x = 1`), но не вокруг `=` в аргументах по умолчанию без аннотаций (`f(x=1)`);
- импорты — каждый с новой строки, в начале файла;
- сравнение с `None` через `is` / `is not`, а не `==`;
- использовать «истинность» объекта: `if seq:` вместо `if len(seq) != 0:`.

## Частые вопросы на собеседовании (Q/A)

**1. Чем `for` в Python отличается от `for` в C/Java?**
В Python `for` итерирует по элементам любого итерируемого объекта (foreach), а не по счётчику с условием. Для счётчика используют `range`. Нет «классического» `for(i=0; i<n; i++)`.

**2. `range` — это список?**
Нет. Это ленивый неизменяемый объект-последовательность, который вычисляет элементы по требованию и не хранит их в памяти. Поддерживает индексацию, срезы, `len`, `in`. Чтобы получить список — `list(range(...))`.

**3. Что делает блок `else` у цикла и когда выполняется?**
Выполняется, если цикл завершился без `break` (включая случай пустой последовательности — итераций ноль, break не было → `else` сработает). При выходе через `break` блок `else` пропускается. Применяется для паттерна «искали и не нашли».

**4. В чём разница `break` и `continue`?**
`break` досрочно выходит из ближайшего цикла целиком. `continue` пропускает остаток тела текущей итерации и переходит к следующей. Оба действуют только на ближайший охватывающий цикл.

**5. Объясните ловушку изменяемого аргумента по умолчанию.**
Значение по умолчанию вычисляется один раз при определении функции и переиспользуется между вызовами. Если оно изменяемое (`list`, `dict`), мутации накапливаются между вызовами. Решение: ставить `None` по умолчанию и создавать объект внутри функции (`if x is None: x = []`).

**6. Когда и сколько раз вычисляются значения аргументов по умолчанию?**
Один раз — в момент выполнения инструкции `def` (определения функции), а не при каждом вызове. Хранятся в `func.__defaults__` / `func.__kwdefaults__`.

**7. Что такое LEGB?**
Порядок поиска имён: Local (локальные в функции) → Enclosing (охватывающие функции) → Global (модуль) → Built-in (встроенные). Присваивание по умолчанию создаёт локальное имя; `global` и `nonlocal` меняют это поведение.

**8. Почему возникает `UnboundLocalError`?**
Если в теле функции есть присваивание имени, оно считается локальным во всей функции. Обращение к нему до присваивания (или при намерении использовать глобальное без `global`) даёт `UnboundLocalError`.

**9. Чем `*args` отличается от `**kwargs`?**
`*args` собирает лишние позиционные аргументы в кортеж. `**kwargs` собирает лишние именованные аргументы в словарь. В сигнатуре порядок: обычные, `*args`, keyword-only, `**kwargs`. При вызове `*`/`**` наоборот распаковывают коллекции в аргументы.

**10. Зачем нужны `/` и `*` в сигнатуре функции?**
`/` помечает параметры до неё как позиционно-только (нельзя передать по имени) — полезно для стабильности API. `*` (или `*args`) делает параметры после неё keyword-only (только по имени) — улучшает читаемость флагов.

**11. Что выведет `[lambda: i for i in range(3)]` при вызове всех лямбд? Почему?**
`[2, 2, 2]`. Все лямбды захватывают одну переменную `i` (позднее связывание), а к моменту вызова цикл уже закончился со значением `i == 2`. Фикс: `lambda i=i: i`, фабрика функций или `functools.partial`.

**12. Что такое замыкание (closure)?**
Вложенная функция, которая запоминает (захватывает) переменные охватывающей области видимости и продолжает иметь к ним доступ после завершения внешней функции. Захватывается переменная, а не значение.

**13. Чем `lambda` отличается от `def`? Когда какую использовать?**
`lambda` — анонимная функция из одного выражения, без операторов и аннотаций имени. `def` — именованная, может содержать любые инструкции, docstring, аннотации. `lambda` уместна как короткий `key`/`callback`; для всего остального — `def` (читабельнее, отлаживается, имеет имя в трейсбеке).

**14. Что возвращает функция без `return`?**
`None`. То же при «голом» `return` без значения.

**15. Проверяются ли аннотации типов во время выполнения?**
Нет. Это подсказки для людей и статических анализаторов (mypy, pyright). Интерпретатор их не проверяет; хранятся в `__annotations__`. Проверку можно навесить вручную (например, через `pydantic` или декораторы).

**16. Как Python передаёт аргументы — по значению или по ссылке?**
«Call by object reference» (по ссылке на объект / по присваиванию). Имя параметра — новая ссылка на тот же объект. Переприсваивание параметра не видно снаружи; мутация изменяемого объекта — видна.

**17. Что делает оператор `match` и чем он лучше цепочки `if`?**
Structural pattern matching (3.10+): сопоставляет значение с образцами и умеет деструктурировать структуры (кортежи, списки, словари, объекты по классам), связывать имена, использовать «защиты» `if`, объединять варианты через `|`, `_` как wildcard. Это не просто `switch`, а сопоставление по форме данных.

**18. Зачем нужен `pass`?**
Это пустая операция-заглушка для мест, где синтаксис требует тело: пустые функции/классы, ветки `if`, заготовки. В отличие от `...` (Ellipsis) семантически одинаково, но `pass` — общепринятая заглушка тела.

## Подводные камни (gotchas)

1. **Изменяемое значение по умолчанию** (`def f(x=[])`) — общий список/словарь на все вызовы. Используйте `None`-sentinel.

2. **Позднее связывание в замыканиях/лямбдах в цикле** — `[lambda: i for i in range(3)]` даёт `[2,2,2]`. Фиксируйте через `i=i` или фабрику.

3. **`else` у цикла** срабатывает и для пустой последовательности, и многих сбивает с толку; легко перепутать с «else, если цикл не выполнялся». Правильно: «else, если не было `break`».

4. **`UnboundLocalError`** — любое присваивание имени делает его локальным на всю функцию; обращение до присваивания падает.

5. **Изменение коллекции во время итерации по ней** (удаление/добавление) ломает обход — итерируйтесь по копии или стройте новый список.

6. **`range` ≠ список** — нельзя `range(3).append(...)`; для модификации преобразуйте в `list`.

7. **Порядок параметров в сигнатуре** — `*args` обязан стоять перед keyword-only и `**kwargs`; нарушение — `SyntaxError`.

8. **Позиционный после keyword при вызове** — `f(a=1, 2)` это `SyntaxError`.

9. **Аннотации не валидируют типы** — не полагайтесь на них как на проверку в рантайме.

10. **`is` vs `==`** — для `None`/синглтонов используйте `is`; `==` может быть переопределён и работать иначе.

11. **Mutable default как кэш** — иногда «ловушку» используют намеренно; не делайте этого без явного комментария, иначе коллеги примут за баг.

## Шпаргалка

```python
# --- Условия ---
if c1:
    ...
elif c2:
    ...
else:
    ...

# --- Циклы и их else (no-break) ---
for x in seq:
    if cond: break
    if skip: continue
else:
    ...            # выполнится, если break не было

while cond:
    ...
else:
    ...            # если выход не через break

# --- range ---
range(stop); range(start, stop); range(start, stop, step)  # ленивый
enumerate(seq, start=0)        # (index, value)

# --- match (3.10+) ---
match value:
    case 1 | 2:        ...      # объединение
    case (x, y):       ...      # деструктуризация
    case [a, *rest]:   ...      # списки
    case {"k": v}:     ...      # словари
    case Point(x=0):   ...      # по классу
    case n if n > 0:   ...      # guard
    case _:            ...      # wildcard

# --- Функции ---
def f(pos_only, /, normal, *args, kw_only, kw2="x", **kwargs) -> int:
    """Docstring."""
    return 0

f(1, 2, 3, 4, kw_only=5, extra=6)
f(*[1, 2], **{"kw_only": 5})       # распаковка при вызове

# --- Изменяемый дефолт (правильно) ---
def g(x=None):
    if x is None:
        x = []

# --- Область видимости ---
global name        # менять глобальную
nonlocal name      # менять переменную охватывающей функции

# --- Замыкание в цикле (фикс) ---
[lambda i=i: i for i in range(3)]   # => [0, 1, 2]

# --- lambda ---
key = lambda x: (x.priority, -x.size)
sorted(data, key=key, reverse=True)

# --- Аннотации / docstrings ---
f.__annotations__      # подсказки типов (не проверяются)
f.__doc__              # строка документации
f.__defaults__         # значения позиционных дефолтов
```
