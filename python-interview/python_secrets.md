# secrets — подготовка к собеседованию

## Что это и зачем

`secrets` — модуль стандартной библиотеки (с Python 3.6, PEP 506) для генерации **криптографически стойких** случайных чисел и токенов. Предназначен для секретов: паролей, токенов сброса пароля, session id, API-ключей, CSRF-токенов, соли.

Главная причина существования: модуль `random` использует генератор Mersenne Twister — он **предсказуем**. Зная достаточно его выходов, можно восстановить внутреннее состояние и предсказать все будущие значения. Для секретов это катастрофа. `secrets` же использует системный CSPRNG (`os.urandom` → `/dev/urandom`, `getrandom()`, `CryptGenRandom`), который проектировался для криптографии.

Правило: всё, что должно быть невозможно угадать → `secrets`. Всё остальное (симуляции, тасование, тестовые данные, игры) → `random`.

## Ключевые концепции

- **CSPRNG vs PRNG:** `secrets`/`os.urandom` — криптостойкий ГПСЧ (непредсказуем, не восстанавливается по выходам). `random` — обычный детерминированный ГПСЧ (быстрый, воспроизводимый, предсказуемый).
- **Энтропия токена:** стойкость токена определяется числом случайных бит. Рекомендация — не меньше 256 бит (32 байта) для долгоживущих секретов.
- **Кодирование токенов:** `token_bytes` (сырые байты), `token_hex` (hex, 2 символа/байт), `token_urlsafe` (base64url, ~1.3 символа/байт, безопасно для URL).
- **`secrets.choice`** — равномерный безопасный выбор из последовательности (для генерации паролей/passphrase).
- **`compare_digest`** (реэкспорт из `hmac`) — сравнение секретов в постоянном времени.
- **`SystemRandom`** — класс из `random`, использующий тот же системный источник; `secrets` построен поверх него.
- **`secrets.randbelow(n)`** — безопасное целое в `[0, n)` без смещения (modulo bias), в отличие от наивного `os.urandom() % n`.

## Основные функции/классы/методы

### Генерация токенов

```python
import secrets

# Сырые случайные байты
secrets.token_bytes(32)        # b'\x9f...' — 32 байта = 256 бит энтропии

# Hex-строка (удобно для логов/БД, только [0-9a-f])
secrets.token_hex(32)          # '9f3a...' длиной 64 символа (2 символа на байт)

# URL-safe base64 (для ссылок, токенов сброса, query-параметров)
secrets.token_urlsafe(32)      # 'Gx7...' ~43 символа, символы [A-Za-z0-9_-]

# Если nbytes не указан — берётся разумный дефолт (>= 32 байт)
secrets.token_urlsafe()
```

### Безопасные случайные числа

```python
import secrets

# Целое в диапазоне [0, n) без modulo bias
secrets.randbelow(100)         # 0..99

# k случайных бит как целое
secrets.randbits(256)          # большое целое, ровно 256 бит энтропии

# Безопасный выбор элемента из последовательности
secrets.choice(["red", "green", "blue"])
```

### Генерация надёжного пароля

```python
import secrets
import string

def generate_password(length: int = 16) -> str:
    # Алфавит: буквы + цифры + спецсимволы
    alphabet = string.ascii_letters + string.digits + "!@#$%^&*-_"
    while True:
        pwd = "".join(secrets.choice(alphabet) for _ in range(length))
        # Проверяем, что есть хотя бы по одному из нужных классов символов
        if (any(c.islower() for c in pwd)
                and any(c.isupper() for c in pwd)
                and any(c.isdigit() for c in pwd)):
            return pwd
```

### Генерация passphrase (стиль XKCD / diceware)

```python
import secrets

def generate_passphrase(num_words: int = 4, sep: str = "-") -> str:
    with open("/usr/share/dict/words") as f:
        words = [w.strip() for w in f if w.strip().isalpha()]
    return sep.join(secrets.choice(words) for _ in range(num_words))
```

### Токен сброса пароля / API-ключ

```python
import secrets

def make_reset_token() -> str:
    # URL-safe токен, который пойдёт в ссылку письма
    return secrets.token_urlsafe(32)

def make_api_key() -> str:
    # Префикс для удобной идентификации + случайная часть
    return "sk_live_" + secrets.token_hex(24)
```

### Безопасное сравнение секретов

```python
import secrets

# Сравнение токена из запроса с эталоном — в постоянном времени
def is_valid(provided: str, expected: str) -> bool:
    return secrets.compare_digest(provided, expected)
```

### Безопасное тасование/выборка (через SystemRandom)

```python
import random

# secrets не имеет shuffle/sample напрямую — используем SystemRandom
sysrand = random.SystemRandom()
deck = list(range(52))
sysrand.shuffle(deck)              # криптостойкое тасование
picks = sysrand.sample(range(1000), k=5)
```

## Частые вопросы на собеседовании

**Q1. Чем `secrets` отличается от `random` и когда что использовать?**
`random` — обычный ГПСЧ (Mersenne Twister): быстрый, воспроизводимый по seed, но **предсказуемый**, поэтому небезопасен для секретов. `secrets` использует системный CSPRNG (`os.urandom`): непредсказуем. Правило: токены/пароли/ключи/соль → `secrets`; симуляции, игры, тесты, тасование данных → `random`.

**Q2. Почему `random` нельзя использовать для токенов?**
Mersenne Twister полностью детерминирован: наблюдая 624 последовательных 32-битных выхода, можно восстановить его состояние и предсказать все будущие (и прошлые) значения. Сессионный токен или токен сброса пароля, сгенерированный `random`, теоретически можно предугадать.

**Q3. В чём разница между `token_hex`, `token_urlsafe` и `token_bytes`?**
Одни и те же случайные байты в разном представлении. `token_bytes(n)` — сырые `n` байт. `token_hex(n)` — hex-строка из `2n` символов (только `0-9a-f`). `token_urlsafe(n)` — base64url, ~`1.3n` символов, безопасна для URL и компактнее hex. Энтропия у всех — `8n` бит при одинаковом `n`.

**Q4. Сколько байт энтропии нужно для токена?**
Рекомендация документации и практики — не меньше 32 байт (256 бит) для серьёзных секретов (session id, API-ключи). Этого достаточно, чтобы перебор был вычислительно невозможен. Меньше 16 байт (128 бит) для долгоживущих секретов не стоит.

**Q5. Что такое modulo bias и как `secrets.randbelow` его избегает?**
Наивный `os.urandom_int % n` даёт неравномерное распределение, если `n` не делит диапазон нацело (некоторые значения чаще). `secrets.randbelow(n)` использует rejection sampling — отбрасывает «лишние» значения, давая строго равномерное распределение в `[0, n)`.

**Q6. Почему `secrets.compare_digest`, а не `==`?**
`==` для строк сравнивает посимвольно и завершается на первом расхождении → время выдаёт, сколько символов совпало (timing-атака на токен). `compare_digest` (тот же, что в `hmac`) сравнивает за постоянное время. Критично при проверке токенов/паролей/подписей.

**Q7. Откуда `secrets` берёт случайность?**
Из системного CSPRNG через `os.urandom`: на Linux это `getrandom()`/`/dev/urandom`, на Windows — `BCryptGenRandom`/`CryptGenRandom`. Эти источники собирают энтропию от аппаратных событий и предназначены для криптографии.

**Q8. Как сгенерировать безопасный пароль с требованиями к составу?**
Через `secrets.choice` из заданного алфавита нужной длины, затем проверять наличие требуемых классов символов (см. пример) либо строить пароль гарантированно из всех классов и затем безопасно перемешивать `SystemRandom().shuffle`.

**Q9. У `secrets` нет `shuffle`/`sample` — как сделать безопасное перемешивание?**
Использовать `random.SystemRandom()` — он реализует весь интерфейс `random`, но поверх системного CSPRNG. `secrets` сам построен на `SystemRandom`.

**Q10. `secrets` подходит для генерации соли паролей?**
Да: `secrets.token_bytes(16)` — отличный источник уникальной соли. Эквивалентно `os.urandom(16)`. Соль не секретна, но должна быть непредсказуемой и уникальной.

**Q11. `os.urandom` или `secrets` — что использовать?**
`secrets` построен поверх `os.urandom` и даёт удобный высокоуровневый API (токены, choice, randbelow, compare_digest). Для прикладного кода предпочитайте `secrets`; `os.urandom` — когда нужны просто сырые байты на низком уровне.

## Подводные камни (gotchas)

- **Использование `random` для секретов** — самая частая и опасная ошибка. Предсказуемые токены/пароли.
- **Сравнение токенов через `==`** — timing-атака; берите `secrets.compare_digest`.
- **Слишком короткие токены** — менее 128–256 бит энтропии перебираемы. Не экономьте байты.
- **`random.seed()` не делает `random` безопасным** — даже с «секретным» seed Mersenne Twister остаётся предсказуемым.
- **Modulo bias** при ручной генерации чисел из байтов — используйте `randbelow`, а не `% n`.
- **Путаница длины:** `token_hex(32)` даёт строку из 64 символов, но это 32 байта энтропии — не путайте число символов и байт.
- **Логирование секретов** — сгенерированные токены/ключи не должны попадать в логи и трейсбеки.
- **Повторное использование токена** — одноразовые токены (сброс пароля) должны инвалидироваться после использования и иметь TTL.
- **`secrets.choice` по строке, а не списку** — работает, но убедитесь, что алфавит без дубликатов, иначе смещение вероятностей.
- **Хранение токенов в БД в открытом виде** — для долгоживущих секретов (API-ключи) лучше хранить их хеш (SHA-256), а пользователю показывать оригинал один раз.

## Лучшие практики

- Для любых секретов — только `secrets` (или `os.urandom`), никогда `random`.
- Минимум 32 байта (256 бит) энтропии для важных токенов.
- `token_urlsafe` — для токенов в URL/письмах; `token_hex` — где нужен только hex; `token_bytes` — для соли/ключей.
- Сравнивайте секреты через `secrets.compare_digest` (постоянное время).
- Для чисел без смещения — `randbelow`/`randbits`, не `% n`.
- Безопасное тасование/выборка — через `random.SystemRandom()`.
- Долгоживущие секреты храните хешированными (SHA-256) и с TTL; одноразовые инвалидируйте.
- Не логируйте секреты; не коммитьте их в репозиторий.
- Соль для паролей — `secrets.token_bytes(16)`, уникальная на каждый пароль.

## Шпаргалка

```python
import secrets, string

# --- Токены ---
secrets.token_bytes(32)     # 32 сырых байта (256 бит)
secrets.token_hex(32)       # '...' 64 hex-символа
secrets.token_urlsafe(32)   # '...' ~43 символа, безопасно для URL

# --- Числа ---
secrets.randbelow(n)        # 0..n-1, без modulo bias
secrets.randbits(256)       # целое из 256 случайных бит
secrets.choice(seq)         # безопасный выбор элемента

# --- Безопасное сравнение ---
secrets.compare_digest(a, b)   # постоянное время (анти-timing)

# --- Пароль ---
alpha = string.ascii_letters + string.digits + "!@#$%"
pwd = "".join(secrets.choice(alpha) for _ in range(16))

# --- Соль ---
salt = secrets.token_bytes(16)

# --- Безопасное тасование/выборка (нет в secrets) ---
import random
random.SystemRandom().shuffle(lst)
random.SystemRandom().sample(pop, k)

# ПРАВИЛА:
#  - секреты -> secrets / os.urandom; НИКОГДА random
#  - сравнение -> secrets.compare_digest, не ==
#  - >= 32 байт энтропии; числа -> randbelow (не % n)
#  - token_hex(32) = 64 символа, но 32 байта энтропии
```
