# Модуль `textwrap` — подготовка к собеседованию

## Что это и зачем

`textwrap` — модуль стандартной библиотеки для **форматирования и переноса текста** по ширине. Решает задачи:

- перенос длинного текста на строки фиксированной ширины (`wrap`, `fill`);
- сокращение текста с многоточием (`shorten`);
- удаление общего отступа (`dedent`) — критично для многострочных строковых литералов и docstring;
- добавление префикса к строкам (`indent`) — например, для цитирования или комментирования.

Применяется в CLI-инструментах (вывод help/текста по ширине терминала), генерации отчётов, логировании, форматировании сообщений, работе с docstring.

## Ключевые концепции

- **Ширина (`width`)** — целевая максимальная длина строки (по умолчанию 70).
- **Класс `TextWrapper`** — конфигурируемый объект; функции `wrap`/`fill` — удобные обёртки над ним.
- **Сохранение слов** — по умолчанию текст не рвётся посреди слова (`break_long_words` управляет этим).
- **Нормализация пробелов** — по умолчанию схлопывает повторяющиеся пробелы (`replace_whitespace`, `drop_whitespace`).
- **`dedent` работает по общему отступу** — удаляет максимальный общий префикс из пробелов всех строк.

## Основные функции/классы/методы

### `wrap` — перенос в список строк

```python
import textwrap

text = "Python — это высокоуровневый язык программирования общего назначения."

# Возвращает СПИСОК строк, каждая не длиннее width
lines = textwrap.wrap(text, width=30)
for line in lines:
    print(repr(line))
# 'Python — это'
# 'высокоуровневый язык'
# 'программирования общего'
# 'назначения.'
```

### `fill` — перенос в одну строку с `\n`

```python
import textwrap

text = "Python — это высокоуровневый язык программирования общего назначения."

# fill == "\n".join(wrap(...)) — возвращает готовую строку
print(textwrap.fill(text, width=30))
# Python — это
# высокоуровневый язык
# программирования общего
# назначения.

# Дополнительные параметры
print(textwrap.fill(text, width=40,
                    initial_indent="  • ",      # отступ первой строки
                    subsequent_indent="    "))   # отступ остальных
#   • Python — это высокоуровневый
#     язык программирования общего
#     назначения.
```

### `shorten` — сокращение с многоточием

```python
import textwrap

text = "Python — это высокоуровневый язык программирования."

# Обрезает по словам до width, добавляя placeholder
print(textwrap.shorten(text, width=30))
# Python — это [...]

print(textwrap.shorten(text, width=30, placeholder=" …"))
# Python — это …

# Если текст короче width — возвращается как есть (но пробелы нормализуются!)
print(textwrap.shorten("короткий   текст", width=50))
# короткий текст  (повторные пробелы схлопнуты)
```

### `dedent` — удаление общего отступа

```python
import textwrap

# Часто используется с многострочными литералами в коде с отступами
def get_help():
    text = '''
        Использование: tool [опции]
            -h  показать справку
            -v  версия
    '''
    return textwrap.dedent(text)

print(get_help())
# (пустая строка)
# Использование: tool [опции]
#     -h  показать справку
#     -v  версия

# dedent удаляет ОБЩИЙ минимальный отступ всех непустых строк
sample = "    строка1\n      строка2\n    строка3"
print(textwrap.dedent(sample))
# строка1
#   строка2
# строка3
# (общие 4 пробела убраны, относительный отступ строки2 сохранён)
```

### `indent` — добавление префикса

```python
import textwrap

text = "первая строка\nвторая строка\nтретья"

# Добавляет prefix к каждой строке (по умолчанию только к непустым)
print(textwrap.indent(text, "> "))
# > первая строка
# > вторая строка
# > третья

# С predicate можно управлять, к каким строкам применять
multi = "код\n\nещё код\n"
print(textwrap.indent(multi, "# ", predicate=lambda line: True))
# # код
# # 
# # ещё код
```

### Класс `TextWrapper` (конфигурируемый)

```python
import textwrap

# Когда нужно переиспользовать конфигурацию
wrapper = textwrap.TextWrapper(
    width=40,
    initial_indent="    ",       # отступ первой строки
    subsequent_indent="    ",    # отступ остальных
    break_long_words=False,      # не рвать длинные слова
    break_on_hyphens=False,      # не рвать по дефисам
    max_lines=3,                 # максимум строк (остальное -> placeholder)
    placeholder=" ...",
    expand_tabs=True,            # заменять табы пробелами
    replace_whitespace=True,     # все whitespace -> пробел
    drop_whitespace=True,        # убирать пробелы в начале/конце строк
)

text = "Очень-длинное-слово и обычный текст, который нужно перенести аккуратно."
print(wrapper.fill(text))
print("---")
print(wrapper.wrap(text))  # список строк
```

## Частые вопросы на собеседовании

**Q1: Разница между `wrap` и `fill`?**
A: `wrap` возвращает **список строк**, `fill` — **одну строку**, в которой эти строки соединены через `\n`. По сути `fill(t) == "\n".join(wrap(t))`. `wrap` удобен, когда нужно постобработать каждую строку; `fill` — для прямого вывода.

**Q2: Что делает `dedent` и зачем он нужен?**
A: Удаляет общий ведущий пробельный отступ из всех строк. Незаменим для многострочных строковых литералов внутри функций/классов, где они визуально имеют отступ из-за вложенности кода, но в итоговой строке отступ не нужен (help-тексты, SQL-запросы, шаблоны). Важно: учитывается **минимальный общий** отступ, относительные отступы сохраняются.

**Q3: Почему `shorten` иногда меняет короткий текст?**
A: `shorten` сначала нормализует пробелы (схлопывает повторяющиеся и убирает по краям), потом проверяет длину. Поэтому даже текст короче `width` может измениться из-за нормализации пробелов.

**Q4: Как `indent` решает, к каким строкам добавлять префикс?**
A: По умолчанию добавляет только к строкам, состоящим **не только** из пробелов (пустые строки пропускает). Поведение настраивается параметром `predicate` — функцией, возвращающей bool для каждой строки.

**Q5: Как заставить `wrap` не разрывать длинные слова?**
A: Установить `break_long_words=False`. Тогда слово длиннее `width` останется целым и строка может превысить ширину. Также `break_on_hyphens=False` запрещает разрывы по дефисам.

**Q6: Что произойдёт с табуляцией и переводами строк во входе `wrap`?**
A: По умолчанию `expand_tabs=True` (табы → пробелы по `tabsize`) и `replace_whitespace=True` (все whitespace-символы, включая `\n`, `\t`, заменяются на одиночный пробел). Поэтому переносы строк во входном тексте «теряются». Если нужно сохранить параграфы — обрабатывайте их отдельно или отключите `replace_whitespace`.

**Q7: Как ограничить вывод несколькими строками?**
A: Параметр `max_lines` у `TextWrapper`/`fill`. Лишний текст заменяется на `placeholder` (по умолчанию `" [...]"`).

**Q8: `dedent` vs ручное `lstrip()` на строках — в чём отличие?**
A: `lstrip()` убирает весь ведущий whitespace у каждой строки, теряя относительные отступы (структуру). `dedent` убирает только **общий** минимальный отступ, сохраняя вложенную структуру — что обычно и нужно.

## Подводные камни (gotchas)

```python
import textwrap

# 1. dedent НЕ работает, если строки имеют разные типы отступа (таб vs пробел)
mixed = "\tстрока1\n    строка2"  # таб и пробелы — разный отступ
print(repr(textwrap.dedent(mixed)))
# '\tстрока1\n    строка2' — ничего не удалено, общего префикса нет!

# 2. Первая пустая строка в литерале остаётся после dedent
s = """
    текст"""
print(repr(textwrap.dedent(s)))  # '\nтекст' — ведущий \n сохраняется
# Часто комбинируют с .strip() или .lstrip('\n')

# 3. wrap теряет переводы строк (replace_whitespace=True)
print(textwrap.wrap("строка1\nстрока2", width=20))
# ['строка1 строка2'] — \n стал пробелом, всё в одну логическую строку

# 4. shorten нормализует пробелы даже для коротких строк
print(textwrap.shorten("a    b", width=80))  # 'a b'

# 5. Очень узкий width + длинное слово
print(textwrap.wrap("суперкалифрагилистик", width=5))
# ['супер', 'калиф', 'рагил', 'истик'] — слово разрывается (break_long_words=True)
print(textwrap.wrap("суперкалифрагилистик", width=5, break_long_words=False))
# ['суперкалифрагилистик'] — целиком, превышает width

# 6. indent с predicate=None пропускает строки-пробелы
print(repr(textwrap.indent("a\n  \nb", "> ")))
# '> a\n  \n> b' — строка из пробелов НЕ получила префикс
```

## Лучшие практики

- Для help-текстов и многострочных литералов в коде используйте `dedent` (часто `textwrap.dedent(...).strip()`).
- Для динамической ширины терминала берите `shutil.get_terminal_size()` и передавайте в `width`.
- Если нужно сохранять абзацы — разбивайте текст по `\n\n` и применяйте `fill` к каждому абзацу отдельно.
- Создавайте `TextWrapper` один раз, если форматируете много текста с одной конфигурацией.
- `break_long_words=False` для текстов с URL/путями/кодом, чтобы не рвать их.
- Используйте `shorten` для превью/анонсов с осмысленным `placeholder`.
- Помните про нормализацию пробелов — для preformatted-текста отключайте `replace_whitespace`/`expand_tabs`.
