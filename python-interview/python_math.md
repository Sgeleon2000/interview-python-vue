# math — подготовка к собеседованию

## Что это и зачем

`math` — это встроенный модуль стандартной библиотеки Python, предоставляющий доступ к математическим функциям, определённым стандартом C. Он работает **только с числами типа `float`** (и `int`, которые приводятся к `float`), но **не работает с комплексными числами** — для них есть отдельный модуль `cmath`.

Зачем нужен:
- Быстрые и точные (насколько позволяет IEEE 754) реализации тригонометрии, логарифмов, экспонент.
- Целочисленные функции: `gcd`, `lcm`, `factorial`, `comb`, `perm`, `isqrt`.
- Корректное сравнение чисел с плавающей точкой через `isclose`.
- Округление в большую/меньшую сторону: `floor`, `ceil`, `trunc`.
- Константы: `pi`, `e`, `tau`, `inf`, `nan`.

Большинство функций — это тонкие обёртки над C-библиотекой `libm`, поэтому они очень быстрые, но возвращают `float` с ограниченной точностью.

## Ключевые концепции

- **Только вещественные числа.** `math.sqrt(-1)` бросит `ValueError`, тогда как `cmath.sqrt(-1)` вернёт `1j`.
- **Возврат типов.** Большинство функций возвращают `float`. Исключения — целочисленные функции (`gcd`, `factorial`, `comb`, `perm`, `isqrt`, `floor`, `ceil`, `trunc` для `int`), которые возвращают `int`.
- **IEEE 754.** Поддержка специальных значений: `inf` (бесконечность), `-inf`, `nan` (не-число). `nan != nan` — всегда `False`.
- **Производительность.** Для скалярных операций `math` быстрее, чем оператор `**` или NumPy (нет накладных расходов на массивы).

## Основные функции/классы/методы

### Константы

```python
import math

print(math.pi)    # 3.141592653589793  — число пи
print(math.e)     # 2.718281828459045  — основание натурального логарифма
print(math.tau)   # 6.283185307179586  — tau = 2*pi (полный оборот в радианах)
print(math.inf)   # inf  — положительная бесконечность
print(-math.inf)  # -inf
print(math.nan)   # nan  — "не число" (Not a Number)
```

### Округление: floor, ceil, trunc

```python
import math

print(math.floor(3.7))   # 3   — округление ВНИЗ (к минус бесконечности)
print(math.floor(-3.2))  # -4  — внимание: -3.2 округляется до -4!
print(math.ceil(3.2))    # 4   — округление ВВЕРХ (к плюс бесконечности)
print(math.ceil(-3.7))   # -3
print(math.trunc(3.7))   # 3   — отбрасывание дробной части (к нулю)
print(math.trunc(-3.7))  # -3  — отличие от floor!

# Эти функции возвращают int, когда аргумент int-совместим
print(type(math.floor(3.7)))  # <class 'int'>
```

### Степени, корни, логарифмы

```python
import math

print(math.sqrt(16))        # 4.0  — квадратный корень (всегда float)
print(math.isqrt(17))       # 4    — ЦЕЛОЧИСЛЕННЫЙ корень (floor(sqrt)), возвращает int
print(math.pow(2, 10))      # 1024.0 — степень, всегда float (в отличие от **)
print(2 ** 10)              # 1024   — оператор **, сохраняет int
print(math.exp(1))          # 2.718281828459045 — e^x
print(math.expm1(1e-10))    # 1.00000000005e-10 — точный exp(x)-1 для малых x

print(math.log(math.e))     # 1.0  — натуральный логарифм (ln)
print(math.log(100, 10))    # 2.0  — логарифм по основанию 10
print(math.log10(1000))     # 3.0  — log по основанию 10 (точнее, чем log(x,10))
print(math.log2(8))         # 3.0  — log по основанию 2
print(math.log1p(1e-15))    # 1e-15 — точный log(1+x) для малых x
```

`expm1` и `log1p` нужны для численной устойчивости: при малых `x` вычисление `exp(x) - 1` напрямую теряет точность из-за вычитания близких чисел.

### gcd и lcm (НОД и НОК)

```python
import math

print(math.gcd(12, 18))        # 6  — наибольший общий делитель
print(math.gcd(12, 18, 24))    # 6  — поддерживает несколько аргументов (с Python 3.9)
print(math.gcd())              # 0  — gcd без аргументов = 0
print(math.gcd(0, 0))          # 0

print(math.lcm(4, 6))          # 12 — наименьшее общее кратное (с Python 3.9)
print(math.lcm(4, 6, 8))       # 24
print(math.lcm())              # 1  — lcm без аргументов = 1 (нейтральный элемент)
print(math.lcm(4, 0))          # 0  — lcm с нулём = 0
```

### isclose — корректное сравнение float

```python
import math

# Прямое сравнение float опасно!
print(0.1 + 0.2 == 0.3)                  # False !!!
print(math.isclose(0.1 + 0.2, 0.3))      # True  — правильно

# Сигнатура: isclose(a, b, *, rel_tol=1e-09, abs_tol=0.0)
# Условие: |a-b| <= max(rel_tol * max(|a|,|b|), abs_tol)

# rel_tol — относительная погрешность (по умолчанию ~9 значащих цифр)
print(math.isclose(1000000.0, 1000000.1, rel_tol=1e-6))  # True

# abs_tol нужен для сравнения с нулём (rel_tol тут бесполезен!)
print(math.isclose(0.0, 1e-10))                  # False (rel_tol не работает с 0)
print(math.isclose(0.0, 1e-10, abs_tol=1e-9))    # True
```

### Факториал и комбинаторика

```python
import math

print(math.factorial(5))    # 120  — 5! = 5*4*3*2*1
print(math.factorial(0))    # 1    — 0! = 1 по определению

# comb(n, k) — число сочетаний C(n,k) = n! / (k! * (n-k)!) — "n choose k"
print(math.comb(5, 2))      # 10   — сколько способов выбрать 2 из 5 без учёта порядка

# perm(n, k) — число размещений P(n,k) = n! / (n-k)! — с учётом порядка
print(math.perm(5, 2))      # 20   — сколько способов выбрать 2 из 5 с учётом порядка
print(math.perm(5))         # 120  — perm(n) = n! (k по умолчанию = n)
```

### Тригонометрия (в радианах!)

```python
import math

print(math.sin(math.pi / 2))   # 1.0
print(math.cos(0))             # 1.0
print(math.tan(math.pi / 4))   # 0.9999999999999999 (≈1, ошибка float)

# Перевод градусы <-> радианы
print(math.radians(180))       # 3.141592653589793  (градусы -> радианы)
print(math.degrees(math.pi))   # 180.0              (радианы -> градусы)

# Обратные функции
print(math.atan2(1, 1))        # 0.7853... — учитывает знаки и квадрант
print(math.hypot(3, 4))        # 5.0 — sqrt(x^2+y^2), без переполнения
print(math.hypot(3, 4, 12))    # 13.0 — многомерная (с Python 3.8)
```

### Проверки и прочее

```python
import math

print(math.isnan(float('nan')))   # True
print(math.isinf(float('inf')))   # True
print(math.isfinite(3.14))        # True

print(math.fabs(-5))              # 5.0 — модуль (всегда float, в отличие от abs)
print(math.copysign(3, -1))       # -3.0 — модуль первого со знаком второго
print(math.fmod(-7, 3))           # -1.0 — остаток со знаком делимого (как в C)
print(-7 % 3)                     # 2    — Python % имеет знак делителя!
print(math.remainder(7, 3))       # 1.0 — IEEE-остаток (ближайшее к нулю)

# Точная сумма (компенсация ошибок округления)
print(sum([0.1] * 10))            # 0.9999999999999999
print(math.fsum([0.1] * 10))      # 1.0  — fsum суммирует без накопления ошибки

# Произведение элементов (с Python 3.8)
print(math.prod([1, 2, 3, 4]))    # 24
```

## Частые вопросы на собеседовании

**Q1: В чём разница между `math.floor`, `math.ceil` и `math.trunc`?**
A: `floor` округляет к минус бесконечности (вниз), `ceil` — к плюс бесконечности (вверх), `trunc` — к нулю (отбрасывает дробную часть). Для отрицательных чисел они различаются: `floor(-3.2) = -4`, `ceil(-3.2) = -3`, `trunc(-3.2) = -3`.

**Q2: Чем `math.pow` отличается от оператора `**`?**
A: `math.pow` всегда возвращает `float` и работает только с числами, преобразуемыми в `float`. Оператор `**` сохраняет тип (`2**10` даёт `int`), поддерживает большие целые без потери точности и работает с комплексными числами через сам тип. Для больших целочисленных степеней используйте `**`.

**Q3: Почему `0.1 + 0.2 != 0.3` и как правильно сравнивать float?**
A: Числа `0.1`, `0.2`, `0.3` не имеют точного представления в двоичной плавающей точке (IEEE 754), поэтому накапливается ошибка. Правильно сравнивать через `math.isclose(a, b)`, который проверяет относительную и абсолютную погрешности.

**Q4: Как сравнить число с нулём через `isclose`?**
A: `rel_tol` бесполезен при сравнении с нулём (так как `max(|a|,|b|)` тоже стремится к нулю). Нужно задать `abs_tol`: `math.isclose(x, 0.0, abs_tol=1e-9)`.

**Q5: Чем `math.sqrt(n)` отличается от `math.isqrt(n)`?**
A: `sqrt` возвращает `float` (приближённое значение с возможной ошибкой округления), `isqrt` возвращает `int` — целочисленный квадратный корень `floor(sqrt(n))`, вычисленный точно даже для огромных чисел. `isqrt(10**100)` точен, а `sqrt` потеряет точность.

**Q6: Как ведёт себя `nan` в сравнениях?**
A: `nan` не равен ничему, включая самого себя: `float('nan') == float('nan')` — это `False`. Поэтому для проверки используют `math.isnan(x)`. Это же ломает сортировку и поиск `nan` в списке через `in`.

**Q7: Почему `math.fsum` точнее, чем встроенный `sum`?**
A: `sum` накапливает ошибки округления на каждом шаге. `fsum` использует алгоритм компенсированного суммирования (отслеживает частичные суммы), давая корректно округлённый результат. `sum([0.1]*10) == 0.9999...`, а `fsum([0.1]*10) == 1.0`.

**Q8: В чём разница между `math.fmod` и оператором `%`?**
A: `math.fmod` повторяет поведение C: знак результата совпадает со знаком делимого (`fmod(-7,3) = -1.0`). Оператор `%` в Python даёт знак делителя (`-7 % 3 = 2`). `fmod` предпочтителен для float.

**Q9: Что делает `math.comb(n, k)` и `math.perm(n, k)`?**
A: `comb` — число сочетаний (без учёта порядка), `perm` — число размещений (с учётом порядка). Они вычисляются эффективно, без построения полного факториала, и не переполняются (возвращают точный `int`).

**Q10: Зачем нужны `expm1` и `log1p`?**
A: Для численной устойчивости при малых аргументах. `exp(x)-1` и `log(1+x)` при `x→0` теряют точность из-за вычитания близких чисел / округления. `expm1(x)` и `log1p(x)` вычисляют их точно.

## Подводные камни (gotchas)

- **Точность float (главное!).** Все функции, возвращающие `float`, подвержены ошибкам IEEE 754. `math.tan(math.pi/4)` даёт `0.9999999999999999`, а не `1.0`. Никогда не сравнивайте результаты `==`, используйте `isclose`.

- **`floor` для отрицательных чисел.** `math.floor(-0.5) == -1`, а не `0`. Частая ошибка при работе с индексами/координатами.

- **`math.sqrt` от отрицательного → `ValueError`.** Не `nan`! Для комплексного результата нужен `cmath.sqrt`.

- **`nan` ломает логику.** `nan` в `min`/`max`/`sorted` даёт непредсказуемый результат; `nan in [nan]` может вернуть `True` (из-за identity-проверки `is` перед `==`), а `nan == nan` — `False`.

- **`math.pow(0, 0)` и `0**0`.** Оба возвращают `1.0`/`1` в Python, но это математически спорно — стоит знать.

- **Переполнение `float`.** `math.exp(1000)` бросит `OverflowError`, а `math.factorial` от большого числа работает (возвращает огромный `int`), но `math.pow` переполнится.

- **`gcd`/`lcm` с несколькими аргументами** появились только в Python 3.9. В более старых версиях принимают ровно два аргумента.

- **`math.log(x, base)` менее точен**, чем специализированные `log2`/`log10`, так как делит два логарифма. Для оснований 2 и 10 используйте специальные функции.

## Лучшие практики

- Для сравнения `float` всегда используйте `math.isclose` с подходящими `rel_tol`/`abs_tol`.
- Для денег и точных вычислений используйте `decimal.Decimal`, а не `math`/`float`.
- Целочисленный корень — `math.isqrt`, а не `int(math.sqrt(n))` (последнее может ошибиться на 1 для больших чисел).
- Для суммирования множества float используйте `math.fsum`.
- Для комбинаторики используйте `math.comb`/`math.perm` вместо ручного факториала — они быстрее и точнее.
- Для тригонометрии помните, что аргументы в радианах; используйте `math.radians`/`math.degrees`.
- Для больших целочисленных степеней используйте `**`, а не `math.pow`.

## Шпаргалка

```python
import math

# Константы
math.pi, math.e, math.tau, math.inf, math.nan

# Округление
math.floor(x)   # вниз (к -inf)
math.ceil(x)    # вверх (к +inf)
math.trunc(x)   # к нулю

# Степени/корни/логи
math.sqrt(x)         # корень (float)
math.isqrt(n)        # целочисленный корень (int)
math.pow(x, y)       # x**y как float
math.exp(x)          # e**x
math.log(x[, base])  # ln или log_base
math.log2(x), math.log10(x)
math.expm1(x), math.log1p(x)  # точные для малых x

# Целочисленное
math.gcd(*ints)      # НОД
math.lcm(*ints)      # НОК
math.factorial(n)    # n!
math.comb(n, k)      # сочетания C(n,k)
math.perm(n, k)      # размещения P(n,k)

# Сравнение float
math.isclose(a, b, rel_tol=1e-9, abs_tol=0.0)
math.isnan(x), math.isinf(x), math.isfinite(x)

# Прочее
math.fsum(iter)      # точная сумма
math.prod(iter)      # произведение
math.hypot(*coords)  # евклидова норма
math.fabs(x)         # |x| как float
math.copysign(x, y)  # |x| со знаком y
```
