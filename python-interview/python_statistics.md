# statistics — подготовка к собеседованию

## Что это и зачем

`statistics` — модуль стандартной библиотеки (с Python 3.4) для **базовой описательной статистики** над числовыми данными. Не требует NumPy, работает с обычными списками, поддерживает `int`, `float`, `Decimal`, `Fraction`.

Зачем нужен:
- Средние величины: среднее, медиана, мода.
- Меры разброса: дисперсия, стандартное отклонение.
- Квантили/перцентили через `quantiles`.
- Корреляция, ковариация, линейная регрессия (Python 3.8+/3.10+).
- Численно устойчивые реализации (например, `fmean`, точное `mean` для дробей).

Это «карманная статистика» для скриптов и собеседований, без зависимостей. Для тяжёлых вычислений — NumPy/pandas/SciPy.

## Ключевые концепции

- **Сохранение типов.** `mean([Fraction, Fraction])` вернёт `Fraction`, `mean([Decimal, ...])` — `Decimal`. `fmean` всегда `float`.
- **Выборка vs генеральная совокупность.** `variance`/`stdev` — для **выборки** (делят на n−1, поправка Бесселя). `pvariance`/`pstdev` — для **всей популяции** (делят на n).
- **Медиана и её варианты.** `median`, `median_low`, `median_high`, `median_grouped` — для разных нужд (чётная длина, дискретные данные, сгруппированные).
- **Мода.** `mode` (одна, самая частая) и `multimode` (все самые частые) — работают и с нечисловыми данными.
- **Квантили.** `quantiles(data, n=4)` делит на n равных частей (квартили при n=4).

## Основные функции/классы/методы

### Средние величины

```python
import statistics as st

data = [2, 4, 4, 4, 5, 5, 7, 9]

print(st.mean(data))     # 5    — среднее арифметическое
print(st.fmean(data))    # 5.0  — быстрое float-среднее (Python 3.8+)
print(st.median(data))   # 4.5  — медиана (среднее двух центральных при чётной длине)
print(st.mode(data))     # 4    — мода (самое частое значение)
print(st.multimode([1, 1, 2, 2, 3]))  # [1, 2] — все самые частые (Python 3.8+)

# Геометрическое и гармоническое (Python 3.8/3.10+)
print(st.geometric_mean([1, 4, 16]))   # 4.0
print(st.harmonic_mean([40, 60]))      # 48.0 (например, средняя скорость)

# Сохранение типов
from fractions import Fraction
print(st.mean([Fraction(1,2), Fraction(1,3)]))  # 5/12 (точно!)
from decimal import Decimal
print(st.mean([Decimal('0.1'), Decimal('0.2')]))  # 0.15 (Decimal)
```

### Медиана: варианты

```python
import statistics as st

data = [1, 3, 5, 7]

print(st.median(data))        # 4.0 — (3+5)/2, может не быть в данных
print(st.median_low(data))    # 3   — нижнее из двух центральных (всегда из данных)
print(st.median_high(data))   # 5   — верхнее из двух центральных

# median_grouped — для сгруппированных/непрерывных данных (интерполяция)
ages = [52, 52, 53, 54]
print(st.median_grouped(ages))  # 52.5 (с шириной интервала по умолчанию 1)
```

### Дисперсия и стандартное отклонение

```python
import statistics as st

sample = [2, 4, 4, 4, 5, 5, 7, 9]

# ВЫБОРОЧНЫЕ (делят на n-1, поправка Бесселя) — когда данные это выборка
print(st.variance(sample))   # 4.571428571428571  — выборочная дисперсия
print(st.stdev(sample))      # 2.138089935299395  — выборочное СКО

# ГЕНЕРАЛЬНОЙ СОВОКУПНОСТИ (делят на n) — когда данные это вся популяция
print(st.pvariance(sample))  # 4.0  — дисперсия популяции
print(st.pstdev(sample))     # 2.0  — СКО популяции

# Можно передать заранее известное среднее (ускоряет, избегает повторного прохода)
m = st.mean(sample)
print(st.variance(sample, m))

# variance требует минимум 2 точки, иначе StatisticsError
try:
    st.variance([5])
except st.StatisticsError as e:
    print('Ошибка:', e)   # variance requires at least two data points
```

### quantiles — квантили, перцентили, квартили

```python
import statistics as st

data = list(range(1, 101))   # 1..100

# Квартили (n=4): три точки, делящие на 4 части
print(st.quantiles(data, n=4))   # [25.25, 50.5, 75.75]  (Q1, Q2/медиана, Q3)

# Децили (n=10), перцентили (n=100)
print(st.quantiles(data, n=10)[0])    # первый дециль
print(st.quantiles(data, n=100)[89])  # ~90-й перцентиль

# Два метода: 'exclusive' (по умолчанию) и 'inclusive'
print(st.quantiles(data, n=4, method='exclusive'))  # для выборки из большей популяции
print(st.quantiles(data, n=4, method='inclusive'))  # когда данные = вся популяция
```

### Корреляция, ковариация, регрессия (Python 3.10+)

```python
import statistics as st

x = [1, 2, 3, 4, 5]
y = [2, 4, 6, 8, 10]   # y = 2x

print(st.correlation(x, y))   # 1.0  — коэффициент корреляции Пирсона
print(st.covariance(x, y))    # 5.0  — ковариация

# Линейная регрессия: y = slope*x + intercept
slope, intercept = st.linear_regression(x, y)
print(slope, intercept)       # 2.0 0.0
```

### NormalDist — работа с нормальным распределением (Python 3.8+)

```python
import statistics as st

nd = st.NormalDist(mu=100, sigma=15)   # среднее 100, СКО 15

print(nd.mean, nd.stdev)        # 100 15
print(nd.cdf(115))              # 0.8413... — P(X <= 115)
print(nd.pdf(100))              # плотность в точке
print(nd.inv_cdf(0.95))         # квантиль 95% (~124.7)
print(nd.samples(3, seed=42))   # 3 случайных значения из распределения

# Можно построить по данным
nd2 = st.NormalDist.from_samples([1.2, 2.3, 1.9, 2.1, 1.7])
print(round(nd2.mean, 3), round(nd2.stdev, 3))
```

## Частые вопросы на собеседовании

**Q1: В чём разница между `variance`/`stdev` и `pvariance`/`pstdev`?**
A: `variance`/`stdev` вычисляют **выборочные** оценки — делят на (n−1) (поправка Бесселя), используются когда данные это выборка из большей популяции. `pvariance`/`pstdev` делят на n и применяются, когда у вас вся генеральная совокупность.

**Q2: Чем `mean` отличается от `fmean`?**
A: `mean` сохраняет тип входных данных (`Fraction`, `Decimal`, `int`→`float`) и точен для них, но медленнее. `fmean` всегда возвращает `float` и работает заметно быстрее, оптимизирован для больших списков чисел. Для денег/дробей — `mean`, для скорости — `fmean`.

**Q3: Что вернёт `median` для чётного числа элементов?**
A: Среднее двух центральных значений (`float`), которого может не быть в исходных данных: `median([1,2,3,4])` = `2.5`. Если нужно реальное значение из данных — `median_low`/`median_high`.

**Q4: Чем `mode` отличается от `multimode`?**
A: `mode` возвращает **одно** самое частое значение (исторически бросал ошибку при нескольких модах, но с Python 3.8 возвращает первое встреченное). `multimode` возвращает **список всех** значений с максимальной частотой. Оба работают и с нечисловыми данными (строками).

**Q5: Как посчитать квартили или перцентили?**
A: `statistics.quantiles(data, n=4)` для квартилей (даёт Q1, Q2, Q3), `n=100` для перцентилей, `n=10` для децилей. Есть `method='exclusive'` (по умолчанию) и `'inclusive'`.

**Q6: Поддерживает ли `statistics` Decimal и Fraction?**
A: Да, и сохраняет типы. `mean([Fraction(1,2), Fraction(1,3)])` вернёт точную `Fraction`, что важно для финансов и точных расчётов, где float недопустим.

**Q7: Зачем нужна поправка Бесселя (n−1)?**
A: Выборочная дисперсия с делением на n систематически занижает дисперсию популяции. Деление на (n−1) даёт несмещённую оценку дисперсии генеральной совокупности по выборке.

**Q8: Что делает `harmonic_mean` и где применяется?**
A: Гармоническое среднее — обратное к среднему обратных величин. Применяется для усреднения скоростей, отношений (например, средняя скорость при равных расстояниях), F1-меры в ML.

**Q9: Когда использовать `statistics`, а когда NumPy?**
A: `statistics` — для небольших данных, скриптов без зависимостей, когда важно сохранение типов (Decimal/Fraction). NumPy — для больших массивов, векторизации и производительности (на порядки быстрее).

**Q10: Что произойдёт при `stdev([5])`?**
A: Бросится `statistics.StatisticsError` — выборочные `variance`/`stdev` требуют минимум 2 точки. `pvariance`/`pstdev` работают и с одной (вернут 0).

## Подводные камни (gotchas)

- **Точность float (главное!).** `fmean` и большинство функций возвращают `float` с ошибками IEEE 754. Для точных денежных расчётов передавайте `Decimal`/`Fraction` в `mean`/`variance`, которые сохраняют тип. Не сравнивайте результаты через `==`.

- **Выборка vs популяция.** Путаница `variance` (n−1) и `pvariance` (n) — классическая ошибка. Неверный выбор искажает результат, особенно на малых выборках.

- **`median` чётной длины даёт значение НЕ из данных.** `median([1,2,3,4]) = 2.5`. Если нужен реальный элемент — `median_low`/`median_high`.

- **`mode` на пустых данных** → `StatisticsError`. И исторически поведение при нескольких модах менялось между версиями (до 3.8 бросал ошибку).

- **`quantiles` методы дают разные числа.** `exclusive` и `inclusive` возвращают разные значения квартилей — нужно понимать, какой статистический смысл вам нужен.

- **Пустой ввод → `StatisticsError`.** Почти все функции требуют непустых данных; обрабатывайте этот случай.

- **`statistics` медленный** на больших данных (чистый Python). На миллионах точек используйте NumPy.

- **`median_grouped`** предполагает интервальную ширину и интерполирует — легко получить неожиданный результат, если данные не сгруппированные.

## Лучшие практики

- Чётко определяйте, выборка у вас или популяция, и выбирайте `variance`/`stdev` vs `pvariance`/`pstdev` соответственно.
- Для точных расчётов передавайте `Decimal`/`Fraction` в `mean` (сохраняет тип); `fmean` — когда нужна скорость и допустим `float`.
- Для квантилей осознанно выбирайте `method` (`exclusive`/`inclusive`).
- Проверяйте на пустые/одноточечные данные, ловите `StatisticsError`.
- Для крупных данных переходите на NumPy/pandas; `statistics` — для скриптов и прототипов.
- Используйте `NormalDist` для аналитической работы с нормальным распределением (CDF, перцентили) без SciPy.

## Шпаргалка

```python
import statistics as st

# Средние
st.mean(data)            # сохраняет тип (Decimal/Fraction)
st.fmean(data)           # быстрое, всегда float (3.8+)
st.median(data)          # медиана (среднее центральных)
st.median_low/median_high(data)   # из данных
st.mode(data)            # одна мода
st.multimode(data)       # все моды (3.8+)
st.geometric_mean(data)  # геом. среднее (3.8+)
st.harmonic_mean(data)   # гарм. среднее

# Разброс
st.variance(data)   # ВЫБОРКА (n-1)
st.stdev(data)      # ВЫБОРКА (n-1)
st.pvariance(data)  # ПОПУЛЯЦИЯ (n)
st.pstdev(data)     # ПОПУЛЯЦИЯ (n)

# Квантили
st.quantiles(data, n=4)             # квартили
st.quantiles(data, n=100)           # перцентили
st.quantiles(data, n=4, method='inclusive')

# Связи (3.10+)
st.correlation(x, y)
st.covariance(x, y)
st.linear_regression(x, y)   # (slope, intercept)

# Нормальное распределение (3.8+)
nd = st.NormalDist(mu, sigma)
nd.cdf(x); nd.inv_cdf(p); nd.pdf(x); nd.samples(k)

# Исключение: statistics.StatisticsError (пустые/малые данные)
```
