# io — подготовка к собеседованию

## Что это и зачем

`io` — модуль стандартной библиотеки, реализующий **систему ввода-вывода Python**: иерархию потоков (streams), файловые объекты, буферизацию, текстовые и бинарные обёртки. Именно `io` стоит за встроенной функцией `open()` — она возвращает объекты из этого модуля.

Самые практичные классы — `StringIO` и `BytesIO`: **файлоподобные объекты в памяти**. Они позволяют использовать любой код, ожидающий файл (`f.read()`, `f.write()`, `f.seek()`), без реального файла на диске — для тестов, обработки данных в памяти, перенаправления вывода.

```python
import io

# Текст в памяти как файл
buf = io.StringIO()
buf.write("строка 1\n")
buf.write("строка 2\n")
print(buf.getvalue())     # получить всё содержимое как str

# Бинарные данные в памяти как файл
bb = io.BytesIO(b"\x89PNG...")
header = bb.read(4)       # читаем как из файла
```

## Ключевые концепции

### Три уровня потоков

`io` определяет иерархию абстрактных базовых классов:

```
IOBase                       (базовый интерфейс: close, seek, tell, readable...)
├── RawIOBase               (сырой небуферизованный байтовый I/O: FileIO)
├── BufferedIOBase          (буферизованный байтовый I/O)
│   ├── BufferedReader      (буфер на чтение)
│   ├── BufferedWriter      (буфер на запись)
│   ├── BufferedRandom      (чтение+запись)
│   └── BytesIO             (буфер байтов в памяти)
└── TextIOBase              (текстовый I/O с кодировкой)
    ├── TextIOWrapper       (обёртка байтового потока в текст: то, что даёт open())
    └── StringIO            (текстовый буфер в памяти)
```

Три типичных слоя при открытии текстового файла:
1. **`FileIO`** (raw) — общается с ОС сырыми байтами.
2. **`BufferedReader`/`BufferedWriter`** — добавляет буфер (меньше системных вызовов).
3. **`TextIOWrapper`** — декодирует байты в `str` по кодировке, обрабатывает переводы строк.

```python
f = open("file.txt", "r", encoding="utf-8")
# f -> TextIOWrapper, f.buffer -> BufferedReader, f.buffer.raw -> FileIO
```

### Текстовый vs бинарный режим

- **Текстовый** (`"r"`, `"w"`, `"a"`): работаете со `str`, происходит **кодирование/декодирование** (encoding) и **трансляция переводов строк** (`\r\n` ↔ `\n`).
- **Бинарный** (`"rb"`, `"wb"`, `"ab"`): работаете с `bytes`, никакой обработки — байты как есть. Обязателен для изображений, архивов, сетевых протоколов.

### Буферизация

Буфер уменьшает число системных вызовов: запись копится в памяти и сбрасывается пачкой. Из-за этого данные могут быть «ещё не на диске» — нужен `flush()`/`close()`. Параметр `buffering` в `open()` управляет режимом.

## Основные функции/классы/методы

### StringIO — текстовый буфер в памяти

```python
import io

s = io.StringIO()                  # пустой, можно дать начальное значение io.StringIO("init")
s.write("Привет\n")               # пишем str
s.write("Мир\n")
print(s.tell())                    # текущая позиция
s.seek(0)                          # вернуться в начало для чтения
print(s.readline())                # 'Привет\n'
print(s.getvalue())                # ВСЁ содержимое целиком (даже после seek)
s.close()                          # после close getvalue() уже недоступен

# Типичное применение: захват вывода функции, ожидающей файл
import csv
out = io.StringIO()
writer = csv.writer(out)
writer.writerow(["a", "b", "c"])
csv_text = out.getvalue()          # 'a,b,c\r\n'
```

### BytesIO — бинарный буфер в памяти

```python
import io

b = io.BytesIO()
b.write(b"\x00\x01\x02")           # пишем bytes
b.seek(0)
print(b.read(2))                   # b'\x00\x01'
print(b.getvalue())                # все байты b'\x00\x01\x02'

# Применение: работа с библиотеками, которые пишут в файл (PIL, zipfile)
from zipfile import ZipFile
mem = io.BytesIO()
with ZipFile(mem, "w") as zf:
    zf.writestr("hello.txt", "содержимое")
mem.seek(0)
zip_bytes = mem.getvalue()         # готовый zip в памяти, можно отдать в HTTP-ответ
```

### open() и режимы

```python
# Сигнатура: open(file, mode='r', buffering=-1, encoding=None,
#                  errors=None, newline=None, ...)

# Режимы доступа
open("f.txt", "r")    # чтение (по умолчанию), файл должен существовать
open("f.txt", "w")    # запись, ОБРЕЗАЕТ файл до нуля (или создаёт)
open("f.txt", "a")    # дозапись в конец (создаёт при отсутствии)
open("f.txt", "x")    # эксклюзивное создание, FileExistsError если существует
open("f.txt", "r+")   # чтение+запись, файл должен существовать, не обрезает
open("f.txt", "w+")   # чтение+запись, ОБРЕЗАЕТ
open("f.txt", "a+")   # чтение+дозапись

# Модификаторы типа
open("f.bin", "rb")   # бинарный -> bytes
open("f.txt", "rt")   # текстовый (t по умолчанию) -> str

# Текстовый режим всегда с кодировкой!
with open("data.txt", "r", encoding="utf-8") as f:
    text = f.read()
```

### Методы файловых объектов

```python
with open("data.txt", "r", encoding="utf-8") as f:
    f.read()           # всё содержимое (или read(n) — n символов/байт)
    f.readline()       # одна строка (с \n)
    f.readlines()      # список всех строк
    for line in f:     # ленивая итерация по строкам — предпочтительно для больших файлов
        ...
    f.tell()           # текущая позиция
    f.seek(0)          # перемещение (offset, whence)
    f.seekable()       # поддерживает ли seek

with open("out.txt", "w", encoding="utf-8") as f:
    f.write("строка")  # вернёт число записанных символов
    f.writelines(["a\n", "b\n"])
    f.flush()          # принудительно сбросить буфер на диск
```

### Буферизация и newline

```python
# buffering:
#   -1  — по умолчанию (буферизация выбирается автоматически)
#    0  — без буфера (ТОЛЬКО бинарный режим)
#    1  — построчная буферизация (только текстовый режим)
#   >1  — размер буфера в байтах
open("f.txt", "w", buffering=1)        # line-buffered
open("f.bin", "wb", buffering=0)       # без буфера

# newline управляет трансляцией переводов строк (важно для CSV!)
open("data.csv", "w", encoding="utf-8", newline="")   # рекомендация для csv-модуля
#   newline=''     — отключить трансляцию (csv сам управляет \r\n)
#   newline=None   — универсальный режим (по умолчанию): \r\n -> \n при чтении
#   newline='\n'   — конкретный разделитель
```

### Кодировки и обработка ошибок

```python
# encoding — кодировка для текстового режима
open("f.txt", encoding="utf-8")
open("f.txt", encoding="cp1251")       # legacy-текст

# errors — стратегия при ошибке декодирования/кодирования
open("f.txt", encoding="utf-8", errors="strict")   # по умолчанию: бросить UnicodeDecodeError
open("f.txt", encoding="utf-8", errors="ignore")   # пропускать «битые» байты
open("f.txt", encoding="utf-8", errors="replace")  # заменять на � (U+FFFD)
open("f.txt", encoding="ascii", errors="backslashreplace")  # \xNN

# Конвертация байтов <-> текста вручную
data = "тест".encode("utf-8")          # str -> bytes
text = data.decode("utf-8")            # bytes -> str

# Обернуть существующий байтовый поток в текстовый
raw = io.BytesIO("текст".encode("utf-8"))
wrapper = io.TextIOWrapper(raw, encoding="utf-8")
print(wrapper.read())
```

## Частые вопросы на собеседовании

**Q1: В чём разница между `StringIO` и `BytesIO`?**
`StringIO` — файлоподобный объект в памяти для **текста** (`str`), работает с кодировкой на уровне Python-строк. `BytesIO` — для **байтов** (`bytes`), сырые данные без кодировки. Выбор зависит от того, текст у вас или бинарные данные.

**Q2: Зачем нужны in-memory потоки, если есть файлы?**
Для тестирования (подменить файл буфером), работы с библиотеками, ожидающими файловый объект (csv, zipfile, PIL), генерации данных в памяти без записи на диск (например, отдать файл в HTTP-ответ), перенаправления stdout/stderr.

**Q3: Что возвращает `open()` на самом деле?**
В текстовом режиме — `TextIOWrapper`, который оборачивает `BufferedReader/Writer`, который оборачивает `FileIO`. В бинарном — `BufferedReader/Writer` поверх `FileIO`. С `buffering=0` в бинарном — голый `FileIO`.

**Q4: Чем текстовый режим отличается от бинарного?**
Текстовый работает со `str`, выполняет кодирование/декодирование (encoding) и трансляцию переводов строк. Бинарный работает с `bytes` без всякой обработки. Для не-текстовых данных (картинки, архивы) обязателен бинарный.

**Q5: Что делает буферизация и зачем `flush()`?**
Буфер накапливает записи в памяти и сбрасывает их на диск пачкой, уменьшая число системных вызовов. Из-за этого данные могут быть ещё не на диске — `flush()` принудительно сбрасывает буфер, `close()` тоже делает flush. Без этого другой процесс может не увидеть свежие данные.

**Q6: Какие значения `buffering` бывают?**
`-1` — авто (по умолчанию), `0` — без буфера (только бинарный), `1` — построчно (только текст), `>1` — размер буфера в байтах.

**Q7: Зачем `newline=""` при работе с csv?**
Модуль `csv` сам управляет окончаниями строк (`\r\n`). Если не отключить трансляцию переводов строк (`newline=""`), на Windows получите лишние пустые строки между записями (двойной `\r`).

**Q8: Какая кодировка используется по умолчанию в `open()` в текстовом режиме?**
До Python 3.15 — зависящая от локали (`locale.getencoding()`), что приводит к багам (на Windows cp1251/cp1252). Поэтому **всегда указывайте `encoding="utf-8"` явно**. В 3.15 планируется UTF-8 по умолчанию (PEP 686), а с 3.10 есть предупреждение `EncodingWarning`.

**Q9: Чем `read()` отличается от итерации по файлу?**
`read()` (без аргумента) загружает весь файл в память — опасно для больших файлов. Итерация `for line in f` читает построчно, лениво, с малым потреблением памяти. Для больших файлов всегда итерируйте.

**Q10: Что делает параметр `errors` в `open()`?**
Определяет поведение при ошибке кодирования/декодирования: `strict` (бросить исключение, по умолчанию), `ignore` (пропустить), `replace` (заменить на �), `backslashreplace` и др.

**Q11: Как получить содержимое `StringIO`/`BytesIO`?**
Методом `getvalue()` — он возвращает всё содержимое буфера независимо от текущей позиции. В отличие от `read()`, который читает с текущей позиции до конца.

**Q12: Можно ли «обернуть» байтовый поток в текстовый?**
Да, через `io.TextIOWrapper(byte_stream, encoding=...)`. Так, например, работают `sys.stdin.buffer` (байты) и `sys.stdin` (текст). Это полезно для декодирования сетевых/субпроцессных байтовых потоков.

**Q13: Что произойдёт при открытии файла в режиме `"w"`?**
Файл немедленно **обрезается до нуля** (или создаётся, если нет). Любые существующие данные теряются ещё до первой записи. Для дозаписи нужен `"a"`, для чтения без потери — `"r+"`.

## Подводные камни (gotchas)

1. **Кодировка по умолчанию зависит от локали.** Не указав `encoding`, можно получить разный результат на Windows и Linux. Всегда `encoding="utf-8"`.

2. **`StringIO` для `str`, `BytesIO` для `bytes` — не путать.** Запись `str` в `BytesIO` или `bytes` в `StringIO` бросит `TypeError`.

3. **Буферизация скрывает данные.** Пока не `flush()`/`close()`, данные могут быть не на диске — другой процесс их не видит.

4. **Режим `"w"` уничтожает файл сразу.** Даже если потом произойдёт исключение, исходные данные уже потеряны.

5. **`read()` на огромном файле** загрузит всё в память и может уронить процесс. Итерируйте построчно.

6. **`newline` и csv.** Без `newline=""` модуль `csv` на Windows вставляет лишние пустые строки.

7. **`getvalue()` после `close()`** для `StringIO`/`BytesIO` бросает `ValueError` — забирайте значение до закрытия.

8. **Смешение текстового и бинарного.** `f.write(b"...")` в текстовый файл или `f.write("...")` в бинарный → `TypeError`.

9. **`seek` в текстовом режиме ограничен.** В текстовом режиме можно делать `seek` только к позициям, возвращённым `tell()` (или 0), произвольный offset не гарантирован из-за многобайтных кодировок.

10. **`errors="ignore"` молча теряет данные.** Удобно, но опасно — можно незаметно «потерять» символы.

11. **Незакрытые файлы.** Без `with` файл может остаться открытым до сборки мусора, буфер не сброситься. Всегда используйте `with`.

12. **`TextIOWrapper` закрывает обёрнутый поток.** При закрытии wrapper закроется и нижележащий байтовый поток (если не указать `close=False` через детач).

## Лучшие практики

- Всегда открывайте файлы через `with open(...) as f:` — гарантирует закрытие и flush.
- **Всегда** указывайте `encoding="utf-8"` в текстовом режиме.
- Для больших файлов итерируйте построчно (`for line in f`), не используйте `read()`/`readlines()`.
- Для csv открывайте файл с `newline=""`.
- Используйте `BytesIO`/`StringIO` для тестов и работы с библиотеками, ожидающими файловый объект.
- Забирайте `getvalue()` до закрытия буфера.
- Выбирайте бинарный режим (`"rb"/"wb"`) для любых не-текстовых данных.
- Для надёжной записи на диск (критичные данные) после `flush()` вызывайте `os.fsync(f.fileno())`.
- Осознанно выбирайте `errors`-политику; `strict` по умолчанию лучше для отлова проблем с данными.
- Используйте `"x"` для эксклюзивного создания, чтобы не перезаписать существующий файл случайно.

## Шпаргалка

```python
import io

# In-memory потоки
s = io.StringIO("init"); s.write("..."); s.seek(0); s.getvalue()   # текст
b = io.BytesIO(b"...");  b.write(b"..."); b.seek(0); b.getvalue()  # байты

# open() режимы
# r  чтение      w  запись(обрезает)   a  дозапись   x  эксклюзивное создание
# r+ чтение+запись (не обрезает)        w+ чтение+запись(обрезает)  a+ чтение+дозапись
# +b бинарный (bytes)                   +t текстовый (str, по умолч.)
with open("f.txt", "r", encoding="utf-8") as f:
    f.read(); f.readline(); [l for l in f]   # итерация — для больших файлов
with open("f.txt", "w", encoding="utf-8", newline="") as f:
    f.write("..."); f.writelines([...]); f.flush()

# Параметры open()
open(path, mode, buffering=-1, encoding="utf-8",
     errors="strict", newline=None)
#   buffering: -1 авто | 0 без буфера(bin) | 1 построчно(text) | N байт
#   errors: strict | ignore | replace | backslashreplace
#   newline: None универс. | '' для csv | '\n'

# Кодирование вручную
"текст".encode("utf-8")        # str  -> bytes
b"...".decode("utf-8")         # bytes -> str

# Обернуть байтовый поток в текст
io.TextIOWrapper(byte_stream, encoding="utf-8")

# Иерархия: FileIO(raw) -> BufferedReader/Writer -> TextIOWrapper
# open() в тексте даёт TextIOWrapper; f.buffer -> Buffered...; f.buffer.raw -> FileIO
```
