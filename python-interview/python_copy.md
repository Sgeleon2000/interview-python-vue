# copy — подготовка к собеседованию

## Что это и зачем

Модуль `copy` из стандартной библиотеки даёт инструменты для **создания копий объектов**. Это нужно, потому что в Python переменные хранят не сами объекты, а **ссылки** на них. Присваивание `b = a` не создаёт новый объект — оно создаёт ещё одно имя для того же самого объекта в памяти. Поэтому изменение через `b` видно и через `a`.

```python
a = [1, 2, 3]
b = a            # это НЕ копия — это второе имя того же списка
b.append(4)
print(a)         # => [1, 2, 3, 4]   изменили через b, видно через a
print(a is b)    # => True           один и тот же объект
```

Когда нам нужен **независимый** объект (чтобы менять его, не задевая оригинал), и встаёт вопрос о копировании. Модуль `copy` предлагает два вида копирования:

- **`copy.copy(x)`** — поверхностное (shallow). Создаёт новый объект-контейнер верхнего уровня, но вложенные объекты остаются **общими** (копируются ссылки на них).
- **`copy.deepcopy(x)`** — глубокое (deep). Рекурсивно копирует объект и **все** вложенные объекты, получая полностью независимую структуру.

Зачем это важно знать на собеседовании: понимание ссылочной семантики, разницы mutable/immutable и того, как глубоко идёт копирование — базовый признак того, что кандидат понимает модель памяти Python. Ошибки на этой почве (общий вложенный список, mutable default argument) — классические баги в продакшене.

## Ключевые концепции

### 1. Присваивание vs shallow vs deep

Возьмём вложенную структуру — список списков — и сравним три подхода.

```python
import copy

original = [[1, 2, 3], [4, 5, 6]]

assigned = original              # присваивание — тот же объект
shallow  = copy.copy(original)   # поверхностная копия
deep     = copy.deepcopy(original)  # глубокая копия

# Проверим идентичность объекта верхнего уровня
print(assigned is original)  # => True   (тот же список)
print(shallow  is original)  # => False  (новый список верхнего уровня)
print(deep     is original)  # => False  (новый список)

# А теперь — вложенные элементы
print(assigned[0] is original[0])  # => True   тот же вложенный список
print(shallow[0]  is original[0])  # => True   ВНИМАНИЕ: вложенный общий!
print(deep[0]     is original[0])  # => False  вложенный тоже скопирован
```

Графически:

```
original ──► [ ref0, ref1 ]
                 │     │
                 ▼     ▼
              [1,2,3] [4,5,6]

assigned ──► то же самое (та же стрелка, что и original)

shallow  ──► [ ref0, ref1 ]   ← НОВЫЙ контейнер,
                 │     │          но ref0/ref1 указывают
                 ▼     ▼          на ТЕ ЖЕ вложенные списки
              [1,2,3] [4,5,6]

deep     ──► [ ref0', ref1' ] ← НОВЫЙ контейнер
                 │      │
                 ▼      ▼
              [1,2,3]' [4,5,6]'  ← НОВЫЕ вложенные списки
```

### 2. Демонстрация: shallow копирует только верхний уровень

```python
import copy

original = [[1, 2, 3], [4, 5, 6]]
shallow  = copy.copy(original)

# Меняем верхний уровень копии — оригинал НЕ затронут
shallow.append([7, 8, 9])
print(original)  # => [[1, 2, 3], [4, 5, 6]]            без изменений
print(shallow)   # => [[1, 2, 3], [4, 5, 6], [7, 8, 9]]

# Меняем ВЛОЖЕННЫЙ объект — затронут и оригинал (он общий!)
shallow[0].append(999)
print(original)  # => [[1, 2, 3, 999], [4, 5, 6]]   <-- оригинал изменился!
print(shallow)   # => [[1, 2, 3, 999], [4, 5, 6], [7, 8, 9]]
```

С `deepcopy` такой проблемы нет:

```python
import copy

original = [[1, 2, 3], [4, 5, 6]]
deep     = copy.deepcopy(original)

deep[0].append(999)
print(original)  # => [[1, 2, 3], [4, 5, 6]]        НЕ изменился
print(deep)      # => [[1, 2, 3, 999], [4, 5, 6]]
```

### 3. Способы поверхностного копирования

`copy.copy(x)` — не единственный способ сделать shallow-копию. У встроенных контейнеров есть свои идиомы, и все они эквивалентны по глубине (копируют только верхний уровень).

```python
import copy

lst = [[1, 2], [3, 4]]

c1 = copy.copy(lst)   # через модуль copy
c2 = list(lst)        # конструктор list
c3 = lst[:]           # срез целиком
c4 = lst.copy()       # метод .copy() у списка

# Все — поверхностные: новый верхний уровень, общие вложенные
for c in (c1, c2, c3, c4):
    print(c is lst, c[0] is lst[0])
# => False True
# => False True
# => False True
# => False True

# Для словарей:
d = {"a": [1, 2], "b": [3, 4]}
dc1 = copy.copy(d)
dc2 = dict(d)
dc3 = d.copy()
print(dc1 is d, dc1["a"] is d["a"])  # => False True   (тоже shallow)

# Для множеств:
s = {1, 2, 3}
print(set(s) is s, s.copy() is s)    # => False False  (новые множества)
```

### 4. memo-словарь

`deepcopy` принимает второй параметр — `memo` (словарь-«мемоизатор»). Он решает две задачи:

1. **Циклические ссылки** — без `memo` рекурсивный обход зациклился бы навсегда.
2. **Сохранение совместного использования** — если на один и тот же объект ссылаются несколько раз, в копии он тоже должен остаться одним объектом, а не продублироваться.

Ключ memo — `id()` исходного объекта, значение — его уже созданная копия.

```python
import copy

shared = [1, 2, 3]
data = [shared, shared]   # один и тот же список дважды
print(data[0] is data[1])  # => True

deep = copy.deepcopy(data)
print(deep[0] is deep[1])  # => True   совместность СОХРАНЕНА благодаря memo
print(deep[0] is shared)   # => False  но это уже новая копия
```

### 5. deepcopy сохраняет циклы

```python
import copy

lst = [1, 2]
lst.append(lst)          # список ссылается сам на себя
print(lst[2] is lst)     # => True

deep = copy.deepcopy(lst)  # НЕ зацикливается — спасает memo
print(deep[2] is deep)   # => True   цикл воспроизведён в копии
print(deep is lst)       # => False  но это независимая копия
print(deep[:2])          # => [1, 2]
```

### 6. Изменяемость и копирование неизменяемых объектов

Для неизменяемых (immutable) объектов копирование часто бессмысленно: раз объект нельзя изменить, нет смысла создавать его дубликат. Поэтому `copy`/`deepcopy` могут вернуть **тот же самый объект**.

```python
import copy

x = 42
print(copy.copy(x) is x)      # => True   int неизменяем
print(copy.deepcopy(x) is x)  # => True

s = "hello"
print(copy.copy(s) is s)      # => True   str неизменяем

t = (1, 2, 3)
print(copy.copy(t) is t)      # => True   кортеж из неизменяемых
print(copy.deepcopy(t) is t)  # => True

# НО: кортеж с изменяемым содержимым deepcopy скопирует
t2 = (1, [2, 3])
dt2 = copy.deepcopy(t2)
print(dt2 is t2)              # => False  внутри есть mutable список
print(dt2[1] is t2[1])        # => False  вложенный список скопирован
```

## Основные функции/классы/методы

### `copy.copy(x)`

Возвращает поверхностную копию `x`. Логика выбора способа копирования:
1. Для неизменяемых атомарных типов (int, str, bytes, tuple из immutable, None, функции, классы и др.) — возвращает объект как есть.
2. Если у объекта есть метод `__copy__()` — вызывает его.
3. Иначе использует протокол `__reduce_ex__`/`__reduce__` (тот же, что и pickle) для воссоздания объекта.

```python
import copy

class Box:
    def __init__(self, items):
        self.items = items

b = Box([1, 2, 3])
c = copy.copy(b)
print(c is b)                 # => False  новый Box
print(c.items is b.items)     # => True   список общий (shallow!)
c.items.append(4)
print(b.items)                # => [1, 2, 3, 4]   оригинал затронут
```

### `copy.deepcopy(x, memo=None)`

Возвращает глубокую копию: рекурсивно копирует объект и всё содержимое. Внутренне ведёт `memo`-словарь.

```python
import copy

class Box:
    def __init__(self, items):
        self.items = items

b = Box([1, 2, 3])
c = copy.deepcopy(b)
print(c is b)                 # => False
print(c.items is b.items)     # => False  список тоже скопирован
c.items.append(4)
print(b.items)                # => [1, 2, 3]   оригинал НЕ затронут
```

### `__copy__(self)`

Метод-хук для кастомизации поверхностного копирования. Вызывается из `copy.copy()`. Должен вернуть готовую копию.

```python
import copy

class Cache:
    def __init__(self, data):
        self.data = data
        self.hits = 0          # счётчик не должен переноситься в копию

    def __copy__(self):
        # создаём «пустой» экземпляр в обход __init__
        cls = self.__class__
        new = cls.__new__(cls)
        # копируем поля верхнего уровня, hits сбрасываем
        new.data = self.data   # shallow: ссылка на тот же data
        new.hits = 0
        return new

orig = Cache({"key": "value"})
orig.hits = 100
copied = copy.copy(orig)
print(copied.hits)            # => 0      (сброшен нашим __copy__)
print(copied.data is orig.data)  # => True (shallow — общий dict)
```

### `__deepcopy__(self, memo)`

Метод-хук для кастомизации глубокого копирования. Принимает `memo` — и его **обязательно** нужно прокидывать дальше во вложенные `deepcopy`, иначе сломаются циклы и совместность объектов.

```python
import copy

class Node:
    def __init__(self, value):
        self.value = value
        self.children = []

    def __deepcopy__(self, memo):
        cls = self.__class__
        new = cls.__new__(cls)
        # КЛЮЧЕВОЙ момент: регистрируем себя в memo ДО рекурсии,
        # чтобы циклические ссылки нашли уже созданную копию
        memo[id(self)] = new
        # прокидываем memo во вложенные deepcopy
        new.value = copy.deepcopy(self.value, memo)
        new.children = copy.deepcopy(self.children, memo)
        return new

root = Node("root")
child = Node("child")
root.children.append(child)
child.children.append(root)   # цикл root <-> child

clone = copy.deepcopy(root)
print(clone is root)                      # => False
print(clone.children[0].children[0] is clone)  # => True  цикл сохранён, не задублирован
```

Если в `__deepcopy__` **не** прокинуть `memo`, цикл приведёт к бесконечной рекурсии и `RecursionError`.

### `copy.replace(obj, /, **changes)` (Python 3.13+)

Новая функция: создаёт **изменённую копию** объекта, заменяя указанные поля. Объект должен поддерживать протокол `__replace__`. Из коробки работает для `dataclass`, `namedtuple`, `datetime` и др. По сути аналог `dataclasses.replace`, но обобщённый.

```python
import copy
from dataclasses import dataclass

@dataclass
class Point:
    x: int
    y: int
    z: int = 0

p = Point(1, 2, 3)
p2 = copy.replace(p, y=20)    # копия с заменённым y
print(p)                      # => Point(x=1, y=2, z=3)   оригинал цел
print(p2)                     # => Point(x=1, y=20, z=3)

# Кастомизация через __replace__:
class Config:
    def __init__(self, host, port):
        self.host = host
        self.port = port

    def __replace__(self, /, **changes):
        return Config(
            host=changes.get("host", self.host),
            port=changes.get("port", self.port),
        )

c = Config("localhost", 8080)
c2 = copy.replace(c, port=9090)
print(c2.host, c2.port)       # => localhost 9090
```

`copy.replace` не делает глубокую копию полей — это «обновлённая поверхностная» копия (новый объект, неуказанные поля переносятся по ссылке).

## Частые вопросы на собеседовании

**Q1. В чём разница между присваиванием, `copy.copy` и `copy.deepcopy`?**

A. Присваивание (`b = a`) не создаёт новый объект — это новое имя для того же объекта (`a is b` → `True`), любое мутирующее изменение видно через оба имени. `copy.copy` создаёт новый объект верхнего уровня, но вложенные объекты остаются общими (копируются ссылки) — поверхностное копирование. `copy.deepcopy` рекурсивно копирует объект и все вложенные структуры, получая полностью независимый граф объектов.

---

**Q2. Почему после shallow-копии изменение вложенного списка меняет и оригинал?**

A. Потому что shallow-копия дублирует только контейнер верхнего уровня, а в его ячейках лежат **те же самые ссылки** на вложенные объекты. Когда мы делаем `shallow[0].append(x)`, мы мутируем общий вложенный объект, на который указывают и оригинал, и копия. Чтобы вложенные объекты тоже были независимыми, нужен `deepcopy`.

```python
import copy
a = [[1]]
b = copy.copy(a)
b[0].append(2)
print(a)  # => [[1, 2]]   общий вложенный список
```

---

**Q3. Какими способами можно сделать поверхностную копию списка/словаря?**

A. Списка: `copy.copy(lst)`, `list(lst)`, `lst[:]`, `lst.copy()`. Словаря: `copy.copy(d)`, `dict(d)`, `d.copy()`, `{**d}`. Множества: `set(s)`, `s.copy()`. Все они поверхностные — копируют верхний уровень, вложенные объекты общие. Для глубокого копирования любой из них не подходит, нужен `copy.deepcopy`.

---

**Q4. Что такое `memo` в `deepcopy` и зачем он нужен?**

A. `memo` — это словарь `{id(оригинала): копия}`, который `deepcopy` ведёт во время рекурсивного обхода. Он решает две задачи: (1) предотвращает бесконечную рекурсию при **циклических ссылках** — встретив уже обработанный объект, deepcopy берёт готовую копию из memo; (2) сохраняет **совместное использование** — если один объект встречается в структуре несколько раз, в копии он останется единственным экземпляром, а не продублируется. При реализации `__deepcopy__` важно прокидывать `memo` во вложенные вызовы и регистрировать себя в нём до рекурсии.

---

**Q5. Как реализовать кастомное копирование класса?**

A. Определить методы `__copy__(self)` для `copy.copy` и `__deepcopy__(self, memo)` для `copy.deepcopy`. Они должны вернуть готовую копию. Типичный приём — создать экземпляр через `cls.__new__(cls)` в обход `__init__`, затем заполнить нужные поля. В `__deepcopy__` обязательно: зарегистрировать `memo[id(self)] = new` до рекурсивного копирования полей и передавать `memo` дальше: `copy.deepcopy(self.field, memo)`. Это нужно для подзадач, для тяжёлых-в-инициализации объектов или когда часть состояния (кэши, соединения, счётчики) копировать не следует.

---

**Q6. Какие объекты `copy`/`deepcopy` НЕ копируют, а возвращают как есть?**

A. Модули, функции, методы, классы (типы), `None`, а также неизменяемые атомарные значения. Кроме того, не копируются ресурсо-подобные объекты, которые не имеют осмысленной семантики копирования — сокеты, открытые файлы, дескрипторы, потоки и подобное (их `__reduce__` либо отсутствует, либо бросает ошибку; часто их специально исключают). Это сделано намеренно: дублировать функцию или модуль бессмысленно и опасно, а скопировать открытый сокет невозможно. Для immutable объектов возврат того же объекта — оптимизация, ведь их всё равно нельзя изменить.

```python
import copy, math
print(copy.deepcopy(math) is math)        # => True  модуль не копируется
f = lambda x: x
print(copy.deepcopy(f) is f)              # => True  функция не копируется
print(copy.deepcopy(int) is int)          # => True  класс не копируется
```

---

**Q7. Копирует ли `copy.copy(5)` или `copy.copy("abc")` объект?**

A. Нет, для неизменяемых атомарных объектов (int, str, bytes, bool, None, кортежи из immutable) возвращается **тот же** объект — `copy.copy(x) is x` будет `True`. Раз объект нельзя изменить, отдельная копия не нужна, и Python экономит на её создании. Кортеж — особый случай: если он содержит изменяемые элементы, `deepcopy` всё же создаст новый кортеж с копиями содержимого.

**Q8. Почему mutable default argument опасен и как это связано с копированием?**

A. Значение по умолчанию вычисляется **один раз** при определении функции и затем переиспользуется при каждом вызове. Если это изменяемый объект (список, словарь), все вызовы делят один и тот же объект, и мутации накапливаются между вызовами.

```python
def add_item(item, target=[]):   # АНТИПАТТЕРН
    target.append(item)
    return target

print(add_item(1))  # => [1]
print(add_item(2))  # => [1, 2]   <-- не [2], общий список «протёк»!
```

Связь с копированием: правильное решение — использовать `None` как маркер и **создавать новый объект** внутри функции (фактически делая «свежую копию» дефолта на каждый вызов):

```python
def add_item(item, target=None):
    if target is None:
        target = []          # новый список на каждый вызов
    target.append(item)
    return target

print(add_item(1))  # => [1]
print(add_item(2))  # => [2]   корректно
```

Если же дефолт обязан быть готовым шаблоном, его копируют: `target = copy.deepcopy(DEFAULT_TEMPLATE)`.

**Q9. `deepcopy` медленный — какие есть альтернативы?**

A. `deepcopy` универсален, но дорог: он обходит весь граф объектов, ведёт memo, диспетчеризует по типам и вызывает протокол сериализации. Альтернативы:
- **Ручное копирование** известной структуры (`{k: list(v) for k, v in d.items()}`) — самое быстрое, если структура известна.
- **`pickle.loads(pickle.dumps(obj))`** — иногда быстрее `deepcopy` для больших структур, но требует pickle-совместимости.
- **`json.loads(json.dumps(obj))`** — для простых JSON-подобных данных (только базовые типы, теряет типы вроде кортежей/datetime).
- **Конструкторы коллекций** для shallow-нужд, если глубокая копия не требуется.
- Иногда лучший вариант — **не копировать вовсе**, а использовать неизменяемые структуры (tuple, frozenset, frozen dataclass).

```python
import copy, pickle, json

data = {"items": [1, 2, 3], "meta": {"v": 1}}

d1 = copy.deepcopy(data)
d2 = pickle.loads(pickle.dumps(data))
d3 = json.loads(json.dumps(data))
print(d1 == d2 == d3)   # => True   (для JSON-совместимых данных)
```

**Q10. Какую роль играет `__reduce__` в копировании?**

A. Когда у объекта нет `__copy__`/`__deepcopy__`, модуль `copy` использует тот же протокол сериализации, что и `pickle`: вызывает `__reduce_ex__(protocol)` (по умолчанию делегирует к `__reduce__`). Этот метод возвращает «рецепт» воссоздания объекта — обычно кортеж `(callable, args, state, ...)`. `copy` применяет рецепт: вызывает `callable(*args)`, затем восстанавливает состояние (через `__setstate__` или присваивание `__dict__`). Поэтому переопределение `__reduce__`/`__getstate__`/`__setstate__` влияет одновременно и на pickle, и на copy. Если объект непиклимый и не определяет copy-хуки, `deepcopy` упадёт с ошибкой.

```python
import copy

class Temp:
    def __init__(self, x):
        self.x = x
    def __getstate__(self):
        # исключаем поле из копии/сериализации
        state = self.__dict__.copy()
        return state
    def __setstate__(self, state):
        self.__dict__.update(state)

t = Temp(10)
c = copy.deepcopy(t)   # работает через __reduce_ex__ -> __getstate__/__setstate__
print(c.x, c is t)     # => 10 False
```

**Q11. Что вернёт `copy.copy` для кортежа из неизменяемых элементов?**

A. Тот же кортеж. Кортеж неизменяем, его элементы (например, числа) тоже, поэтому копировать нечего — возвращается исходный объект (`is` → `True`). Это касается только «полностью неизменяемых» кортежей. Если внутри кортежа лежит изменяемый объект, то `copy.copy` всё равно вернёт тот же кортеж (shallow), а `copy.deepcopy` создаст новый кортеж с глубокими копиями элементов.

**Q12. Когда достаточно shallow, а когда обязательно deep?**

A. Shallow достаточно, если: структура «плоская» (нет вложенных изменяемых объектов), либо вложенные объекты неизменяемы, либо вы намеренно хотите разделять вложенное состояние между копиями. Deep обязателен, если нужно **полностью независимое** дерево объектов, которое можно мутировать, не влияя на оригинал (вложенные списки/словари/объекты). На практике для конфигов, снапшотов состояния, undo/redo чаще нужен deep; для быстрого «отрежу верхний уровень» — shallow.

## Подводные камни (gotchas)

1. **Shallow-копия не защищает вложенные объекты.** `copy.copy`, `list()`, `[:]`, `.copy()` копируют только верхний уровень. Мутация вложенного объекта затронет и оригинал.

2. **`dict.copy()` и `{**d}` — поверхностные.** Значения-списки/словари остаются общими.
   ```python
   import copy
   d = {"a": [1]}
   c = d.copy()
   c["a"].append(2)
   print(d)  # => {'a': [1, 2]}   общий список
   ```

3. **Mutable default argument.** Дефолт вычисляется один раз; изменяемый дефолт «протекает» между вызовами. Используйте `None` + создание объекта внутри.

4. **Забыть `memo` в `__deepcopy__`.** Без `memo[id(self)] = new` и передачи `memo` дальше — `RecursionError` на циклах и дублирование совместно используемых объектов.

5. **`deepcopy` тяжёлый.** На больших/глубоких структурах заметно медленный. Не вызывайте его в горячих циклах без необходимости.

6. **`deepcopy` падает на «некопируемых» ресурсах.** Если объект содержит сокет/файл/блокировку без поддержки copy-протокола — будет исключение. Решение: исключить такие поля через `__getstate__`/`__deepcopy__`.

7. **Immutable могут вернуться как тот же объект.** `copy.copy(x) is x` для int/str/tuple — это нормально, не баг. Не полагайтесь на то, что копия всегда новый объект.

8. **`copy.replace` (3.13+) — не deep.** Неуказанные поля переносятся по ссылке; это «обновлённая поверхностная копия», а не глубокая.

9. **`json`-копирование теряет типы.** Кортежи становятся списками, ключи-числа становятся строками, datetime/set/None-семантика искажается. Подходит только для строго JSON-совместимых данных.

10. **`copy` и `pickle` связаны.** Переопределяя `__reduce__`/`__getstate__`/`__setstate__`, вы меняете поведение обоих. Тестируйте оба сценария.

---

## Лучшие практики

- **Выбирайте минимально достаточную глубину.** Не используйте `deepcopy`, если хватает shallow или вообще копии не нужно.
- **Предпочитайте неизменяемые структуры.** `tuple`, `frozenset`, `frozen=True` dataclass, `NamedTuple` снимают вопрос копирования вовсе.
- **Для dataclass используйте `copy.replace` / `dataclasses.replace`** вместо ручного клонирования с правками.
- **В `__deepcopy__` всегда регистрируйте `memo[id(self)] = new` до рекурсии** и прокидывайте `memo` во вложенные `deepcopy`.
- **Исключайте несериализуемое состояние** (соединения, локи, кэши) через `__getstate__`/`__deepcopy__`/`__copy__`.
- **Никогда не используйте mutable default arguments.** Применяйте паттерн `arg=None` + создание внутри.
- **Профилируйте, если копирование на горячем пути.** Рассмотрите ручное копирование или `pickle` как более быстрые альтернативы.
- **Помните о связке copy ↔ pickle.** Один протокол `__reduce__` обслуживает оба; покрывайте тестами.
- **Документируйте семантику копирования** ваших классов, если она нетривиальна (что делится, что клонируется, что сбрасывается).

## Шпаргалка

```python
import copy

# --- Три уровня "копирования" ---
b = a                    # присваивание: тот же объект (a is b -> True)
s = copy.copy(a)         # shallow: новый верхний уровень, вложенное общее
d = copy.deepcopy(a)     # deep: полностью независимый граф объектов

# --- Способы shallow-копии ---
list(x); x[:]; x.copy()  # список
dict(x); x.copy(); {**x} # словарь
set(x); x.copy()         # множество

# --- deepcopy с memo (циклы и совместность) ---
copy.deepcopy(obj)              # memo создаётся автоматически
deep[0] is deep[1]             # совместность сохранена
deep[2] is deep                # цикл сохранён

# --- Кастомизация ---
class C:
    def __copy__(self):
        new = self.__class__.__new__(self.__class__)
        new.__dict__.update(self.__dict__)   # shallow
        return new
    def __deepcopy__(self, memo):
        new = self.__class__.__new__(self.__class__)
        memo[id(self)] = new                 # ВАЖНО: до рекурсии
        for k, v in self.__dict__.items():
            setattr(new, k, copy.deepcopy(v, memo))  # прокидываем memo
        return new

# --- Immutable: возвращается тот же объект ---
copy.copy(42) is 42            # True (int)
copy.copy("x") is "x"          # True (str)
copy.copy((1, 2)) is (1, 2)    # обычно True (tuple из immutable)

# --- НЕ копируются (возвращаются как есть) ---
# модули, функции, методы, классы, None, сокеты, файлы

# --- copy.replace (Python 3.13+) ---
copy.replace(point, y=20)      # копия с заменой поля (shallow-обновление)

# --- Альтернативы deepcopy (по скорости/задаче) ---
pickle.loads(pickle.dumps(x))  # часто быстрее на больших структурах
json.loads(json.dumps(x))      # только JSON-совместимые данные
{k: list(v) for k, v in d.items()}  # ручная копия известной структуры

# --- Mutable default argument (антипаттерн -> фикс) ---
def f(x, acc=None):
    if acc is None:
        acc = []               # новый объект на каждый вызов
    acc.append(x)
    return acc
```
