# Встроенные функции (Built-in Functions) — подготовка к собеседованию

## Что это и зачем

**Встроенные функции (built-in functions)** — это функции и классы, которые доступны в любом месте кода Python без явного импорта. Они находятся в неявно доступном модуле `builtins` (в CPython реализованы на C для скорости) и составляют «стандартный инструментарий» языка.

Зачем это знать на собеседовании:
- Это базовый словарь языка: `len`, `range`, `enumerate`, `zip`, `sorted` встречаются в каждом скрипте.
- Многие из них тесно связаны с протоколами Python (итераторы, дескрипторы, дандер-методы `__len__`, `__iter__`, `__call__`, `__hash__`). Знание built-in функций = знание того, как устроен язык изнутри.
- На middle/senior уровне спрашивают не «что делает `sorted`», а нюансы: ленивость `map`/`filter`, стабильность сортировки, разница `is`/`==`, опасность `eval`/`exec`, как `super()` ищет методы по MRO, чем `property` отличается от обычного атрибута.

Технически многие «функции» — на самом деле классы (`int`, `list`, `dict`, `type`, `property`, `enumerate`, `zip`, `map`, `filter`, `range`, `reversed`, `slice`), вызов которых создаёт объект. Это важное различие, которое всплывает в каверзных вопросах.

## Ключевые концепции

- **`builtins` — это модуль.** Все встроенные имена живут в `builtins`. `__builtins__` доступен в каждом модуле. Имена ищутся по правилу **LEGB** (Local → Enclosing → Global → Built-in), поэтому локальная переменная `list` «затеняет» встроенную функцию.
- **Многие built-in полагаются на дандер-методы.** `len(x)` вызывает `x.__len__()`, `iter(x)` — `x.__iter__()`, `repr(x)` — `x.__repr__()`, `abs(x)` — `x.__abs__()`, `hash(x)` — `x.__hash__()`, `bool(x)` — `x.__bool__()`/`__len__()`. Реализуя протоколы, вы делаете свои объекты совместимыми со встроенными функциями.
- **Ленивость (lazy evaluation).** В Python 3 `range`, `map`, `filter`, `zip`, `enumerate`, `reversed`, `dict.keys()` возвращают ленивые объекты/итераторы, а не списки. Они вычисляются по требованию и экономят память.
- **Итераторы одноразовы.** `map`, `filter`, `zip`, `enumerate`, генераторы исчерпываются после первого прохода. `range` — исключение: это переиспользуемая последовательность, а не итератор.
- **Функции vs классы.** `sorted`, `len`, `sum`, `min`, `max`, `any`, `all` — настоящие функции. `list`, `dict`, `set`, `int`, `str`, `type`, `enumerate`, `zip`, `map`, `filter`, `range`, `property`, `staticmethod`, `classmethod` — классы (конструкторы типов).
- **Идентичность vs равенство.** `id(x)` — адрес/идентификатор объекта, `is` сравнивает идентичность, `==` — значение. `hash(x)` — целое для хеш-таблиц (требует неизменяемости).
- **Самоанализ (introspection).** `type`, `isinstance`, `issubclass`, `dir`, `vars`, `getattr`, `hasattr`, `callable`, `repr` позволяют исследовать объекты во время выполнения — основа метапрограммирования, ORM, сериализаторов.
- **Динамическое выполнение кода.** `eval`, `exec`, `compile` исполняют строки как код — мощно и опасно (инъекции). `globals()`/`locals()` дают доступ к пространствам имён.

## Основные функции/классы/методы

### print — вывод

```python
# sep, end, file, flush — ключевые параметры
print("a", "b", "c", sep="-", end="!\n")   # => a-b-c!

import sys
print("ошибка", file=sys.stderr)            # вывод в stderr
print("loading...", end="", flush=True)     # принудительно сбросить буфер

# print возвращает None — частая ловушка
x = print("hi")
print(x)                                     # => None
```

### len — длина

```python
len([1, 2, 3])      # => 3
len("привет")       # => 6 (количество символов, не байтов)
len({"a": 1})       # => 1

# len() вызывает __len__; работает только для Sized-объектов
class Box:
    def __len__(self):
        return 42
len(Box())          # => 42

# для генераторов len не работает — нет __len__
# len(x for x in range(3))  # TypeError
```

### range — арифметическая последовательность

```python
range(5)            # 0,1,2,3,4
range(2, 8)         # 2..7
range(10, 0, -2)    # 10,8,6,4,2
list(range(0, 1, 0.1))  # TypeError: range принимает только int

# range — ленивая ПОСЛЕДОВАТЕЛЬНОСТЬ (не итератор): можно индексировать, брать len, проходить много раз
r = range(1_000_000)
print(len(r), r[100], 500 in r)  # => 1000000 100 True (проверка "in" работает за O(1))
```

### enumerate — индекс + значение

```python
for i, val in enumerate(["a", "b", "c"], start=1):
    print(i, val)   # => 1 a / 2 b / 3 c

# Это класс, возвращает ленивый итератор пар (index, item)
list(enumerate("xy"))  # => [(0, 'x'), (1, 'y')]
```

### zip — объединение последовательностей

```python
names = ["Аня", "Боб"]
ages = [30, 25]
list(zip(names, ages))      # => [('Аня', 30), ('Боб', 25)]

# Останавливается по самой короткой
list(zip([1, 2, 3], [10, 20]))  # => [(1, 10), (2, 20)]

# strict=True (Python 3.10+) — ошибка при разной длине
# list(zip([1,2,3], [10,20], strict=True))  # ValueError

# Идиома "распаковка/транспонирование" через zip(*...)
pairs = [(1, 'a'), (2, 'b'), (3, 'c')]
nums, letters = zip(*pairs)
print(nums, letters)        # => (1, 2, 3) ('a', 'b', 'c')
```

### map / filter — функциональные преобразования (ленивые)

```python
m = map(str.upper, ["a", "b"])
print(m)                # => <map object ...> (ленивый итератор!)
print(list(m))          # => ['A', 'B']
print(list(m))          # => []  (итератор исчерпан)

# map по нескольким итераторам
list(map(lambda x, y: x + y, [1, 2], [10, 20]))  # => [11, 22]

# filter оставляет элементы, для которых функция истинна
list(filter(lambda x: x % 2, range(6)))          # => [1, 3, 5]
list(filter(None, [0, 1, "", "x", [], [1]]))     # => [1, 'x', [1]] (filter None убирает ложные)

# Часто list/genexpr читабельнее: [x.upper() for x in ["a","b"]]
```

### sorted — сортировка (всегда новый список)

```python
sorted([3, 1, 2])                       # => [1, 2, 3]
sorted([3, 1, 2], reverse=True)         # => [3, 2, 1]
sorted(["bb", "a", "ccc"], key=len)     # => ['a', 'bb', 'ccc']

# Сортировка по нескольким ключам
people = [("Аня", 30), ("Боб", 30), ("Вика", 25)]
sorted(people, key=lambda p: (-p[1], p[0]))  # сначала по возрасту убыв., потом по имени

# Сортировка СТАБИЛЬНА (Timsort): равные элементы сохраняют исходный порядок
# sorted vs list.sort: sorted возвращает НОВЫЙ список, list.sort сортирует на месте и возвращает None
data = [3, 1, 2]
print(data.sort())   # => None (частая ошибка: x = data.sort())
```

### sum, min, max — агрегация

```python
sum([1, 2, 3])              # => 6
sum([1, 2, 3], 100)         # => 106 (второй аргумент — стартовое значение)
sum([[1], [2]], [])         # => [1, 2] (но для списков лучше itertools.chain / "".join для строк)

min([3, 1, 2])              # => 1
max("a", "b", "c")          # => 'c'
max([], default=0)          # => 0 (без default — ValueError на пустом)

# key работает как в sorted
max(["a", "bbb", "cc"], key=len)  # => 'bbb'
min(people, key=lambda p: p[1])   # самый молодой
```

### any / all — логические свёртки

```python
any([0, "", None, 5])       # => True (хоть один истинный)
all([1, "x", [0]])          # => True (все истинные; [0] — непустой список = True)

all([])                     # => True  (vacuous truth — пустое = все истинны)
any([])                     # => False

# Короткое замыкание (short-circuit): останавливаются на первом решающем элементе
all(x > 0 for x in [1, -1, очень_дорогая_функция()])  # остановится на -1
```

### isinstance / issubclass — проверка типов

```python
isinstance(5, int)                  # => True
isinstance(True, int)               # => True (bool — подкласс int!)
isinstance(5, (int, float))         # => True (кортеж типов = "или")

issubclass(bool, int)               # => True
issubclass(int, object)             # => True

# С abc-протоколами
from collections.abc import Iterable
isinstance([1, 2], Iterable)        # => True

# isinstance уважает наследование, type(x) == X — НЕТ (см. gotchas)
```

### getattr / setattr / hasattr / delattr — динамические атрибуты

```python
class User:
    name = "Аня"

u = User()
getattr(u, "name")              # => 'Аня'
getattr(u, "age", 0)            # => 0 (default, если нет атрибута)
hasattr(u, "name")              # => True
setattr(u, "age", 30)           # u.age = 30
delattr(u, "age")               # del u.age

# Динамический доступ по имени из строки — основа сериализаторов/ORM
field = "name"
print(getattr(u, field))        # => 'Аня'
```

### super — обращение к родителю по MRO

```python
class A:
    def greet(self):
        return "A"

class B(A):
    def greet(self):
        return "B->" + super().greet()  # вызов следующего в MRO

class C(A):
    def greet(self):
        return "C->" + super().greet()

class D(B, C):
    def greet(self):
        return "D->" + super().greet()

print(D().greet())              # => D->B->C->A
print([c.__name__ for c in D.__mro__])  # => ['D', 'B', 'C', 'A', 'object']
# super() идёт по MRO (C3-линеаризация), а НЕ просто к "родителю"
```

### property — управляемые атрибуты (дескриптор)

```python
class Circle:
    def __init__(self, r):
        self._r = r

    @property
    def radius(self):           # геттер
        return self._r

    @radius.setter
    def radius(self, value):    # сеттер с валидацией
        if value < 0:
            raise ValueError("Радиус не может быть отрицательным")
        self._r = value

    @property
    def area(self):             # вычисляемый атрибут только для чтения
        return 3.14159 * self._r ** 2

c = Circle(5)
print(c.area)                   # => 78.53975 (без скобок!)
c.radius = 10                   # вызовет setter
# c.radius = -1                 # ValueError
```

### staticmethod / classmethod — методы класса

```python
class Pizza:
    size_default = "M"

    def __init__(self, ingredients):
        self.ingredients = ingredients

    @classmethod
    def margherita(cls):        # альтернативный конструктор; cls = класс (учитывает наследование)
        return cls(["сыр", "томаты"])

    @staticmethod
    def is_valid_size(size):    # просто функция в namespace класса, без self/cls
        return size in {"S", "M", "L"}

p = Pizza.margherita()          # фабричный метод
Pizza.is_valid_size("XL")       # => False

# classmethod знает cls -> корректно работает с подклассами и alternative constructors
# staticmethod не имеет доступа ни к экземпляру, ни к классу
```

### iter / next — протокол итерации

```python
it = iter([1, 2, 3])
next(it)                        # => 1
next(it)                        # => 2
next(it, "конец")               # => 3
next(it, "конец")               # => 'конец' (default вместо StopIteration)

# iter с двумя аргументами: вызывает callable, пока не вернёт sentinel
import random
# читать, пока не встретим строку 'STOP'
# for line in iter(input, 'STOP'): ...

# Свой итератор
class Counter:
    def __init__(self, n): self.n = n; self.i = 0
    def __iter__(self): return self
    def __next__(self):
        if self.i >= self.n: raise StopIteration
        self.i += 1
        return self.i
list(Counter(3))                # => [1, 2, 3]
```

### vars / dir — самоанализ пространства имён

```python
class P:
    cls_attr = 1
    def __init__(self):
        self.x = 10
        self.y = 20

p = P()
vars(p)                         # => {'x': 10, 'y': 20} (это p.__dict__)
vars()                          # без аргумента ≈ locals()

dir(p)                          # список всех имён (атрибуты + методы + унаследованные + дандеры)
[a for a in dir(p) if not a.startswith("_")]  # => ['cls_attr', 'x', 'y']
```

### eval / exec / compile — динамическое выполнение

```python
eval("2 + 3 * 4")               # => 14 (вычисляет ВЫРАЖЕНИЕ, возвращает значение)
exec("x = 5\nprint(x)")         # выполняет ОПЕРАТОРЫ, возвращает None

# Ограничение пространства имён (но это НЕ настоящая песочница!)

```

### repr / format — строковые представления


### hash / id — хеш и идентичность

```python

hash((1, 2))                    # хешируемо (кортеж неизменяем)
```


## Частые вопросы на собеседовании

**Q: Чем `map`/`filter` в Python 3 отличаются от Python 2? Почему это важно?**
A: В Python 2 они возвращали списки, в Python 3 — ленивые итераторы. Плюс: экономия памяти на больших данных, можно работать с бесконечными последовательностями. Минус-ловушка: итератор одноразовый (исчерпывается после первого прохода), у него нет `len`, нельзя индексировать. Чтобы получить список, нужно явно обернуть в `list()`. Часто list/generator comprehension читабельнее.

**Q: В чём разница между `sorted(x)` и `x.sort()`?**
A: `sorted()` — встроенная функция, принимает любой итерируемый объект и возвращает **новый** отсортированный список, исходный не меняется. `list.sort()` — метод списка, сортирует **на месте**, возвращает `None`. Классическая ошибка: `result = my_list.sort()` — `result` будет `None`. Обе используют Timsort и **стабильны** (равные элементы сохраняют порядок), поддерживают `key` и `reverse`.

**Q: Объясните разницу `is` vs `==` и роль `id()`/`hash()`.**
A: `==` сравнивает **значения** (через `__eq__`), `is` — **идентичность** (тот же объект, сравнение `id()`). `id(x)` — уникальный идентификатор (в CPython — адрес в памяти). `hash(x)` — целое для хеш-таблиц; равные объекты обязаны иметь равный хеш, объект должен быть неизменяемым (хешируемым). Ловушка: для малых int (−5..256) и интернированных строк `is` может «случайно» вернуть `True` из-за кеширования — но полагаться на это нельзя. `None`, `True`, `False` — синглтоны, их корректно сравнивать через `is`.

**Q: Почему `isinstance(True, int)` возвращает `True`? Когда использовать `isinstance`, а когда `type() ==`?**
A: `bool` — подкласс `int` (`True == 1`, `False == 0`), поэтому `isinstance(True, int)` истинно. `isinstance` учитывает наследование и абстрактные базовые классы — это предпочтительный способ проверки типа (duck typing-дружественный). `type(x) == X` проверяет **точный** тип без учёта наследования — нужно редко, например когда подкласс должен обрабатываться иначе. Для проверки «число ли это» лучше `isinstance(x, numbers.Number)`.

**Q: Как работает `super()` при множественном наследовании?**
A: `super()` не «вызывает родителя», а идёт по **MRO** (Method Resolution Order), вычисляемому алгоритмом C3-линеаризации. В ромбовидном наследовании `D(B, C)` вызов `super().method()` из `B` может перейти в `C`, а не в общего предка. Это позволяет каждому классу в цепочке выполниться ровно один раз (кооперативное наследование). MRO смотрят через `Cls.__mro__` или `Cls.mro()`. В Python 3 `super()` без аргументов автоматически подставляет класс и `self`.

**Q: Чем опасны `eval` и `exec`? Чем заменить?**
A: Они исполняют произвольный код из строки. Если строка содержит недоверенный пользовательский ввод — это RCE-уязвимость: `eval("__import__('os').system('rm -rf /')")`. Ограничение `globals`/`locals` (`{"__builtins__": {}}`) затрудняет, но не гарантирует безопасность (известны обходы). Замены: `ast.literal_eval` для разбора литералов (списки, числа, строки, словари), `json.loads` для JSON, явные парсеры/диспетчеры для логики. `eval`/`exec` оправданы только для доверенного кода (REPL, шаблонизаторы, метапрограммирование).

**Q: Что вернёт `all([])` и `any([])`? Почему?**
A: `all([])` → `True`, `any([])` → `False`. Это «истинность по умолчанию» (vacuous truth): «все элементы истинны» для пустого множества верно, «хотя бы один истинен» — ложно. Обе функции используют **короткое замыкание**: `all` останавливается на первом ложном элементе, `any` — на первом истинном, поэтому эффективны на генераторах.

**Q: Что особенного в округлении `round()` в Python 3?**
A: Используется **банковское округление** (round half to even): `round(0.5) == 0`, `round(2.5) == 2`, `round(3.5) == 4` — половинки округляются к ближайшему чётному, чтобы уменьшить систематическую ошибку. Плюс проблема float: `round(2.675, 2)` даёт `2.67`, а не `2.68`, потому что `2.675` не представимо точно в двоичной плавающей точке. Для денег используйте `decimal.Decimal`.

**Q: Что делает `zip(*matrix)` и `iter` с двумя аргументами?**
A: `zip(*matrix)` транспонирует: распаковывает строки как отдельные аргументы и группирует их «по столбцам». `iter(callable, sentinel)` создаёт итератор, который вызывает `callable()` без аргументов до тех пор, пока он не вернёт `sentinel` — удобно для чтения потока блоками: `iter(lambda: f.read(1024), b'')`.

**Q: Зачем нужны `__repr__` и `__str__`, какой из них вызывает `repr`/`str`/`print`/f-строка?**
A: `repr(x)` → `__repr__` (однозначное, для отладки, в идеале валидный Python-код для воссоздания объекта). `str(x)`, `print(x)`, `f"{x}"` → `__str__` (читаемое); если `__str__` не определён — используется `__repr__`. В контейнерах (список, словарь) элементы всегда показываются через `repr`. Хорошая практика — всегда определять `__repr__`.

**Q: В чём разница между `staticmethod` и `classmethod`? Когда какой?**
A: `classmethod` получает первым аргументом класс (`cls`) — нужен для альтернативных конструкторов (`dict.fromkeys`, `datetime.now`) и фабричных методов, корректно работающих с подклассами. `staticmethod` не получает ни `self`, ни `cls` — это просто функция в пространстве имён класса, логически связанная с ним, но не использующая его состояние (например, валидатор-утилита). Если методу нужен класс — `classmethod`; если ни класс, ни экземпляр — `staticmethod`; иначе — обычный метод.

**Q: `property` — это что, как реализован?**
A: `property` — встроенный **дескриптор данных**: класс с методами `__get__`/`__set__`/`__delete__`. Он позволяет вызывать методы синтаксисом доступа к атрибуту (`obj.area` вместо `obj.area()`), добавлять валидацию в сеттер, делать вычисляемые/только-для-чтения поля без изменения внешнего API. Поскольку это дескриптор данных, он имеет приоритет над `__dict__` экземпляра.

## Подводные камни (gotchas)

**1. Затенение встроенных имён (shadowing).**
```python
list = [1, 2, 3]            # переопределили встроенный list
# list("abc")               # TypeError: 'list' object is not callable
# Часто страдают: list, dict, str, id, type, sum, max, min, input, file, hash
```

**2. Исчерпание ленивых итераторов.**
```python
data = map(int, ["1", "2", "3"])
print(sum(data))            # => 6
print(sum(data))            # => 0  (итератор уже пуст!)
# То же для zip, filter, enumerate, генераторов. range — исключение (переиспользуем).
```

**3. `zip` молча обрезает по короткому.**
```python
list(zip([1, 2, 3], ['a', 'b']))  # => [(1, 'a'), (2, 'b')] — третий элемент потерян!
# В Python 3.10+ используйте strict=True, чтобы поймать рассинхрон длин.
```

**4. `sum` для конкатенации строк.**
```python
# sum(["a", "b"], "")       # TypeError: запрещено намеренно — медленно (O(n²))
# Правильно: "".join(["a", "b"])
```

**5. `round` — банковское округление + неточность float.**
```python
round(0.5), round(1.5), round(2.5)  # => (0, 2, 2)  не (1, 2, 3)!
round(2.675, 2)                     # => 2.67 (а не 2.68) из-за двоичного float
```

**6. `bool`/`int` пересечение.**
```python
isinstance(True, int)       # => True
{1: "a", True: "b"}         # => {1: 'b'}  (True == 1 -> один ключ!)
[1, 2][True]                # => 2 (True как индекс 1)
```

**7. `dict.get`/`getattr` с default, который дорого вычислять.**
```python
# default вычисляется ВСЕГДА, даже если не нужен
# getattr(obj, "x", expensive())  # expensive() вызывается всегда
# Используйте hasattr + ветвление или try/except, если default дорогой
```

**8. Изменяемый аргумент по умолчанию и `id`.**
```python
def add(item, target=[]):   # КЛАССИЧЕСКАЯ ошибка
    target.append(item)
    return target
add(1); add(2)              # => [1, 2] — список общий между вызовами!
# Правильно: target=None, внутри target = target or []
```

**9. `reversed` не работает с генераторами и `map`/`filter`/`zip`.**
```python
# reversed(x for x in range(3))   # TypeError: нет __reversed__/__len__
# reversed(map(...))              # TypeError
# Нужно сначала материализовать: reversed(list(...))
```

**10. `locals()` внутри функции — снимок, его правка не меняет переменные.**
```python
def f():
    a = 1
    locals()["a"] = 99
    return a                # => 1 (а не 99!)
```

**11. `hash` строк рандомизирован между запусками.**
```python
# hash("x") даёт разные значения в разных процессах (защита от hash-DoS).
# Нельзя сохранять hash() в файл/БД и сравнивать между запусками.
# Отключается через PYTHONHASHSEED=0, но в проде так делать не стоит.
```

**12. `type(x) == X` ломает наследование.**
```python
class MyInt(int): pass
x = MyInt(5)
type(x) == int              # => False
isinstance(x, int)          # => True  (обычно нужно именно это)
```

## Лучшие практики

- **Не затеняйте встроенные имена.** Не называйте переменные `list`, `dict`, `id`, `type`, `sum`, `input`, `file`, `str`. Линтеры (`flake8`, `ruff` — правило A001) ловят это.
- **Предпочитайте comprehensions простым `map`/`filter` с `lambda`.** `[f(x) for x in xs]` читабельнее `list(map(f, xs))`. `map`/`filter` хороши с уже существующей функцией (`map(int, parts)`).
- **Материализуйте ленивые объекты осознанно.** Если итератор нужен дважды — сохраните `list(...)`. Если данные большие/бесконечные — держите ленивость и используйте `itertools`.
- **Всегда определяйте `__repr__`** для своих классов (отладка, логи). `__str__` — по необходимости.
- **Для проверки типов используйте `isinstance` (с кортежем типов или ABC),** а не `type(x) ==`. Но не злоупотребляйте — duck typing/протоколы часто лучше.
- **Никогда не передавайте недоверенный ввод в `eval`/`exec`.** Используйте `ast.literal_eval`, `json.loads` или явные парсеры.
- **Используйте `enumerate` вместо `range(len(x))`** для индексов, и `zip` вместо параллельной индексации.
- **Файлы открывайте через `with open(..., encoding="utf-8")`** — гарантированное закрытие и явная кодировка.
- **Для денег используйте `decimal.Decimal`,** не `round` на `float`.
- **`sorted(..., key=...)`** — предпочитайте `key` вместо устаревшего `cmp`; для сложных ключей возвращайте кортеж.
- **`super()` без аргументов** в Python 3 — стандарт; проектируйте классы для кооперативного множественного наследования.

## Шпаргалка

**Ленивые (итераторы/последовательности):** `range` (переиспользуемая последовательность), `map`, `filter`, `zip`, `enumerate`, `reversed` — одноразовые итераторы (кроме `range`).

**Полагаются на дандеры:** `len`→`__len__`, `iter`→`__iter__`, `next`→`__next__`, `repr`→`__repr__`, `str`→`__str__`, `abs`→`__abs__`, `hash`→`__hash__`, `bool`→`__bool__`/`__len__`, `format`→`__format__`, `reversed`→`__reversed__`, `callable`→наличие `__call__`.

**На самом деле классы (конструкторы):** `int`, `str`, `list`, `dict`, `set`, `tuple`, `bool`, `type`, `range`, `enumerate`, `zip`, `map`, `filter`, `reversed`, `slice`, `property`, `staticmethod`, `classmethod`, `super`.

**Самоанализ:** `type`, `isinstance(x, (A,B))`, `issubclass`, `dir`, `vars`==`x.__dict__`, `getattr/setattr/hasattr/delattr`, `callable`, `globals/locals`.

**Идентичность/значение:** `is`/`id` — идентичность, `==` — значение, `hash` — для хеш-таблиц (только неизменяемые).

**Системы счисления:** `bin`/`oct`/`hex` (→строка с префиксом), `int(s, base)` (обратно), `ord`/`chr` (символ↔код).

**Математика:** `abs`, `round` (банковское!), `divmod(a,b)`→`(a//b, a%b)`, `pow(a,b,m)` (модульное), `sum(it, start)`, `min/max(..., key=, default=)`.

**Опасно:** `eval`/`exec` (RCE) → используйте `ast.literal_eval`. `input` всегда возвращает `str`. `open` — только с `with` и `encoding`.

**Логика:** `any([])`→`False`, `all([])`→`True`, обе с коротким замыканием.

**Ключевые ловушки:** `list.sort()` возвращает `None`; ленивые итераторы исчерпываются; `zip` обрезает по короткому; `isinstance(True, int)`→`True`; `round(2.5)`→`2`; не затеняйте встроенные имена.
