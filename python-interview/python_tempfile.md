# tempfile — подготовка к собеседованию

## Что это и зачем

`tempfile` — модуль стандартной библиотеки для **безопасного создания временных файлов и директорий**. Главная ценность — безопасность: он генерирует непредсказуемые уникальные имена, открывает файлы с правильными флагами (исключая гонки и атаки симлинков) и предоставляет инструменты для автоматической очистки.

Зачем не делать «вручную»:
- Самописное `open("/tmp/myfile_" + str(random.random()))` уязвимо к **race conditions** и **symlink-атакам** (TOCTOU), коллизиям имён, утечкам файлов.
- `tempfile` использует системный безопасный механизм создания (`O_CREAT | O_EXCL`), правильную директорию (`$TMPDIR`/`/tmp`/`%TEMP%`), ограничительные права (0600).

```python
import tempfile

# Временный файл, который сам удалится при закрытии
with tempfile.TemporaryFile(mode="w+") as f:
    f.write("промежуточные данные")
    f.seek(0)
    print(f.read())
# файл уже удалён

# Временная директория с авто-очисткой
with tempfile.TemporaryDirectory() as tmp:
    print("Рабочая папка:", tmp)
# папка и всё содержимое удалены
```

## Ключевые концепции

- **Высокоуровневые vs низкоуровневые API.** Высокоуровневые (`TemporaryFile`, `NamedTemporaryFile`, `TemporaryDirectory`, `SpooledTemporaryFile`) — контекстные менеджеры с авто-очисткой. Низкоуровневые (`mkstemp`, `mkdtemp`) — возвращают «сырые» дескрипторы/пути, очистку делаете вы.
- **Безопасность.** Уникальные непредсказуемые имена, флаг `O_EXCL` (атомарное создание, исключающее перезапись), права 0600 (только владелец).
- **Автоудаление.** Большинство объектов удаляют файл/папку при закрытии или выходе из `with`.
- **Расположение и настройка.** Каталог берётся из `tempfile.gettempdir()` (зависит от `TMPDIR`, `TEMP`, `TMP` или дефолта ОС). Можно задать `dir=`, `prefix=`, `suffix=`.

## Основные функции/классы/методы

### TemporaryFile — безымянный временный файл

```python
import tempfile

# Файл без имени в ФС (на Unix часто сразу unlink-нут). Удаляется при закрытии.
with tempfile.TemporaryFile(mode="w+b") as f:
    f.write(b"binary data")
    f.seek(0)
    data = f.read()
# Преимущество: нельзя случайно оставить файл; недостаток: нет пути для других процессов
```

`TemporaryFile` оптимален, когда файл нужен **только текущему процессу** и не нужно его имя.

### NamedTemporaryFile — временный файл С именем

```python
import tempfile

# Имеет реальное имя в ФС (атрибут .name) — можно передать другому процессу/функции
with tempfile.NamedTemporaryFile(
    mode="w+",
    suffix=".json",       # расширение
    prefix="export_",     # префикс имени
    dir="/tmp",           # каталог
    delete=True,          # удалить при закрытии (по умолчанию True)
    encoding="utf-8",
) as f:
    print("Путь:", f.name)
    f.write('{"key": "value"}')
    f.flush()             # важно: данные могут быть в буфере
    external_tool(f.name) # передаём путь другому процессу
# файл удалён при выходе

# delete_on_close (Python 3.12+): не удалять при close(), только при выходе из with
with tempfile.NamedTemporaryFile(delete=True, delete_on_close=False) as f:
    f.write(b"...")
    f.close()             # файл ещё существует
    use_path(f.name)      # можно открыть другим процессом (важно на Windows!)
# здесь удалится
```

Нюанс Windows: открытый `NamedTemporaryFile` нельзя одновременно открыть другим процессом. До 3.12 решали через `delete=False` + ручное удаление; в 3.12+ — через `delete_on_close=False`.

### TemporaryDirectory — временная директория

```python
import tempfile, os
from pathlib import Path

with tempfile.TemporaryDirectory(prefix="build_") as tmp:
    workdir = Path(tmp)
    (workdir / "input.txt").write_text("данные", encoding="utf-8")
    (workdir / "sub").mkdir()
    # ... работаем с целой папкой ...
# вся директория со всем содержимым рекурсивно удалена (как rmtree)

# С 3.12 есть параметры ignore_cleanup_errors=True и delete=True/False
```

### SpooledTemporaryFile — файл «в памяти, пока мал»

```python
import tempfile

# Держит данные в памяти (BytesIO/StringIO), пока размер < max_size,
# затем автоматически «сбрасывает» на диск. Экономит I/O для мелких данных.
with tempfile.SpooledTemporaryFile(max_size=1024 * 1024, mode="w+b") as f:
    f.write(b"small")          # пока в памяти
    f.write(b"x" * 2_000_000)  # превысили max_size -> уехало на диск
    f.rollover()               # можно принудительно сбросить на диск
```

### Низкоуровневые: mkstemp и mkdtemp

```python
import tempfile, os

# mkstemp -> (fd, path). Возвращает ОТКРЫТЫЙ файловый дескриптор и путь.
# НЕ удаляется автоматически — вы отвечаете за закрытие fd и удаление файла!
fd, path = tempfile.mkstemp(suffix=".log", prefix="job_", dir="/tmp", text=True)
try:
    with os.fdopen(fd, "w") as f:   # оборачиваем сырой fd в файловый объект
        f.write("log line\n")
    # path остаётся на диске, можно передать другим
finally:
    os.unlink(path)                  # ОБЯЗАТЕЛЬНО удалить вручную

# mkdtemp -> path. Создаёт директорию, очистку делаете вы (shutil.rmtree).
dirpath = tempfile.mkdtemp(prefix="cache_")
try:
    ...
finally:
    import shutil
    shutil.rmtree(dirpath)
```

`mkstemp`/`mkdtemp` дают **максимальный контроль** (например, файл должен пережить процесс или передаваться между процессами), но требуют ручной очистки.

### Настройка местоположения

```python
import tempfile

print(tempfile.gettempdir())       # текущий каталог для временных файлов
print(tempfile.gettempprefix())    # префикс по умолчанию ('tmp')

# Глобально переопределить (влияет на весь модуль)
tempfile.tempdir = "/fast/ssd/tmp"

# tempfile выбирает каталог из переменных окружения: TMPDIR, TEMP, TMP, затем дефолт ОС
```

## Частые вопросы на собеседовании

**Q1: Почему `tempfile` безопаснее, чем самописное создание временного файла?**
Он генерирует криптографически непредсказуемые уникальные имена, создаёт файл атомарно с `O_CREAT | O_EXCL` (нельзя перезаписать существующий → защита от symlink/TOCTOU-атак), ставит права 0600 (доступ только владельцу) и кладёт в правильный безопасный каталог.

**Q2: В чём разница между `TemporaryFile` и `NamedTemporaryFile`?**
`TemporaryFile` не имеет гарантированного имени в ФС (на Unix часто сразу удаляется из директории) — годится только для текущего процесса. `NamedTemporaryFile` имеет видимое имя (`.name`), которое можно передать другому процессу/программе.

**Q3: Когда использовать `mkstemp` вместо `NamedTemporaryFile`?**
Когда нужен полный контроль над жизненным циклом: файл должен пережить процесс, или вы сами управляете моментом удаления, или нужен сырой fd. `mkstemp` не делает авто-очистку — это и плюс (контроль), и минус (риск утечки).

**Q4: В чём проблема `NamedTemporaryFile` на Windows?**
На Windows файл, открытый одним процессом, нельзя одновременно открыть другим. Поэтому передать `f.name` внешней программе, пока файл открыт, не получится. Решение: `delete=False` + ручное удаление, либо `delete_on_close=False` (Python 3.12+).

**Q5: Что такое `SpooledTemporaryFile` и зачем он?**
Файл, который держит данные в памяти, пока их размер меньше `max_size`, и только при превышении сбрасывает на диск. Это оптимизация: для мелких данных не трогаем диск вообще, для крупных — не держим всё в RAM.

**Q6: Как `tempfile` решает, в какой каталог писать?**
Проверяет переменные окружения `TMPDIR`, `TEMP`, `TMP` (в этом порядке), затем системные дефолты (`/tmp` на Unix, `%TEMP%` на Windows). Доступно через `tempfile.gettempdir()`. Можно переопределить параметром `dir=` или глобально `tempfile.tempdir`.

**Q7: Удаляется ли файл `mkstemp` автоматически?**
Нет. `mkstemp` и `mkdtemp` — низкоуровневые, очистку выполняете вы (`os.unlink` / `shutil.rmtree`), обычно в `try/finally`.

**Q8: Зачем `flush()` перед передачей `NamedTemporaryFile.name` другой программе?**
Данные могут оставаться в буфере Python/ОС и ещё не быть записаны на диск. Внешний процесс увидит пустой/неполный файл. `flush()` (а иногда `os.fsync`) гарантирует, что данные на диске.

**Q9: Какие права у создаваемых временных файлов?**
0600 (чтение/запись только для владельца) для файлов и 0700 для директорий — чтобы другие пользователи системы не могли прочитать чувствительные данные.

**Q10: Как обеспечить очистку временной директории даже при исключении?**
Использовать `TemporaryDirectory` как контекстный менеджер (`with`) — очистка произойдёт при выходе из блока в любом случае. Для `mkdtemp` — обернуть в `try/finally` с `shutil.rmtree`.

**Q11: Что делает `delete_on_close` (3.12+)?**
Разделяет «закрытие файла» и «удаление файла». При `delete=True, delete_on_close=False` файл не удаляется при `close()`, а только при выходе из `with`/сборке объекта — что позволяет другим процессам открыть файл по имени после закрытия (важно для Windows).

## Подводные камни (gotchas)

1. **`mkstemp`/`mkdtemp` не самоочищаются** — утечка временных файлов при забытом `unlink`/`rmtree`.

2. **`NamedTemporaryFile` на Windows** нельзя открыть другим процессом, пока он открыт. Используйте `delete_on_close=False` (3.12+) или `delete=False`.

3. **Забытый `flush()`/`fsync()`** перед чтением файла внешним процессом → он видит пустой/частичный файл.

4. **`TemporaryFile` без имени** нельзя передать в другой процесс — для этого нужен `NamedTemporaryFile`.

5. **Утечка дескриптора из `mkstemp`.** `mkstemp` возвращает открытый fd; если обернуть путь в новый `open(path)`, вы получите ДВА дескриптора — забытый исходный fd протечёт. Оборачивайте через `os.fdopen(fd)`.

6. **Глобальный `tempfile.tempdir`** влияет на весь процесс — не меняйте его в библиотеках.

7. **Кодировка.** Для текстового режима указывайте `encoding="utf-8"`, иначе зависит от локали.

8. **`delete=True` и переоткрытие.** Если открыть `NamedTemporaryFile(delete=True)` и закрыть, файл исчезнет — нельзя потом открыть по имени (до 3.12 без `delete=False`).

9. **Чувствительные данные на диске.** Даже после удаления данные могут оставаться на носителе физически — для секретов предпочитайте `TemporaryFile`/in-memory и не пишите их в `/tmp`.

10. **Заполнение `/tmp`.** Большие временные файлы могут забить раздел `/tmp` (часто tmpfs в RAM). Указывайте `dir=` на нужный раздел.

## Лучшие практики

- По умолчанию используйте высокоуровневые контекстные менеджеры (`with TemporaryFile()/TemporaryDirectory()`) ради авто-очистки.
- Для `mkstemp`/`mkdtemp` всегда оборачивайте очистку в `try/finally`.
- Сырой fd из `mkstemp` оборачивайте через `os.fdopen(fd)`, не открывайте путь заново.
- Перед передачей имени файла внешнему процессу делайте `f.flush()` (и при необходимости `os.fsync(f.fileno())`).
- На Windows для межпроцессного доступа используйте `delete_on_close=False` (3.12+).
- Не храните долгоживущие секреты в `/tmp`; для секретов — in-memory `TemporaryFile`.
- Задавайте осмысленные `prefix`/`suffix` для отладки и `dir=` под нужный раздел.
- Никогда не конструируйте временные имена вручную через `random`/`pid` — это небезопасно.

## Шпаргалка

```python
import tempfile, os

# Высокоуровневые (авто-очистка через with)
with tempfile.TemporaryFile(mode="w+b") as f: ...          # безымянный
with tempfile.NamedTemporaryFile(suffix=".txt",
        delete=True, delete_on_close=False) as f:           # с именем (.name)
    f.write(b"..."); f.flush()
with tempfile.TemporaryDirectory(prefix="x_") as d: ...     # директория
with tempfile.SpooledTemporaryFile(max_size=1<<20) as f: ...# память -> диск

# Низкоуровневые (ручная очистка!)
fd, path = tempfile.mkstemp(suffix=".log", dir="/tmp")
try:
    with os.fdopen(fd, "w") as f: f.write("...")
finally:
    os.unlink(path)

dirpath = tempfile.mkdtemp(prefix="cache_")
# ... shutil.rmtree(dirpath) в finally

# Конфигурация
tempfile.gettempdir()        # каталог по умолчанию
tempfile.gettempprefix()     # 'tmp'
# параметры всех: suffix=, prefix=, dir=
# безопасность: уникальные имена, O_EXCL, права 0600
```
