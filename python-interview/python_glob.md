# glob и fnmatch — подготовка к собеседованию

## Что это и зачем

`glob` и `fnmatch` — два связанных модуля стандартной библиотеки для **сопоставления имён файлов с шаблонами в стиле Unix shell** (wildcards).

- **`glob`** — ищет файлы на диске по шаблону (`*.py`, `data/**/*.csv`). Обращается к файловой системе.
- **`fnmatch`** — сопоставляет произвольные **строки** с shell-шаблоном, **не** обращаясь к ФС. Это «движок» сопоставления, на котором по сути стоит `glob`.

Зачем нужны:
- `glob` — быстро получить список путей по маске, без ручного обхода директорий.
- `fnmatch` — фильтровать любые имена (например, список из БД или результат `os.listdir`) по шаблону.

```python
import glob, fnmatch

# glob: ищем на диске
py_files = glob.glob("src/*.py")              # ['src/a.py', 'src/b.py']
all_csv = glob.glob("data/**/*.csv", recursive=True)

# fnmatch: фильтруем строки (не трогаем диск)
names = ["report.txt", "data.csv", "image.png"]
texts = [n for n in names if fnmatch.fnmatch(n, "*.txt")]
```

## Ключевые концепции

### Шаблоны (wildcards)

| Шаблон | Значение |
|--------|----------|
| `*`    | любое число символов (в glob — **не** пересекает `/`) |
| `?`    | ровно один любой символ |
| `[abc]`| один символ из набора |
| `[a-z]`| один символ из диапазона |
| `[!abc]` / `[^abc]` | один символ НЕ из набора |
| `**`   | (только glob с `recursive=True`) любое число вложенных директорий |

### glob vs fnmatch

- `fnmatch` — чистое сопоставление строки и шаблона, без диска. `*` в нём матчит **любые** символы, включая `/`.
- `glob` — поиск на диске; `*` НЕ пересекает разделитель каталогов, рекурсия только через `**` + `recursive=True`. Скрытые файлы (с точки) по умолчанию не попадают под `*`.

### Скрытые файлы (dotfiles)

В `glob` (как в shell) `*` **не** матчит файлы, начинающиеся с точки (`.gitignore`, `.env`). Чтобы их захватить, нужен явный шаблон `.*` или `glob.glob(..., include_hidden=True)` (Python 3.11+).

## Основные функции/классы/методы

### glob.glob и glob.iglob

```python
import glob

# glob.glob(pattern) -> СПИСОК путей (порядок произвольный, НЕ отсортирован!)
glob.glob("*.txt")                       # текстовые файлы в текущей папке
glob.glob("/var/log/*.log")             # по абсолютному пути
glob.glob("img_???.png")                # ровно 3 символа после img_
glob.glob("file[0-9].dat")              # file0.dat..file9.dat

# Рекурсивный поиск: ** работает ТОЛЬКО с recursive=True
glob.glob("**/*.py", recursive=True)     # все .py в дереве (включая текущую папку)
glob.glob("src/**", recursive=True)      # всё дерево src (файлы и папки)

# iglob — то же, но ленивый ИТЕРАТОР (экономит память на больших деревьях)
for path in glob.iglob("logs/**/*.log", recursive=True):
    process(path)

# include_hidden (Python 3.11+) — включить файлы с точки под '*'
glob.glob("*", include_hidden=True)

# root_dir (Python 3.10+) — искать относительно каталога, возвращая относительные пути
glob.glob("*.py", root_dir="/home/user/project")
```

### glob.escape

```python
import glob

# Если в имени есть спецсимволы шаблона ([], *, ?), их нужно экранировать
literal_dir = "weird[name]"
pattern = glob.escape(literal_dir) + "/*.txt"   # ищем буквально в папке "weird[name]"
glob.glob(pattern)
```

### fnmatch

```python
import fnmatch

# fnmatch — регистронезависим/чувствителен в зависимости от ОС (нормализует регистр на Windows!)
fnmatch.fnmatch("Report.TXT", "*.txt")     # True на Windows, False на Linux

# fnmatchcase — ВСЕГДА учитывает регистр, независимо от ОС
fnmatch.fnmatchcase("Report.TXT", "*.txt") # False везде
fnmatch.fnmatchcase("report.txt", "*.txt") # True

# filter — отфильтровать список имён по шаблону (быстрее, чем цикл с fnmatch)
names = ["a.py", "b.txt", "c.py"]
fnmatch.filter(names, "*.py")              # ['a.py', 'c.py']

# translate — преобразовать shell-шаблон в РЕГУЛЯРНОЕ выражение
import re
regex = fnmatch.translate("*.py")          # '(?s:.*\\.py)\\Z'
pattern = re.compile(regex)
pattern.match("test.py")                   # для повторного использования компилируем
```

### Сравнение с pathlib.glob

```python
from pathlib import Path
import glob

# pathlib: метод на объекте Path, возвращает ГЕНЕРАТОР объектов Path
list(Path("src").glob("*.py"))             # [PosixPath('src/a.py'), ...]
list(Path("src").rglob("*.py"))            # рекурсивно (= glob('**/*.py'))
list(Path("src").glob("**/*.py"))          # ** без отдельного флага recursive!

# glob-модуль: функция, возвращает СПИСОК строк
glob.glob("src/*.py")                      # ['src/a.py', ...]
glob.glob("src/**/*.py", recursive=True)   # нужен recursive=True
```

Ключевые отличия `pathlib.Path.glob` от `glob.glob`:
- pathlib возвращает объекты `Path`, glob — строки.
- pathlib возвращает генератор, `glob.glob` — список (есть `iglob` для ленивости).
- В pathlib `**` рекурсивен **по умолчанию** (без флага), в glob нужен `recursive=True`.
- pathlib работает относительно объекта `Path`; glob — относительно cwd или абсолютно.
- В pathlib `**` не пересекает в отдельных версиях скрытые папки иначе; обе системы по умолчанию игнорируют dotfiles под `*`.

## Частые вопросы на собеседовании

**Q1: В чём разница между `glob` и `fnmatch`?**
`glob` ищет реальные файлы на диске по шаблону (обращается к ФС). `fnmatch` сопоставляет произвольную строку с шаблоном, не трогая диск. `glob` использует `fnmatch` под капотом для сопоставления имён.

**Q2: Как сделать рекурсивный поиск в модуле `glob`?**
Шаблон `**` плюс обязательный аргумент `recursive=True`: `glob.glob("**/*.py", recursive=True)`. Без `recursive=True` символы `**` ведут себя как одиночный `*`.

**Q3: Чем `glob.glob` отличается от `pathlib.Path.glob`?**
`glob.glob` возвращает список строк и требует `recursive=True` для `**`. `Path.glob` возвращает генератор объектов `Path`, и `**` в нём рекурсивен без дополнительного флага. У `Path` ещё есть `rglob` как сокращение для `**/`.

**Q4: Матчит ли `*` скрытые файлы (с точки)?**
Нет, по умолчанию `*` в glob не захватывает файлы, начинающиеся с `.` (как в Unix shell). Нужен явный `.*` или `include_hidden=True` (Python 3.11+).

**Q5: Пересекает ли `*` границы директорий?**
В `glob` — нет: `*` матчит в пределах одного уровня и не включает `/`. Для рекурсии нужен `**`. В `fnmatch` — да: там `*` матчит любые символы, включая `/`.

**Q6: Возвращает ли `glob.glob` отсортированный список?**
Нет, порядок зависит от ФС и произволен. Если нужен детерминированный порядок — оберните в `sorted(...)`.

**Q7: Чем `fnmatch` отличается от `fnmatchcase`?**
`fnmatch` нормализует регистр согласно ОС (на Windows — регистронезависим, на Linux — зависим). `fnmatchcase` всегда учитывает регистр на любой ОС. Для предсказуемости часто берут `fnmatchcase`.

**Q8: Как искать файл, в имени которого есть `[` или `*`?**
Экранировать спецсимволы через `glob.escape(...)`, иначе они будут истолкованы как шаблон.

**Q9: Что делает `fnmatch.translate`?**
Преобразует shell-шаблон в строку регулярного выражения. Полезно, когда нужно скомпилировать шаблон один раз через `re.compile` и применять многократно для производительности.

**Q10: Как ограничить glob поиском в конкретной папке без склейки путей?**
С Python 3.10 параметр `root_dir`: `glob.glob("*.txt", root_dir="/data")` — ищет в `/data`, возвращая относительные пути.

**Q11: Чем `iglob` лучше `glob`?**
`iglob` возвращает итератор и не материализует весь список в памяти — это важно для огромных деревьев. `glob.glob` собирает всё в список сразу.

## Подводные камни (gotchas)

1. **`**` без `recursive=True` не работает** в модуле `glob` — ведёт себя как `*`. Частая ошибка.

2. **`glob.glob` не сортирует результат.** Не полагайтесь на порядок; используйте `sorted()` для воспроизводимости (например, в тестах).

3. **Dotfiles игнорируются `*`.** `glob.glob("*")` не вернёт `.env`, `.git`. Нужен `.*` или `include_hidden=True`.

4. **`fnmatch` зависит от регистра ОС.** На Windows `fnmatch("A.TXT", "*.txt")` → True, на Linux → False. Это неочевидно; используйте `fnmatchcase`, если нужна одинаковость.

5. **Спецсимволы в реальных именах.** Если в пути есть `[`, `]`, `*`, `?` — без `glob.escape` поиск даст неожиданный результат или ничего.

6. **`*` в `fnmatch` пересекает `/`, а в `glob` — нет.** Один и тот же шаблон ведёт себя по-разному в двух модулях.

7. **Рекурсивный `**` может быть очень медленным** на больших деревьях и не следит за петлями симлинков особо умно — может зациклиться/долго работать.

8. **`glob` следует за симлинками** при рекурсии (в зависимости от версии), что может приводить к дублям или зацикливанию.

9. **Относительность к cwd.** `glob.glob("*.py")` зависит от текущего рабочего каталога — меняется при `os.chdir`. Лучше абсолютные пути или `root_dir`.

10. **`fnmatch.translate` даёт regex, не `re.Pattern`.** Нужно ещё `re.compile`. И результат содержит `\Z`/`(?s:...)`, не «голый» шаблон.

## Лучшие практики

- Для рекурсии всегда явно указывайте `recursive=True` и используйте `**`.
- Оборачивайте результат `glob.glob` в `sorted()`, если важен детерминированный порядок.
- Для огромных деревьев используйте `iglob` (или `pathlib` генераторы) вместо `glob`.
- Экранируйте пользовательские части пути через `glob.escape`.
- Для регистронезависимости/независимости явно выбирайте `fnmatch` или `fnmatchcase`.
- Если шаблон применяется много раз к разным строкам — компилируйте через `fnmatch.translate` + `re.compile`.
- В новом коде для путей предпочитайте `pathlib.Path.glob`/`rglob` (объекты, генераторы, рекурсия без флага).
- Используйте абсолютные пути или `root_dir`, чтобы не зависеть от cwd.

## Шпаргалка

```python
import glob, fnmatch, re

# glob (диск -> список строк)
glob.glob("*.py")                          # текущая папка
glob.glob("dir/*.txt")                     # подпапка
glob.glob("**/*.py", recursive=True)       # РЕКУРСИВНО (нужен recursive=True!)
glob.iglob("**/*.log", recursive=True)     # ленивый итератор
glob.glob("*", include_hidden=True)        # включая dotfiles (3.11+)
glob.glob("*.py", root_dir="/proj")        # относительно каталога (3.10+)
glob.escape("name[1]")                     # экранировать спецсимволы

# fnmatch (строки, без диска)
fnmatch.fnmatch("a.TXT", "*.txt")          # регистр по ОС
fnmatch.fnmatchcase("a.txt", "*.txt")      # всегда учитывает регистр
fnmatch.filter(names, "*.py")              # фильтр списка
re.compile(fnmatch.translate("*.py"))      # шаблон -> regex для повторов

# pathlib (объекты Path, генератор)
from pathlib import Path
Path("src").glob("*.py")                   # один уровень
Path("src").rglob("*.py")                  # рекурсивно (= glob('**/*.py'))
# ** в pathlib рекурсивен БЕЗ recursive=True

# Шаблоны: *  ?  [abc]  [a-z]  [!abc]  ** (рекурсия)
```
