# random — подготовка к собеседованию

## Что это и зачем

`random` — модуль стандартной библиотеки для генерации **псевдослучайных** чисел. Под капотом — алгоритм **Mersenne Twister (MT19937)**, который детерминирован: при одинаковом начальном состоянии (seed) выдаёт одинаковую последовательность.

Зачем нужен:
- Случайные числа, выборки, перемешивание для симуляций, тестов, игр, генерации данных.
- **Воспроизводимость** через `seed()` — критично для тестов и научных экспериментов.
- Различные распределения: равномерное, нормальное (Гаусса), экспоненциальное и др.

Важно: **`random` НЕ криптографически стойкий**. Для паролей, токенов, ключей нужен `secrets` или `random.SystemRandom`.

## Ключевые концепции

- **Псевдослучайность.** Генератор детерминирован: одно и то же `seed` → одна и та же последовательность. Это не недостаток, а основа воспроизводимости.
- **Глобальный генератор.** Функции модуля (`random.random()`, `random.randint()`) используют один общий экземпляр `random.Random`. Можно создать независимые экземпляры `random.Random(seed)`.
- **MT19937.** Период 2^19937−1, отличное статистическое качество, но **предсказуем**: по достаточному числу выходов можно восстановить внутреннее состояние.
- **SystemRandom.** Использует ОС-источник энтропии (`os.urandom`), непредсказуем, но не поддерживает `seed`/воспроизводимость.
- **Полуинтервалы.** `random()` ∈ [0, 1), `randrange(n)` ∈ [0, n), но `randint(a, b)` ∈ [a, b] (включительно!).

## Основные функции/классы/методы

### seed и воспроизводимость

```python
import random

random.seed(42)
print(random.random())   # 0.6394267984578837 (всегда то же при seed=42)
print(random.randint(1, 100))  # 81

random.seed(42)          # сброс к тому же состоянию
print(random.random())   # 0.6394267984578837 — ВОСПРОИЗВЕЛИ

# Сохранить/восстановить состояние без seed
state = random.getstate()
x1 = random.random()
random.setstate(state)
x2 = random.random()
print(x1 == x2)          # True — то же состояние
```

### Базовые числа

```python
import random
random.seed(0)

print(random.random())          # float ∈ [0.0, 1.0)
print(random.uniform(1.5, 6.5)) # float ∈ [1.5, 6.5] равномерно
print(random.randint(1, 6))     # int ∈ [1, 6] ВКЛЮЧИТЕЛЬНО (как бросок кубика)
print(random.randrange(10))     # int ∈ [0, 10) — НЕ включая 10
print(random.randrange(0, 10, 2))  # чётное из 0,2,4,6,8
print(random.getrandbits(8))    # int из 8 случайных бит ∈ [0, 255]
```

### Выбор из последовательности

```python
import random
random.seed(1)

colors = ['red', 'green', 'blue', 'yellow']

print(random.choice(colors))        # один элемент

# sample — выборка БЕЗ повторений (k уникальных)
print(random.sample(colors, 2))     # ['blue', 'red'] — 2 разных

# choices — выборка С повторениями (можно с весами)
print(random.choices(colors, k=3))  # ['green', 'green', 'red'] — могут повторяться
print(random.choices(colors, weights=[10, 1, 1, 1], k=5))  # red чаще (вес 10)

# shuffle — перемешать список НА МЕСТЕ (мутирует!)
deck = list(range(5))
random.shuffle(deck)
print(deck)   # например [2, 0, 4, 1, 3]
```

### Распределения

```python
import random
random.seed(7)

# Нормальное (Гаусса): mu — среднее, sigma — стандартное отклонение
print(random.gauss(0, 1))        # ~ N(0, 1)
print(random.normalvariate(100, 15))  # IQ-подобное распределение

# Экспоненциальное: lambd = 1/mean
print(random.expovariate(0.5))   # среднее = 2

# Прочие
print(random.triangular(0, 10, 2))    # треугольное (low, high, mode)
print(random.betavariate(2, 5))       # бета-распределение
print(random.lognormvariate(0, 1))    # логнормальное
print(random.paretovariate(2))        # Парето
print(random.vonmisesvariate(0, 4))   # круговое
```

`gauss` быстрее `normalvariate`, но `gauss` не потокобезопасен (использует кэшированное значение между вызовами).

### SystemRandom — на источнике энтропии ОС

```python
import random

sysrand = random.SystemRandom()   # использует os.urandom
print(sysrand.randint(1, 100))    # непредсказуемо
print(sysrand.random())

# seed/getstate/setstate НЕ работают (нет внутреннего состояния)
sysrand.seed(42)   # игнорируется, не даёт воспроизводимости
# Подходит, когда нужна непредсказуемость без криптотребований
```

### Независимые генераторы (важно для многопоточности/тестов)

```python
import random

# Каждый экземпляр — независимая последовательность
rng1 = random.Random(42)
rng2 = random.Random(42)
print(rng1.random() == rng2.random())  # True — одинаковый seed

# Не влияет на глобальный генератор и наоборот
rng_a = random.Random(1)
rng_b = random.Random(2)
print(rng_a.randint(1, 100), rng_b.randint(1, 100))  # независимы
```

## Частые вопросы на собеседовании

**Q1: Почему `random` нельзя использовать для криптографии?**
A: Mersenne Twister предсказуем: он не криптостойкий, и по сравнительно небольшому числу выходов (624 значения) можно восстановить всё внутреннее состояние и предсказать будущие. Для токенов, паролей, ключей используйте модуль `secrets` или `random.SystemRandom` (на `os.urandom`).

**Q2: Чем `randint(a, b)` отличается от `randrange(a, b)`?**
A: `randint(a, b)` включает оба конца: ∈ [a, b]. `randrange(a, b)` работает как `range`: ∈ [a, b), правый конец не включён. `randint(1, 6)` — это `randrange(1, 7)`.

**Q3: В чём разница между `sample` и `choices`?**
A: `sample(seq, k)` — выборка **без повторений** (k уникальных элементов, k ≤ len). `choices(seq, k=k)` — выборка **с повторениями** (элементы могут дублироваться) и поддерживает веса. `sample` — для лотереи без возврата, `choices` — для бросков с возвратом.

**Q4: Как обеспечить воспроизводимость случайных чисел?**
A: Зафиксировать `random.seed(N)` перед генерацией. При одинаковом seed последовательность идентична. Альтернатива — сохранять/восстанавливать состояние через `getstate()`/`setstate()`. Для изоляции используйте отдельный `random.Random(seed)`.

**Q5: `random.shuffle` возвращает новый список?**
A: Нет, он перемешивает список **на месте** (мутирует аргумент) и возвращает `None`. Чтобы не менять оригинал, используйте `random.sample(lst, len(lst))`, который вернёт новый перемешанный список.

**Q6: Что произойдёт при `random.sample(seq, k)`, если k > len(seq)?**
A: Бросается `ValueError`, так как нельзя выбрать больше уникальных элементов, чем есть. Для повторений используйте `choices`.

**Q7: Безопасен ли глобальный генератор `random` в многопоточности?**
A: Сам Mersenne Twister защищён GIL для отдельных операций, но общий генератор разделяется между потоками, что мешает воспроизводимости и может вызывать гонки в `gauss` (кэш). Для изоляции дайте каждому потоку свой `random.Random()`.

**Q8: Чем `SystemRandom` отличается от обычного `random`?**
A: `SystemRandom` берёт энтропию из `os.urandom` (ОС), он непредсказуем и подходит там, где предсказуемость нежелательна. Но он не поддерживает `seed`/`getstate`, значит не воспроизводим, и медленнее.

**Q9: Как сделать взвешенный случайный выбор?**
A: `random.choices(population, weights=[...], k=n)` — веса задают относительную вероятность. Есть также `cum_weights` (кумулятивные веса) для эффективности.

**Q10: `random.gauss` или `random.normalvariate` — что выбрать?**
A: `gauss(mu, sigma)` быстрее (использует один вызов на пару значений с кэшем), но не потокобезопасен. `normalvariate` чуть медленнее, но безопасен в многопоточности. В одном потоке — `gauss`.

**Q11: Включает ли `random.random()` значение 1.0?**
A: Нет, диапазон [0.0, 1.0) — полуоткрытый, `1.0` исключён, `0.0` включён.

## Подводные камни (gotchas)

- **НЕ для криптографии (главное!).** Предсказуем. Пароли/токены → `secrets`.

- **`randint` включает правый конец**, в отличие от `range`/`randrange`. Частый off-by-one баг.

- **`shuffle` мутирует и возвращает `None`.** `lst = random.shuffle(lst)` обнулит переменную. 

- **Точность float в распределениях.** Результаты `random()`, `uniform()`, `gauss()` — это `float` со всеми ограничениями IEEE 754. Не сравнивайте их через `==`, накопленные суммы могут не сойтись точно. Для денег/точных долей `random` не подходит.

- **Глобальный seed влияет на весь процесс.** Установка `random.seed()` в библиотеке может неожиданно сломать воспроизводимость другого кода. Используйте изолированный `Random()`.

- **`sample` с k > len → ValueError**, а не «сколько получится».

- **`uniform(a, b)` может вернуть значение чуть больше `b`** из-за округления float в редких краевых случаях (документировано).

- **`SystemRandom.seed()` молча игнорируется** — иллюзия воспроизводимости.

- **Разные версии Python могут менять алгоритмы распределений** → воспроизводимость гарантируется в пределах версии. `random()` стабилен, но `sample`/`choices` могли менять реализацию между версиями.

## Лучшие практики

- Для безопасности (токены, пароли, соль) — `secrets`/`SystemRandom`, никогда `random`.
- Для тестов и экспериментов фиксируйте `seed` для воспроизводимости.
- Для изоляции состояния (потоки, независимые потоки чисел) создавайте отдельные `random.Random(seed)`.
- Не мутируйте оригинал: вместо `shuffle` используйте `sample(lst, len(lst))`, если нужен новый список.
- Помните про границы: `randint` включает конец, `randrange`/`random` — нет.
- Для взвешенного выбора — `choices(..., weights=...)`.
- Не используйте `random` там, где нужна точность десятичных значений — это `float`.

## Шпаргалка

```python
import random

# Воспроизводимость
random.seed(42)
state = random.getstate(); random.setstate(state)

# Числа
random.random()           # float [0, 1)
random.uniform(a, b)      # float [a, b]
random.randint(a, b)      # int [a, b] ВКЛЮЧИТЕЛЬНО
random.randrange(start, stop, step)  # int [start, stop)
random.getrandbits(k)     # int из k бит

# Выбор
random.choice(seq)                    # один элемент
random.sample(seq, k)                 # k БЕЗ повторений (новый список)
random.choices(seq, weights=w, k=k)   # k С повторениями (+ веса)
random.shuffle(lst)                   # ПЕРЕМЕШАТЬ НА МЕСТЕ (return None)

# Распределения
random.gauss(mu, sigma)         # нормальное (быстрое, не потокобезоп.)
random.normalvariate(mu, sigma) # нормальное (потокобезоп.)
random.expovariate(lambd)       # экспоненциальное
random.triangular(low, high, mode)

# Криптостойкость / непредсказуемость
sr = random.SystemRandom()      # os.urandom, без seed
# Для секретов лучше: import secrets

# Изолированный генератор
rng = random.Random(seed)
```
