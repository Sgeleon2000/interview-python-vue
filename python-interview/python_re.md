# Модуль `re` (регулярные выражения) — подготовка к собеседованию

## Что это и зачем

`re` — модуль стандартной библиотеки для работы с **регулярными выражениями** (regex): поиск, проверка, извлечение и замена подстрок по шаблону. Используется для:

- валидации форматов (email, телефон, дата, IP);
- парсинга текстов и логов;
- извлечения данных (scraping, токенизация);
- сложных замен (с группами, обратными ссылками, функциями).

Regex в Python реализованы через бэктрекинговый движок (NFA), что мощно, но требует осторожности с производительностью (catastrophic backtracking).

## Ключевые концепции

- **Шаблон (pattern)** — строка с метасимволами, описывающая множество строк.
- **Raw-строки (`r"..."`)** — обязательны для шаблонов, чтобы `\` не интерпретировался Python (например `\d` вместо `\\d`).
- **Compiled pattern** — `re.compile()` создаёт объект `Pattern` с методами; ускоряет повторное использование.
- **Match-объект** — результат успешного поиска; содержит позиции, группы.
- **Группы** — захваченные части `(...)`, доступны по номеру/имени.
- **Жадность vs ленивость** — `*`/`+` захватывают максимум, `*?`/`+?` — минимум.
- **Lookaround** — проверки «вокруг» без захвата: lookahead `(?=)`, lookbehind `(?<=)`.

## Основные функции/классы/методы

### Базовый синтаксис метасимволов

```python
# .      — любой символ (кроме \n, если нет DOTALL)
# ^ $    — начало/конец строки
# *      — 0 и более
# +      — 1 и более
# ?      — 0 или 1
# {n}    — ровно n раз
# {n,m}  — от n до m раз
# []     — класс символов: [a-z], [^abc] (отрицание)
# |      — или
# ()     — группа
# \d \D  — цифра / не цифра
# \w \W  — [a-zA-Z0-9_] / не словесный
# \s \S  — пробельный / не пробельный
# \b \B  — граница слова / не граница
# \A \Z  — начало/конец всей строки (не зависит от MULTILINE)
```

### `re.compile`

```python
import re

# Компиляция шаблона для многократного использования (быстрее)
pattern = re.compile(r"\d{4}-\d{2}-\d{2}")

print(pattern.match("2026-06-20"))   # <re.Match object; match='2026-06-20'>
print(pattern.findall("даты: 2026-06-20 и 2025-01-01"))
# ['2026-06-20', '2025-01-01']
```

### `match`, `search`, `fullmatch`

```python
import re

text = "год 2026 месяц 06"

# match — только с НАЧАЛА строки
print(re.match(r"\d+", text))         # None (начинается с 'год')
print(re.match(r"год", text))         # <Match 'год'>

# search — где угодно в строке (первое совпадение)
m = re.search(r"\d+", text)
print(m.group())                      # 2026
print(m.start(), m.end())             # 4 8
print(m.span())                       # (4, 8)

# fullmatch — вся строка должна совпасть целиком
print(re.fullmatch(r"\d+", "12345"))  # <Match '12345'>
print(re.fullmatch(r"\d+", "123abc")) # None
```

### `findall` и `finditer`

```python
import re

text = "телефоны: +7-900-123-45-67, +7-911-765-43-21"

# findall возвращает СПИСОК строк (или кортежей, если есть группы)
print(re.findall(r"\+7-\d{3}-\d{3}-\d{2}-\d{2}", text))
# ['+7-900-123-45-67', '+7-911-765-43-21']

# С группами findall возвращает кортежи групп!
print(re.findall(r"(\d{3})-(\d{3})", text))
# [('900', '123'), ('911', '765')]

# finditer возвращает ИТЕРАТОР Match-объектов (экономнее по памяти)
for m in re.finditer(r"\+7-\d{3}", text):
    print(m.group(), "на позиции", m.start())
# +7-900 на позиции 10
# +7-911 на позиции 28
```

### `sub` и `subn`

```python
import re

# Простая замена
print(re.sub(r"\s+", " ", "много    пробелов\tи\nтабов"))
# много пробелов и табов

# Замена с обратными ссылками (\1, \g<1>, \g<name>)
print(re.sub(r"(\w+)@(\w+)", r"\2.\1", "user@domain"))
# domain.user

# Замена ФУНКЦИЕЙ (получает Match)
def to_upper(m):
    return m.group(0).upper()
print(re.sub(r"\b\w", to_upper, "привет мир"))  # Привет Мир

# count ограничивает число замен
print(re.sub(r"o", "0", "foo boo", count=2))    # f00 boo

# subn возвращает (строка, число замен)
print(re.subn(r"o", "0", "foo boo"))            # ('f00 b00', 4)
```

### `split`

```python
import re

# Разбиение по шаблону
print(re.split(r"[,;]\s*", "a, b; c,d"))    # ['a', 'b', 'c', 'd']

# Группы в split сохраняют разделители
print(re.split(r"(\d+)", "abc123def456"))   # ['abc', '123', 'def', '456', '']

# maxsplit ограничивает число разбиений
print(re.split(r"\s+", "a b c d", maxsplit=2))  # ['a', 'b', 'c d']
```

### Группы и именованные группы

```python
import re

# Нумерованные группы
m = re.search(r"(\d{4})-(\d{2})-(\d{2})", "2026-06-20")
print(m.group(0))     # 2026-06-20 (вся строка)
print(m.group(1))     # 2026
print(m.groups())     # ('2026', '06', '20')

# Именованные группы (?P<name>...)
m = re.search(r"(?P<year>\d{4})-(?P<month>\d{2})-(?P<day>\d{2})", "2026-06-20")
print(m.group("year"))   # 2026
print(m.groupdict())     # {'year': '2026', 'month': '06', 'day': '20'}

# Незахватывающая группа (?:...) — группирует, но не сохраняет
m = re.search(r"(?:abc)+(\d+)", "abcabc123")
print(m.groups())        # ('123',) — abc не захвачена

# Обратная ссылка ВНУТРИ шаблона — повтор группы
# Найдём удвоенные слова
print(re.findall(r"\b(\w+)\s+\1\b", "это это тест тест ок"))
# ['это', 'тест']
```

### Флаги

```python
import re

# IGNORECASE (re.I) — игнор регистра
print(re.findall(r"привет", "Привет ПРИВЕТ привет", re.IGNORECASE))
# ['Привет', 'ПРИВЕТ', 'привет']

# MULTILINE (re.M) — ^ и $ срабатывают на каждой строке
text = "строка1\nстрока2\nстрока3"
print(re.findall(r"^строка\d", text, re.MULTILINE))
# ['строка1', 'строка2', 'строка3']

# DOTALL (re.S) — точка матчит и \n
print(re.search(r"a.b", "a\nb"))            # None
print(re.search(r"a.b", "a\nb", re.DOTALL)) # <Match 'a\nb'>

# VERBOSE (re.X) — игнор пробелов и комментариев в шаблоне
phone = re.compile(r"""
    \+7        # код страны
    -?\d{3}    # код оператора
    -?\d{7}    # номер
""", re.VERBOSE)
print(phone.search("+7-900-1234567").group())  # +7-900-1234567

# Комбинирование флагов через |
re.findall(r"^abc", text, re.I | re.M)

# Inline-флаги в самом шаблоне: (?i), (?m), (?s), (?x)
print(re.findall(r"(?i)привет", "ПРИВЕТ"))  # ['ПРИВЕТ']
```

### Жадность и ленивость

```python
import re

html = "<b>жирный</b> и <i>курсив</i>"

# Жадный * захватывает МАКСИМУМ
print(re.findall(r"<.*>", html))    # ['<b>жирный</b> и <i>курсив</i>']

# Ленивый *? захватывает МИНИМУМ
print(re.findall(r"<.*?>", html))   # ['<b>', '</b>', '<i>', '</i>']

# Аналогично: +?  ??  {n,m}?
print(re.match(r"\d+?", "12345").group())  # 1 (минимум)
print(re.match(r"\d+", "12345").group())   # 12345 (максимум)

# Possessive/atomic группы (Python 3.11+): (?>...) и a++
# Не откатываются — защита от backtracking
print(re.match(r"(?>\d+)5", "12345"))  # None: \d+ съел всё, не отдаёт 5
```

### Lookahead и lookbehind

```python
import re

# Positive lookahead (?=...) — впереди ДОЛЖНО быть
# Найти число, за которым идёт '₽'
print(re.findall(r"\d+(?=₽)", "100₽ 200$ 300₽"))  # ['100', '300']

# Negative lookahead (?!...) — впереди НЕ должно быть
print(re.findall(r"\d+(?!₽)", "100₽ 200$"))       # ['10', '200'] (число не перед ₽)

# Positive lookbehind (?<=...) — позади ДОЛЖНО быть
print(re.findall(r"(?<=\$)\d+", "$100 €200 $300")) # ['100', '300']

# Negative lookbehind (?<!...) — позади НЕ должно быть
print(re.findall(r"(?<!\$)\b\d+", "$100 200"))     # ['200']

# Практика: проверка пароля (хотя бы 1 цифра, 1 буква, 8+ символов)
strong = re.compile(r"^(?=.*[A-Za-z])(?=.*\d).{8,}$")
print(bool(strong.match("abc12345")))  # True
print(bool(strong.match("abcdefgh")))  # False (нет цифры)
```

Важно: lookbehind в Python должен быть **фиксированной длины** (нельзя `(?<=a+)`).

### Backreferences (обратные ссылки)

```python
import re

# В шаблоне: \1 ссылается на захваченную группу 1
# Проверка парных кавычек/тегов
print(re.findall(r"<(\w+)>.*?</\1>", "<b>x</b><i>y</i>"))
# ['b', 'i']

# В замене: \1, \g<1>, \g<name>
print(re.sub(r"(\d+)-(\d+)", r"\2-\1", "10-20"))  # 20-10
print(re.sub(r"(?P<a>\d+)", r"[\g<a>]", "42"))    # [42]
```

## Частые вопросы на собеседовании

**Q1: Разница между `match`, `search`, `fullmatch`?**
A: `match` проверяет совпадение **с начала** строки (не обязательно до конца). `search` ищет **в любом месте** (первое совпадение). `fullmatch` требует, чтобы совпала **вся строка целиком**. Для валидации формата используют `fullmatch` или `match` с `$`.

**Q2: Чем `findall` отличается от `finditer`?**
A: `findall` возвращает список строк (или кортежей групп) — удобно, но грузит всё в память. `finditer` возвращает ленивый итератор `Match`-объектов — экономнее по памяти и даёт доступ к позициям/группам. Важный подвох: если в шаблоне есть группы, `findall` вернёт кортежи групп, а не полные совпадения.

**Q3: Зачем `re.compile`, если есть модульные функции?**
A: `compile` создаёт объект `Pattern` один раз; при многократном использовании одного шаблона это быстрее (нет повторного парсинга) и читабельнее. Модульные функции (`re.search` и т.д.) внутри кешируют скомпилированные шаблоны (кеш ~512 штук), поэтому для разовых вызовов разница невелика.

**Q4: Жадные vs ленивые квантификаторы — объясните.**
A: Жадные (`*`, `+`, `?`, `{n,m}`) захватывают максимально возможное, затем откатываются (backtracking) при необходимости. Ленивые (`*?`, `+?` и т.д.) захватывают минимум и расширяются по мере надобности. Классический пример: `<.*>` на `<a><b>` захватит всё, а `<.*?>` — только `<a>`.

**Q5: Что такое lookahead/lookbehind и чем отличаются от групп?**
A: Это «утверждения нулевой ширины» — проверяют контекст вокруг позиции, но **не входят в совпадение** и не сдвигают курсор. `(?=)` — positive lookahead, `(?!)` — negative, `(?<=)` — positive lookbehind, `(?<!)` — negative. В отличие от групп, ничего не захватывают. Lookbehind в Python — фиксированной длины.

**Q6: Что такое catastrophic backtracking и как избежать?**
A: Экспоненциальный взрыв числа путей при вложенных/перекрывающихся квантификаторах (`(a+)+`, `(a|a)*`) на несовпадающем вводе. Движок перебирает огромное число комбинаций. Избегают: упрощением шаблона, использованием возможных (possessive) квантификаторов/атомарных групп `(?>...)` (3.11+), более точными классами символов вместо `.`, или библиотекой `regex`/`re2`.

**Q7: Чем отличается захватывающая группа от незахватывающей?**
A: `(...)` захватывает текст в группу (доступна по номеру/имени, влияет на `findall`/`groups`). `(?:...)` группирует для квантификатора/альтернации, но **не сохраняет** — экономит память и не сбивает нумерацию.

**Q8: Зачем raw-строки в regex?**
A: В обычных строках `\` — escape-символ Python (`\n`, `\t`). Чтобы передать движку regex буквальный `\d`, без raw-строки нужно писать `"\\d"`. `r"\d"` решает это. Также избегает конфликтов вроде `\b` (backspace в обычной строке, граница слова в regex).

**Q9: Как `MULTILINE` влияет на `^` и `$`?**
A: По умолчанию `^`/`$` матчат начало/конец всей строки. С `re.MULTILINE` они срабатывают на каждой строке (после/перед `\n`). Для абсолютного начала/конца независимо от флага есть `\A` и `\Z`.

**Q10: Как работает `\1` в шаблоне vs в строке замены?**
A: В **шаблоне** `\1` — backreference: требует повтора уже захваченной группы 1 (например для парных тегов). В **строке замены** `\1` или `\g<1>` подставляет содержимое захваченной группы в результат.

**Q11: Почему `findall(r"(\d)", ...)` даёт неожиданный результат?**
A: При наличии групп `findall` возвращает не полные совпадения, а содержимое групп. Одна группа → список строк групп; несколько → список кортежей. Чтобы получить полные совпадения, уберите группу или используйте `(?:...)`, либо `finditer` + `m.group(0)`.

**Q12: Что делает флаг `VERBOSE` (re.X)?**
A: Позволяет писать «многословные» шаблоны: пробелы и переносы игнорируются (если не экранированы или не в классе `[]`), можно добавлять комментарии `#`. Делает сложные regex читаемыми и документируемыми.

## Подводные камни (gotchas)

```python
import re

# 1. findall с группами возвращает группы, а не совпадения
print(re.findall(r"a(b)c", "abc abc"))   # ['b', 'b'] — а не ['abc', 'abc']!

# 2. Жадность 'съедает' слишком много
print(re.search(r'".*"', 'a "x" b "y"').group())  # "x" b "y" — захватило между!
print(re.search(r'".*?"', 'a "x" b "y"').group()) # "x" — правильно

# 3. . не матчит \n без DOTALL
print(re.search(r"a.b", "a\nb"))  # None

# 4. Забыли raw-строку
print(bool(re.match("\d", "5")))   # работает, но DeprecationWarning/неверно для \b
print(bool(re.match(r"\d", "5")))  # правильно

# 5. Catastrophic backtracking — НЕ запускайте на длинном несовпадении
# re.match(r"(a+)+$", "aaaaaaaaaaaaaaaaaaaaX")  # зависнет!

# 6. group() на None -> AttributeError
m = re.match(r"\d+", "abc")
# m.group()  # AttributeError: 'NoneType' has no attribute 'group'
print(m.group() if m else "нет совпадения")

# 7. Множество замен: \1 в обычной строке замены нужно r"..."
print(re.sub(r"(\d)", "\\1\\1", "ab1"))  # ab11
print(re.sub(r"(\d)", r"\1\1", "ab1"))   # ab11 (читабельнее)

# 8. Пустые совпадения в split/findall
print(re.split(r"x*", "axbxc"))  # ['', 'a', 'b', 'c', ''] — нюансы пустых матчей
```

## Лучшие практики

- **Всегда используйте raw-строки** `r"..."` для шаблонов.
- **Компилируйте** часто используемые шаблоны через `re.compile` (на уровне модуля).
- **Предпочитайте ленивые** квантификаторы там, где нужен минимальный захват.
- **Используйте именованные группы** `(?P<name>...)` для читаемости.
- **Применяйте `(?:...)`** для группировки без захвата.
- **Не парсите HTML/XML регэкспами** в продакшене — используйте парсеры (`lxml`, `html.parser`).
- **Защищайтесь от ReDoS**: избегайте вложенных квантификаторов на пользовательском вводе; ставьте таймауты или используйте `re2`.
- **Анкорьте** валидирующие шаблоны (`^...$` или `fullmatch`).
- Для VERBOSE-шаблонов добавляйте комментарии.
- Для сложных задач рассмотрите сторонний модуль `regex` (lookbehind переменной длины, лучше Unicode).

## Шпаргалка

```python
import re

# Функции
re.match(p, s)        # совпадение с начала
re.search(p, s)       # первое совпадение где угодно
re.fullmatch(p, s)    # совпадение всей строки
re.findall(p, s)      # список совпадений/групп
re.finditer(p, s)     # итератор Match
re.sub(p, repl, s)    # замена (repl может быть функцией)
re.subn(p, repl, s)   # (строка, кол-во)
re.split(p, s)        # разбиение
re.compile(p, flags)  # компиляция -> Pattern

# Match-объект
m.group(0)            # всё совпадение
m.group(n)            # группа n
m.groups()            # кортеж групп
m.groupdict()         # словарь именованных групп
m.start() m.end()     # позиции
m.span()              # (start, end)

# Метасимволы
.  ^ $  * + ?  {n} {n,m}  [] [^]  | ()  \d \w \s \b \A \Z

# Группы
(...)        захватывающая
(?:...)      незахватывающая
(?P<n>...)   именованная
(?P=n) \1    backreference

# Lookaround
(?=...)  positive lookahead
(?!...)  negative lookahead
(?<=...) positive lookbehind (фикс. длина)
(?<!...) negative lookbehind

# Флаги
re.I IGNORECASE | re.M MULTILINE | re.S DOTALL
re.X VERBOSE | re.A ASCII | re.U UNICODE
# inline: (?i) (?m) (?s) (?x)

# Жадность
*  +  ?  {n,m}    — жадные
*? +? ?? {n,m}?   — ленивые
(?>...)  a++      — атомарные/possessive (3.11+)
```
