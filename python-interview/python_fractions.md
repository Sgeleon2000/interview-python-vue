# fractions — подготовка к собеседованию

## Что это и зачем

`fractions` — модуль стандартной библиотеки, предоставляющий класс `Fraction` для **точной арифметики рациональных чисел** (дробей вида числитель/знаменатель). В отличие от `float`, `Fraction` хранит число точно как пару целых.

Зачем нужен:
- **Точные рациональные вычисления.** `Fraction(1, 3) + Fraction(1, 3) + Fraction(1, 3) == 1` — точно, без ошибок float.
- **Автоматическое сокращение.** `Fraction(2, 4)` сразу станет `1/2`.
- **Конвертация float → точная дробь.** Можно увидеть, какое именно рациональное число хранит float.
- **Ограничение знаменателя.** `limit_denominator()` находит лучшую дробную аппроксимацию с малым знаменателем (полезно для нот, передаточных чисел, рациональных приближений π).

## Ключевые концепции

- **Рациональное число = пара (numerator, denominator)** взаимно простых целых, знаменатель всегда положителен.
- **Автоматическая нормализация.** Конструктор сразу сокращает дробь через НОД и переносит знак в числитель.
- **Неизменяемость (immutable).** Как `int` — операции возвращают новый объект; можно использовать как ключи словарей.
- **Часть числовой башни.** `Fraction` — это `numbers.Rational`. Бесшовно смешивается с `int` и (через приведение) с `float`/`Decimal`.
- **Точность ценой скорости.** Знаменатели могут разрастаться, операции медленнее float.

## Основные функции/классы/методы

### Создание Fraction

```python
from fractions import Fraction

print(Fraction(3, 4))        # 3/4 — числитель и знаменатель
print(Fraction(2, 4))        # 1/2 — АВТОМАТИЧЕСКОЕ сокращение
print(Fraction(-3, 4))       # -3/4
print(Fraction(3, -4))       # -3/4 — знак переносится в числитель
print(Fraction(6))           # 6   — целое
print(Fraction('3/4'))       # 3/4 — из строки
print(Fraction('0.25'))      # 1/4 — из десятичной строки (точно!)
print(Fraction('1.5'))       # 3/2

# Из float — показывает ТОЧНОЕ двоичное значение
print(Fraction(0.5))         # 1/2  (0.5 представим точно)
print(Fraction(0.1))         # 3602879701896397/36028797018963968  (!!! ошибка float)

# Из Decimal — точно
from decimal import Decimal
print(Fraction(Decimal('0.1')))  # 1/10
```

### Доступ к числителю и знаменателю

```python
from fractions import Fraction

f = Fraction(6, 8)
print(f.numerator)    # 3 — числитель (после сокращения)
print(f.denominator)  # 4 — знаменатель (всегда > 0)

# Это свойства, дробь неизменяема — присвоить нельзя
```

### Арифметика — точная

```python
from fractions import Fraction

# Классический пример: float здесь врёт
print(0.1 + 0.2 == 0.3)                                   # False
print(Fraction(1,10) + Fraction(2,10) == Fraction(3,10))  # True

a = Fraction(1, 3)
b = Fraction(1, 6)
print(a + b)    # 1/2
print(a - b)    # 1/6
print(a * b)    # 1/18
print(a / b)    # 2
print(a ** 2)   # 1/9
print(a ** -1)  # 3   (обратная)

# Сумма трёх третей — РОВНО 1 (float дал бы 0.9999...)
print(Fraction(1,3) + Fraction(1,3) + Fraction(1,3))   # 1

# Смешивание с int — разрешено
print(Fraction(1, 2) + 1)     # 3/2
# Смешивание с float — даёт float (теряем точность!)
print(Fraction(1, 2) + 0.5)   # 1.0  (тип float)
```

### limit_denominator — ограничение знаменателя

```python
from fractions import Fraction
import math

# Лучшая дробь с знаменателем <= max_denominator
pi = Fraction(math.pi)
print(pi)                              # 884279719003555/281474976710656 (точное двоичное)
print(pi.limit_denominator(10))        # 22/7   (знаменитое приближение π)
print(pi.limit_denominator(100))       # 311/99
print(pi.limit_denominator(1000))      # 355/113 (очень точное приближение π)

# Восстановить "красивую" дробь из float-ошибки
print(Fraction(0.1))                       # 3602879701896397/36028797018963968
print(Fraction(0.1).limit_denominator())   # 1/10  — то, что мы и хотели!

# Аппроксимация любого float
print(Fraction(0.333333).limit_denominator(1000))  # 333/1000... или 1/3 при меньшем лимите
print(Fraction(1.4142135).limit_denominator(100))  # приближение sqrt(2)
```

### Конвертации и сравнения

```python
from fractions import Fraction

f = Fraction(7, 2)
print(float(f))     # 3.5  — в float (возможна потеря точности)
print(int(f))       # 3    — усечение к нулю
print(str(f))       # 7/2

# Сравнения точные, можно с int и float
print(Fraction(1, 3) < Fraction(1, 2))   # True
print(Fraction(1, 2) == 0.5)             # True
print(Fraction(10, 5) == 2)              # True

# Как ключ словаря / элемент множества (immutable + hashable)
d = {Fraction(1, 2): 'половина'}
print(d[Fraction(2, 4)])   # 'половина' (равные дроби — равные ключи)

# as_integer_ratio (с Python 3.8)
print(Fraction(3, 4).as_integer_ratio())   # (3, 4)
```

### from_float и from_decimal

```python
from fractions import Fraction
from decimal import Decimal

# Явные конструкторы (эквивалентны передаче в Fraction())
print(Fraction.from_float(0.5))         # 1/2
print(Fraction.from_decimal(Decimal('0.1')))  # 1/10
```

## Частые вопросы на собеседовании

**Q1: Чем `Fraction` отличается от `float` и `Decimal`?**
A: `float` — двоичная плавающая точка, неточная для большинства десятичных дробей. `Decimal` — десятичная с фиксированной точностью, точная для десятичных дробей. `Fraction` — **точная для ЛЮБЫХ рациональных** чисел (`1/3` точно, чего не может ни float, ни Decimal). Цена — скорость и рост знаменателей.

**Q2: Почему `Fraction(0.1)` даёт огромную дробь?**
A: Потому что `float` `0.1` на самом деле хранит `3602879701896397/36028797018963968` (двоичное приближение). `Fraction` показывает точное значение float. Чтобы получить `1/10`, нужно `Fraction('0.1')`, `Fraction(Decimal('0.1'))` или `Fraction(0.1).limit_denominator()`.

**Q3: Что делает `limit_denominator()`?**
A: Находит ближайшую дробь, у которой знаменатель не превышает заданный предел (по умолчанию 10^6), используя теорию непрерывных дробей. Полезно для аппроксимаций (`π ≈ 355/113`) и восстановления «красивых» дробей из float.

**Q4: Сокращается ли дробь автоматически?**
A: Да, конструктор сразу делит числитель и знаменатель на их НОД и переносит знак в числитель. `Fraction(2, 4)` сразу `1/2`, `Fraction(3, -4)` сразу `-3/4`.

**Q5: Что происходит при `Fraction(1,2) + 0.5`?**
A: Результат — `float` `1.0`, а не `Fraction`. При смешивании с float происходит приведение к float, и точность теряется. С `int` результат остаётся `Fraction`.

**Q6: Можно ли использовать Fraction как ключ словаря?**
A: Да. `Fraction` неизменяем и хэшируем. Равные по значению дроби (`Fraction(1,2)` и `Fraction(2,4)`) имеют одинаковый хэш и считаются одним ключом.

**Q7: Как Fraction связан с числовой башней (numbers)?**
A: `Fraction` — подкласс `numbers.Rational` (и, следовательно, `Real`, `Complex`, `Number`). `isinstance(Fraction(1,2), numbers.Rational)` — `True`. У него есть `.numerator` и `.denominator`, требуемые ABC `Rational`.

**Q8: Как из float получить точную десятичную дробь?**
A: `Fraction(x).limit_denominator()` восстанавливает наиболее вероятную «человеческую» дробь. Либо через строку: `Fraction(str(x))`. Например `Fraction(0.1).limit_denominator()` → `1/10`.

**Q9: Какие минусы у Fraction?**
A: Производительность (медленнее float в разы) и неконтролируемый рост знаменателей при цепочках операций (числа становятся большими, память/время растут). Поэтому в «горячих» вычислениях используют float/Decimal.

**Q10: Как Fraction обрабатывает деление на ноль?**
A: `Fraction(1, 0)` бросает `ZeroDivisionError` уже при создании. Знаменатель 0 недопустим.

## Подводные камни (gotchas)

- **`Fraction(0.1)` ≠ `1/10` (главный gotcha про точность float).** Передача float в конструктор переносит двоичную ошибку. Используйте строку, `Decimal` или `.limit_denominator()`.

- **Смешивание с float понижает до float.** `Fraction(1,3) + 0.1` теряет точность — результат становится неточным `float`. Держите цепочку в `Fraction`.

- **Рост знаменателей.** После многих операций знаменатель может стать огромным целым → замедление и память. Периодически применяйте `limit_denominator()`, если допустимо приближение.

- **Производительность.** В числодробильных циклах `Fraction` в десятки/сотни раз медленнее float. Не для «горячего» кода.

- **`int(Fraction(7,2))` усекает к нулю** (`3`), а не округляет. Для округления используйте `round(Fraction(7,2))` (банковское) или явный режим.

- **Сравнение с float точное, но float сам неточен.** `Fraction(1,3) == 0.3333333333333333` — `False`, так как float справа не равен ровно 1/3.

- **`Fraction('1/0')` и `Fraction(1,0)`** → `ZeroDivisionError`.

## Лучшие практики

- Для точных рациональных вычислений (доли, передаточные числа, музыкальные интервалы) используйте `Fraction`.
- Создавайте из строк или `Decimal`, не из `float`, если важна десятичная точность.
- Используйте `limit_denominator()` для аппроксимаций и «очистки» float.
- Не смешивайте с `float` внутри цепочки точных вычислений.
- Для денег предпочитайте `Decimal` (десятичное округление и форматирование удобнее), для долей/точных отношений — `Fraction`.
- В производительном коде конвертируйте в `float` на финальном шаге, держа точность только там, где она критична.

## Шпаргалка

```python
from fractions import Fraction
from decimal import Decimal

# Создание
Fraction(3, 4)            # 3/4
Fraction(2, 4)            # 1/2 (авто-сокращение)
Fraction('3/4')          # из строки
Fraction('0.25')         # 1/4 (точно)
Fraction(Decimal('0.1')) # 1/10 (точно)
Fraction(0.1)            # ОПАСНО: огромная дробь из float-ошибки

# Свойства
f.numerator              # числитель
f.denominator            # знаменатель (> 0)

# Аппроксимация
Fraction(math.pi).limit_denominator(1000)   # 355/113
Fraction(0.1).limit_denominator()           # 1/10

# Конвертации
float(f), int(f), str(f)
f.as_integer_ratio()     # (num, den)

# Арифметика — точная с int, теряет точность с float
Fraction(1,3) + Fraction(1,3) + Fraction(1,3)  # ровно 1
Fraction(1,2) + 1        # 3/2 (с int — ок)
Fraction(1,2) + 0.5      # 1.0 (float! точность потеряна)
```
