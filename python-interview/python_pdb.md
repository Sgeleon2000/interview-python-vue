# pdb — подготовка к собеседованию

## Что это и зачем

`pdb` (Python Debugger) — встроенный интерактивный отладчик исходного уровня. Позволяет остановить выполнение программы, пошагово исполнять код, осматривать и менять переменные, ставить точки останова, двигаться по стеку вызовов и выполнять произвольный Python прямо в контексте паузы.

Зачем нужен:
- **Понять, почему код ведёт себя не так.** Вместо россыпи `print` — интерактивный осмотр состояния.
- **Поймать момент исключения.** Post-mortem отладка показывает стек и переменные в точке падения.
- **Изучить чужой код.** Пошаговое исполнение помогает разобраться в логике.
- **Менять состояние на лету** и проверять гипотезы без перезапуска.

pdb не требует установки; для более удобного UX есть надстройки (`ipdb`, `pdbpp`, отладчики IDE), но команды совместимы.

## Ключевые концепции

- **Breakpoint (точка останова)** — место, где выполнение приостанавливается и управление передаётся в pdb.
- **breakpoint()** — встроенная функция (с Python 3.7) для входа в отладчик; уважает переменную окружения `PYTHONBREAKPOINT`.
- **Frame (кадр стека)** — контекст одного вызова функции (локальные переменные, позиция). По стеку можно перемещаться вверх/вниз.
- **Step / Next / Continue** — три режима продвижения: внутрь вызова, через вызов, до следующей точки останова.
- **Post-mortem** — отладка *после* возникновения исключения, с восстановленным состоянием стека в точке падения.
- **Conditional breakpoint** — точка останова, срабатывающая только при истинном условии.

## Основные функции/классы/методы

### Вход в отладчик

```python
def buggy(data):
    total = 0
    for x in data:
        breakpoint()        # Python 3.7+: остановка здесь, открывается (Pdb)
        total += x
    return total
```

Старый способ (до 3.7 или для явности):

```python
import pdb
pdb.set_trace()             # эквивалент breakpoint()
```

Запуск всей программы под отладчиком с самого начала:

```bash
python -m pdb script.py arg1 arg2
# Остановится на первой строке; далее команды pdb
```

Отключить/переопределить breakpoint() через окружение:

```bash
PYTHONBREAKPOINT=0 python script.py          # breakpoint() ничего не делает
PYTHONBREAKPOINT=ipdb.set_trace python s.py  # использовать ipdb
```

### Основные команды

В приглашении `(Pdb)` доступны команды (можно сокращать до первой буквы):

```
Навигация выполнения:
  n (next)     выполнить текущую строку, НЕ заходя внутрь вызовов
  s (step)     выполнить строку, ЗАХОДЯ внутрь вызова функции
  c (continue) продолжить до следующей точки останова / конца
  r (return)   выполнять до возврата из текущей функции
  unt (until)  выполнять до строки с номером больше текущей (выход из цикла)
  j (jump) N   перепрыгнуть на строку N (без выполнения промежуточных)

Осмотр:
  l (list)     показать исходный код вокруг текущей строки
  ll (longlist) показать весь код текущей функции
  p выражение  напечатать значение (print)
  pp выражение красивый вывод (pretty-print) для структур
  a (args)     показать аргументы текущей функции
  w (where)    показать стек вызовов (текущая позиция отмечена ->)
  retval       показать значение, которое функция собирается вернуть

Навигация по стеку:
  u (up)       подняться на кадр выше (к вызывающей функции)
  d (down)     спуститься на кадр ниже

Точки останова:
  b (break) N           поставить точку на строке N
  b file:N              точка в другом файле
  b func                точка на входе в функцию
  b N, condition        условная точка (срабатывает если condition истинно)
  tbreak N              временная точка (сработает один раз)
  cl (clear) N          удалить точку останова N
  disable / enable N    выключить / включить точку
  condition N expr      задать/изменить условие у точки N
  ignore N count        игнорировать точку N следующие count раз
  commands N            привязать команды, выполняемые при срабатывании точки

Прочее:
  !выражение   выполнить произвольный Python (присваивание и т. п.)
  display expr автоматически показывать значение при остановках
  interact     открыть полноценный Python-REPL в текущем контексте
  h (help)     справка; h команда — по конкретной
  q (quit)     выйти из отладчика (прервать программу)
```

### Пример сессии

```python
# script.py
def divide(a, b):
    result = a / b
    return result

def run():
    nums = [(10, 2), (5, 0), (9, 3)]
    out = []
    for a, b in nums:
        breakpoint()
        out.append(divide(a, b))
    return out

run()
```

Сессия:

```
(Pdb) l                 # показать код вокруг
(Pdb) p a, b            # напечатать аргументы текущей итерации
(10, 2)
(Pdb) s                 # шагнуть внутрь divide()
(Pdb) p a / b           # проверить выражение
5.0
(Pdb) w                 # стек: run() -> divide()
(Pdb) c                 # продолжить до следующего breakpoint()
(Pdb) p b               # на итерации (5, 0)
0
(Pdb) !out              # посмотреть накопленное (через !)
[5.0]
(Pdb) c                 # дальше -> здесь произойдёт ZeroDivisionError
```

### Условные точки останова и команды

```
(Pdb) b 12, b == 0      # остановиться только когда знаменатель ноль
(Pdb) ignore 1 3        # точку #1 игнорировать первые 3 срабатывания
(Pdb) commands 1
(com) p a, b            # выполнять автоматически при срабатывании точки 1
(com) end
(Pdb) c
```

### Изменение состояния на лету

```
(Pdb) !b = 1            # изменить переменную, чтобы избежать деления на ноль
(Pdb) p b
1
(Pdb) c                 # продолжить с новым значением
```

Восклицательный знак нужен, когда имя команды совпадает с именем переменной (например, `n`, `c`, `p`) или для присваиваний, иначе pdb может трактовать строку как команду.

### Post-mortem отладка

Отладка *после* падения, без расстановки breakpoint заранее.

```python
import pdb

def risky():
    data = {"a": 1}
    return data["missing"]      # KeyError

if __name__ == "__main__":
    try:
        risky()
    except Exception:
        # Войти в отладчик в точке исключения, со стеком на момент падения
        pdb.post_mortem()
```

Самый удобный способ — флаг автозапуска post-mortem:

```bash
python -m pdb -c continue script.py
# Скрипт бежит до исключения, затем pdb открывается в точке падения
```

Или из интерактивной сессии после необработанного исключения:

```python
import pdb
pdb.pm()        # post-mortem последнего исключения (sys.last_traceback)
```

### Отладка с разбивкой по кадрам стека (post-mortem навигация)

```
(Pdb) w           # увидеть весь стек вызовов
(Pdb) u           # подняться к вызывающему коду
(Pdb) p local_var # осмотреть локальные переменные другого кадра
(Pdb) d           # вернуться вниз
```

## Частые вопросы на собеседовании

**Q1. Чем отличаются n (next), s (step) и c (continue)?**
`n` выполняет текущую строку и останавливается на следующей, *не заходя* внутрь вызываемых функций. `s` тоже выполняет строку, но *заходит* внутрь вызова (останавливается на первой его строке). `c` продолжает выполнение до следующей точки останова или конца программы.

**Q2. Что такое breakpoint() и чем он лучше pdb.set_trace()?**
`breakpoint()` — встроенная функция (Python 3.7+), вызывающая отладчик. Преимущества: короче; уважает `PYTHONBREAKPOINT` (можно отключить `=0` или подменить отладчик `=ipdb.set_trace` без правки кода). `pdb.set_trace()` жёстко привязан к pdb.

**Q3. Что такое post-mortem отладка и как её запустить?**
Это отладка *после* возникновения исключения: pdb открывается со стеком и переменными в точке падения, без заранее расставленных breakpoint. Способы: `pdb.post_mortem()` в `except`, `pdb.pm()` после необработанного исключения, или `python -m pdb -c continue script.py`.

**Q4. Как поставить условную точку останова?**
`b LINE, condition` — точка сработает, только когда `condition` истинно. Например, `b 42, x > 100`. Условие можно изменить позже командой `condition N expr` или временно гасить через `ignore N count`.

**Q5. Зачем нужен символ ! перед выражением?**
Чтобы выполнить произвольный Python, особенно присваивание или когда имя переменной совпадает с командой pdb (`n`, `c`, `p`, `b` и т. д.). `!n = 5` присвоит переменной `n`, а просто `n` было бы командой next.

**Q6. Как двигаться по стеку вызовов и зачем?**
`w` (where) печатает стек, `u` (up) поднимает к вызывающей функции, `d` (down) спускает обратно. Это нужно, чтобы осмотреть локальные переменные другого кадра — например, понять, с какими аргументами вызвали текущую функцию.

**Q7. Как продолжить выполнение до выхода из текущей функции?**
Команда `r` (return) — исполняет всё до момента возврата из текущей функции и останавливается перед возвратом (можно посмотреть `retval`).

**Q8. Как отлаживать скрипт, не вставляя breakpoint() в код?**
Запустить под отладчиком: `python -m pdb script.py`. Остановится на первой строке. Для отладки исключения без правок: `python -m pdb -c continue script.py` — прогон до падения и автоматический post-mortem.

**Q9. В чём разница p и pp?**
`p` печатает `repr` значения в одну строку. `pp` (pretty-print) форматирует сложные структуры (вложенные dict/list) с переносами и отступами — читабельнее для больших объектов.

**Q10. Какие есть способы временной точки и пропуска срабатываний?**
`tbreak` ставит точку, удаляемую после первого срабатывания. `ignore N count` пропускает следующие `count` попаданий в точку `N` (полезно, чтобы добраться до нужной итерации цикла). `disable`/`enable` временно выключают/включают точки.

## Подводные камни (gotchas)

- **Команда vs переменная.** Если переменная называется `n`, `c`, `s`, `b`, `l`, `p`, то ввод её имени выполнит *команду*, а не покажет значение. Используйте `p n` или `!n`.
- **breakpoint() остался в проде.** Забытый `breakpoint()` зависнет на сервере без TTY или повесит запрос. Защита: линтер (flake8 `T100`), `PYTHONBREAKPOINT=0` в продакшене, pre-commit хук.
- **Step заходит в стандартную библиотеку.** `s` может «провалиться» в недра CPython/библиотек. Чтобы быстро вернуться — `r` (до возврата) или ставьте точку нужного уровня и `c`.
- **jump ограничен.** `j` нельзя прыгать внутрь/из циклов `for`, `finally`, через объявления — отладчик откажет. Это не телепорт по любому коду.
- **Многопоточность/async.** pdb плохо дружит с несколькими потоками (общий stdin, перемешанный вывод) и не идеален для asyncio. Для async есть `breakpoint()` в корутине, но удобнее IDE-отладчик или `aiomonitor`.
- **Состояние можно изменить и забыть.** `!var = ...` меняет реальное состояние программы; последующее поведение будет «нечестным». Помните, что вы вмешались.
- **post_mortem без активного исключения.** `pdb.pm()`/`post_mortem()` без сохранённого трейсбека выдаст ошибку — нужен реальный последний эксепшен.
- **quit убивает программу.** `q` не «отпускает» программу дальше — он завершает выполнение (`BdbQuit`). Чтобы доиграть до конца, используйте `c`.

## Лучшие практики

- Предпочитайте `breakpoint()` вместо `pdb.set_trace()` — короче и управляемо через `PYTHONBREAKPOINT`.
- В CI/проде ставьте `PYTHONBREAKPOINT=0` и линтер, запрещающий забытые точки останова.
- Для отладки падений используйте post-mortem (`python -m pdb -c continue script.py`) — не нужно угадывать место заранее.
- Применяйте условные точки (`b N, cond`) и `ignore`, чтобы не нажимать `c` сотни раз в цикле.
- Для сложного UX поставьте `ipdb`/`pdbpp` (подсветка, автодополнение) — команды те же; подключайте через `PYTHONBREAKPOINT`.
- Двигайтесь осознанно: `n` по умолчанию, `s` — только когда нужно внутрь, `r` — чтобы выскочить из глубоко вложенного кода.
- Используйте `display expr`, чтобы автоматически видеть, как меняется переменная при каждом шаге.
- Не оставляйте изменения состояния (`!var=...`) незамеченными — они искажают дальнейшее поведение.

## Шпаргалка

```
Вход:
  breakpoint()                 # в коде (Python 3.7+)
  import pdb; pdb.set_trace()  # классика
  python -m pdb script.py      # под отладчиком с начала
  python -m pdb -c continue s  # авто post-mortem при падении
  PYTHONBREAKPOINT=0 python s  # отключить breakpoint()

Движение:
  n  next      следующая строка, не заходя в вызовы
  s  step      зайти внутрь вызова
  c  continue  до следующей точки/конца
  r  return    до возврата из функции
  unt until    до строки ниже текущей (выход из цикла)
  j  jump N    перейти на строку N

Осмотр:
  l / ll       код вокруг / вся функция
  p / pp expr  печать / красивая печать
  a            аргументы функции
  w (where)    стек вызовов
  u / d        вверх / вниз по стеку
  !expr        выполнить Python (присваивания)
  display expr автопоказ при остановках

Точки останова:
  b N              на строке N
  b file:N / b fn  в файле / на функции
  b N, cond        условная
  tbreak N         временная (1 раз)
  ignore N cnt     пропустить cnt срабатываний
  cl N / disable N удалить / выключить
  commands N ...   действия при срабатывании

Post-mortem:
  pdb.post_mortem()   в except
  pdb.pm()            после необработанного исключения

Выход:
  q (quit)     завершить программу
```
