# Vue 3 — Условный рендеринг и списки — конспект и вопросы

## О чём раздел

Условный рендеринг (`v-if` / `v-else-if` / `v-else`, `v-show`) и рендеринг списков (`v-for`) —
базовые директивы шаблона, которые есть в любом приложении. На собеседованиях это богатая тема:
тут и классическое «`v-if` vs `v-show`», и «зачем нужен `key` в `v-for`», и «почему нельзя
вешать `v-if` и `v-for` на один элемент». Понимание этих нюансов отличает junior от middle/senior.

## Ключевые концепции

- `v-if` / `v-else-if` / `v-else` — **условное создание/удаление** элемента из DOM и виртуального
  дерева (настоящий условный рендеринг).
- `v-show` — элемент **всегда в DOM**, переключается лишь `display: none`.
- `v-if` дороже на переключение (создание/уничтожение), `v-show` дороже на старте (всегда рендерится).
- `v-for` итерирует массивы, объекты, числовой диапазон и строки.
- `key` обязателен для `v-for`: даёт Vue **идентичность** элементов, управляет переиспользованием
  DOM и корректностью при перестановках/удалениях.
- `v-if` и `v-for` **на одном элементе не сочетают** — в Vue 3 `v-if` имеет **приоритет** и не
  видит переменную цикла.
- Мутирующие методы массива (`push`, `splice`, `sort` и др.) реактивны; замена массива новой
  ссылкой — тоже.

## Подробный разбор с примерами кода

### v-if / v-else-if / v-else

Условный блок реально добавляется или удаляется из DOM. Слушатели событий и дочерние компоненты
при ложном условии уничтожаются.

```vue
<script setup>
import { ref } from 'vue'

const status = ref('loading') // 'loading' | 'error' | 'success'
</script>

<template>
  <p v-if="status === 'loading'">Загрузка…</p>
  <p v-else-if="status === 'error'">Ошибка!</p>
  <p v-else>Готово</p>
</template>
```

`v-else` / `v-else-if` должны идти **сразу** за элементом с `v-if` (без посторонних узлов между).

### v-if на `<template>`

`<template>` — невидимая обёртка: позволяет условно рендерить **группу элементов** без лишнего
DOM-контейнера. Сам `<template>` в разметку не попадает.

```vue
<template>
  <template v-if="ok">
    <h2>Заголовок</h2>
    <p>Параграф</p>
    <hr />
  </template>
</template>
```

### v-show

Элемент рендерится всегда, переключается CSS-свойство `display`. `v-show` **нельзя** применять
к `<template>` и он **не работает** с `v-else`.

```vue
<script setup>
import { ref } from 'vue'
const visible = ref(true)
</script>

<template>
  <!-- остаётся в DOM, при false получает style="display: none" -->
  <div v-show="visible">Контент-карусель</div>
</template>
```

### v-if vs v-show — частый вопрос

| Критерий                  | `v-if`                              | `v-show`                       |
|---------------------------|-------------------------------------|--------------------------------|
| Где в DOM при false       | элемента нет вовсе                   | есть, `display: none`          |
| Тип переключения          | создание/уничтожение узлов           | переключение CSS               |
| Стоимость переключения    | высокая (rebuild)                    | низкая (только стиль)          |
| Стоимость первого рендера | низкая (если false — не рендерится) | высокая (рендерится всегда)    |
| Поддержка `<template>`    | да                                  | нет                            |
| `v-else` / `v-else-if`    | да                                  | нет                            |
| Ленивость                 | да (рендер при первом true)         | нет (рендерится сразу)         |

**Правило выбора:** `v-show` — когда переключение **частое** (табы, тогглы, ховеры). `v-if` —
когда условие **редко меняется** или блок дорогой/может не понадобиться (тяжёлый компонент,
права доступа, ветви загрузки). Также `v-if` нужен, если важно **не монтировать** компонент
(не запускать его логику/запросы), пока условие ложно.

```vue
<template>
  <!-- частое переключение вкладки — v-show -->
  <section v-show="tab === 'profile'">Профиль</section>

  <!-- тяжёлый модал, открывается редко — v-if (ленивый монтаж) -->
  <HeavyDialog v-if="dialogOpen" @close="dialogOpen = false" />
</template>
```

### v-for по массиву

```vue
<script setup>
import { ref } from 'vue'

const users = ref([
  { id: 1, name: 'Иван' },
  { id: 2, name: 'Мария' },
])
</script>

<template>
  <!-- (item, index) in array -->
  <ul>
    <li v-for="(user, index) in users" :key="user.id">
      {{ index + 1 }}. {{ user.name }}
    </li>
  </ul>
</template>
```

### v-for по объекту

Перебирает значения; доступны `value`, `key`, `index`.

```vue
<script setup>
import { reactive } from 'vue'
const profile = reactive({ name: 'Иван', age: 30, city: 'Москва' })
</script>

<template>
  <!-- (value, key, index) in object -->
  <ul>
    <li v-for="(value, key, index) in profile" :key="key">
      {{ index }}. {{ key }}: {{ value }}
    </li>
  </ul>
</template>
```

### v-for по диапазону и строке

```vue
<template>
  <!-- n идёт от 1 до 5 включительно -->
  <span v-for="n in 5" :key="n">{{ n }} </span>

  <!-- по символам строки -->
  <span v-for="(ch, i) in 'Vue'" :key="i">{{ ch }}</span>
</template>
```

### КЛЮЧИ (`key`) — топ-вопрос про переиспользование DOM

`key` даёт каждому элементу списка **стабильную идентичность**. По умолчанию Vue использует
стратегию «in-place patch»: при изменении данных он **переиспользует** существующие DOM-узлы,
просто меняя их содержимое в том же порядке. Это быстро, но **некорректно**, если у элементов
есть собственное состояние (фокус инпута, значение неуправляемого поля, состояние компонента,
анимация) или если порядок/набор элементов меняется.

С уникальным `key` Vue умеет **сопоставлять** старые и новые узлы по идентичности: переставлять,
переиспользовать или пересоздавать только нужные.

```vue
<script setup>
import { ref } from 'vue'

const items = ref([
  { id: 'a', text: '' },
  { id: 'b', text: '' },
  { id: 'c', text: '' },
])

function shuffle() {
  items.value.reverse()
}
</script>

<template>
  <div v-for="item in items" :key="item.id">
    <!-- БЕЗ key с уникальным id: при reverse значения инпутов
         «прилипнут» к позиции, а не к элементу -->
    <input v-model="item.text" />
  </div>
  <button @click="shuffle">Перевернуть</button>
</template>
```

Требования к `key`:
- **Уникальный** среди соседей и **стабильный** (один и тот же элемент — один и тот же key).
- Примитив (строка/число). Не объект.
- **Не использовать `index`** как key, если список может **переупорядочиваться, фильтроваться
  или из него удаляются элементы** — index привязан к позиции, а не к данным, и ломает
  сопоставление (баги со state, фокусом, чекбоксами).

```vue
<!-- ПЛОХО: index как key при изменяемом списке -->
<li v-for="(item, index) in list" :key="index">{{ item.name }}</li>

<!-- ХОРОШО: стабильный уникальный идентификатор -->
<li v-for="item in list" :key="item.id">{{ item.name }}</li>
```

`key` также применяют **вне `v-for`** — чтобы заставить Vue пересоздать элемент/компонент при
смене ключа (например, перемонтировать форму при смене редактируемой сущности):

```vue
<!-- смена userId полностью пересоздаёт компонент со сбросом состояния -->
<UserForm :key="userId" :user-id="userId" />
```

### v-for на `<template>`

Чтобы отрендерить несколько узлов на каждую итерацию без обёртки, `v-for` ставят на `<template>`;
`key` — на `<template>`.

```vue
<template>
  <ul>
    <template v-for="item in items" :key="item.id">
      <li class="title">{{ item.title }}</li>
      <li class="desc">{{ item.desc }}</li>
    </template>
  </ul>
</template>
```

### v-for с компонентами

`key` обязателен. Данные передаются через пропсы — `v-for` сам ничего не «вкладывает» в компонент.

```vue
<template>
  <TodoItem
    v-for="todo in todos"
    :key="todo.id"
    :title="todo.title"
    :done="todo.done"
    @toggle="toggle(todo.id)"
  />
</template>
```

### ПОЧЕМУ нельзя сочетать v-if и v-for на одном элементе — приоритет

В Vue 3 при наличии обеих директив на одном элементе **`v-if` имеет более высокий приоритет**,
чем `v-for`. Это значит, что `v-if` вычисляется **раньше**, чем создаётся область видимости
цикла, и **не имеет доступа к переменной из `v-for`**.

```vue
<!-- ОШИБКА: v-if выполняется раньше v-for, переменной user ещё нет -->
<li v-for="user in users" v-if="user.isActive" :key="user.id">
  {{ user.name }}
</li>
<!-- В Vue 3 это вызовет ошибку: 'user' is not defined -->
```

Кроме того, даже если бы работало, это неэффективно: фильтрация выполнялась бы на каждый рендер
прямо в шаблоне. Правильные решения:

**Вариант 1 — фильтрация в `computed` (предпочтительно):**

```vue
<script setup>
import { ref, computed } from 'vue'

const users = ref([/* ... */])
const activeUsers = computed(() => users.value.filter((u) => u.isActive))
</script>

<template>
  <li v-for="user in activeUsers" :key="user.id">{{ user.name }}</li>
</template>
```

**Вариант 2 — обёртка `<template v-for>` с `v-if` внутри:**

```vue
<template>
  <template v-for="user in users" :key="user.id">
    <li v-if="user.isActive">{{ user.name }}</li>
  </template>
</template>
```

Вариант с `computed` лучше: кешируется и не создаёт лишних узлов цикла для отфильтрованных
элементов.

### Реактивность мутаций массива

Vue оборачивает реактивные массивы и **перехватывает мутирующие методы**, поэтому они вызывают
обновление представления:
`push`, `pop`, `shift`, `unshift`, `splice`, `sort`, `reverse`.

```vue
<script setup>
import { ref } from 'vue'
const list = ref([3, 1, 2])

list.value.push(4)            // реактивно
list.value.splice(0, 1)       // реактивно
list.value.sort()             // реактивно (мутирует на месте)
</script>
```

Немутирующие методы (`filter`, `map`, `concat`, `slice`) возвращают **новый** массив — нужно
присвоить результат:

```vue
<script setup>
import { ref } from 'vue'
const list = ref([1, 2, 3, 4])

// замена ссылки тоже реактивна
list.value = list.value.filter((n) => n % 2 === 0) // [2, 4]
</script>
```

В Vue 3 (Proxy-реактивность) работают и присваивание по индексу, и изменение `length`, что было
проблемой во Vue 2:

```js
list.value[0] = 99   // реактивно во Vue 3
list.value.length = 1 // реактивно во Vue 3
```

## Полный рабочий пример

Список задач с фильтром (через computed), переключаемой панелью (v-show) и редактированием.

```vue
<script setup>
import { ref, computed } from 'vue'

const filter = ref('all') // 'all' | 'active' | 'done'
const showStats = ref(true)
const newTitle = ref('')

const todos = ref([
  { id: 1, title: 'Изучить v-for', done: true },
  { id: 2, title: 'Понять key', done: false },
  { id: 3, title: 'Разобрать v-if vs v-show', done: false },
])

// Фильтрация в computed — НЕ v-if+v-for на одном элементе
const visibleTodos = computed(() => {
  switch (filter.value) {
    case 'active': return todos.value.filter((t) => !t.done)
    case 'done': return todos.value.filter((t) => t.done)
    default: return todos.value
  }
})

const remaining = computed(() => todos.value.filter((t) => !t.done).length)

function addTodo() {
  const title = newTitle.value.trim()
  if (!title) return
  // push — реактивная мутация
  todos.value.push({ id: Date.now(), title, done: false })
  newTitle.value = ''
}

function removeTodo(id) {
  // замена ссылки через filter — тоже реактивно
  todos.value = todos.value.filter((t) => t.id !== id)
}
</script>

<template>
  <form @submit.prevent="addTodo">
    <input v-model="newTitle" placeholder="Новая задача" />
    <button>Добавить</button>
  </form>

  <div>
    <button @click="filter = 'all'">Все</button>
    <button @click="filter = 'active'">Активные</button>
    <button @click="filter = 'done'">Завершённые</button>
  </div>

  <!-- условный рендеринг ветвями -->
  <p v-if="visibleTodos.length === 0">Нет задач в этом фильтре</p>
  <ul v-else>
    <!-- стабильный key по id, НЕ index -->
    <li v-for="todo in visibleTodos" :key="todo.id">
      <input type="checkbox" v-model="todo.done" />
      <span :style="{ textDecoration: todo.done ? 'line-through' : 'none' }">
        {{ todo.title }}
      </span>
      <button @click="removeTodo(todo.id)">×</button>
    </li>
  </ul>

  <!-- частое переключение панели — v-show -->
  <footer v-show="showStats">Осталось: {{ remaining }}</footer>
  <button @click="showStats = !showStats">Показать/скрыть статистику</button>
</template>
```

## Частые вопросы на собеседовании (Q/A)

**1. В чём разница между `v-if` и `v-show`?**
`v-if` реально добавляет/удаляет элемент из DOM и виртуального дерева (ленивый, дорогой на
переключение). `v-show` всегда держит элемент в DOM и лишь переключает `display`. `v-if` — для
редких изменений и дорогих/ненужных блоков; `v-show` — для частых переключений.

**2. Когда выбрать `v-show`, а когда `v-if`?**
Частое переключение видимости (табы, тогглы) — `v-show`. Редкое условие, тяжёлый компонент,
ленивый монтаж, ветви загрузки, права доступа — `v-if`. Если важно не монтировать компонент
(не запускать его запросы) — только `v-if`.

**3. Зачем нужен `key` в `v-for`?**
Чтобы дать элементам стабильную идентичность. Vue по умолчанию переиспользует DOM-узлы «на
месте», что ломает элементы с собственным состоянием (инпуты, фокус, компоненты) при
перестановке/удалении. `key` позволяет корректно сопоставлять, переиспользовать и пересоздавать
узлы.

**4. Почему нельзя использовать `index` как `key`?**
`index` привязан к позиции, а не к данным. При вставке/удалении/сортировке индексы сдвигаются,
и Vue ошибочно сопоставляет узлы — теряется состояние, путается фокус и значения чекбоксов.
Index допустим лишь для статичных, неизменяемых списков.

**5. Почему нельзя ставить `v-if` и `v-for` на один элемент?**
В Vue 3 `v-if` имеет более высокий приоритет и вычисляется раньше `v-for`, поэтому не видит
переменную цикла — будет ошибка `is not defined`. Решение: фильтровать в `computed` или обернуть
во `<template v-for>` с `v-if` внутри.

**6. Какой приоритет у `v-if` и `v-for` в Vue 3 и Vue 2?**
В Vue 3 приоритет у `v-if` (он «снаружи»). В Vue 2 было наоборот — приоритет у `v-for`. Это
частый «трюковой» вопрос.

**7. Как отфильтровать список правильно?**
Вынести фильтрацию в `computed`, который возвращает отфильтрованный массив, и итерировать по
нему через `v-for`. Это кешируется и не создаёт лишних узлов.

**8. Какие методы массива реактивны?**
Мутирующие, перехваченные Vue: `push`, `pop`, `shift`, `unshift`, `splice`, `sort`, `reverse`.
Немутирующие (`filter`, `map`, `concat`, `slice`) возвращают новый массив — нужно присвоить его.

**9. Реактивно ли присваивание по индексу `arr[0] = x` в Vue 3?**
Да. Vue 3 использует Proxy, поэтому и присваивание по индексу, и изменение `length` реактивны.
Во Vue 2 это не работало (нужен был `Vue.set` / `splice`).

**10. Как итерировать объект и диапазон в `v-for`?**
Объект: `v-for="(value, key, index) in obj"`. Диапазон: `v-for="n in 10"` (от 1 до 10). Также
можно перебирать символы строки.

**11. Можно ли использовать `v-for` на `<template>`?**
Да, чтобы отрендерить группу узлов на каждую итерацию без обёртки. `key` ставится на сам
`<template>`.

**12. Можно ли `v-show` на `<template>` или с `v-else`?**
Нет. `v-show` не поддерживает `<template>` и не работает с `v-else`/`v-else-if`. Для них —
только `v-if`.

**13. Зачем `key` вне `v-for`?**
Чтобы принудительно пересоздать (перемонтировать) элемент/компонент при смене ключа — например,
сбросить состояние формы при переходе к другой сущности (`:key="entityId"`).

**14. Что эффективнее при частом тогглинге — `v-if` или `v-show`?**
`v-show`: переключается только CSS, без создания/уничтожения узлов. `v-if` каждый раз
пересоздаёт поддерево, что дороже.

## Подводные камни (gotchas)

- **`v-if` + `v-for` на одном элементе**: ошибка `is not defined` в Vue 3 — `v-if` не видит
  переменную цикла.
- **`index` как `key`** в изменяемом списке: потеря состояния, неправильный фокус, «прыгающие»
  чекбоксы.
- **Отсутствие `key`** в `v-for`: предупреждение и баги при перестановках (in-place patch).
- **`v-show` на `<template>`** или с `v-else` — не работает.
- **Немутирующие методы массива** (`filter`/`map`) без присваивания результата не меняют список.
- **`v-else` не сразу за `v-if`**: любой узел между ними «разрывает» цепочку условий.
- **Дорогая фильтрация в шаблоне** вместо `computed` — повторяется на каждый рендер.
- **`v-if` против ожиданий ленив**: компонент под ложным `v-if` не смонтирован — его
  `onMounted`/запросы не выполняются, пока условие не станет истинным.

## Лучшие практики

- Для частого переключения — `v-show`, для редкого/дорогого/ленивого — `v-if`.
- Всегда задавайте **стабильный уникальный** `key` в `v-for` (id из данных, не `index`).
- Фильтрацию и сортировку списков выносите в `computed`, не комбинируйте `v-if` с `v-for`.
- Используйте `<template>` для условного/циклического рендеринга групп без лишних обёрток.
- Для перемонтирования компонента при смене сущности применяйте `:key`.
- Помните о Proxy-реактивности Vue 3: присваивание по индексу и `length` работают.
- Не кладите тяжёлые выражения в `v-if`/`v-for` шаблона — выносите в `computed`/методы.

## Шпаргалка

```vue
<!-- Условный рендеринг -->
<p v-if="a">A</p>
<p v-else-if="b">B</p>
<p v-else>C</p>
```
