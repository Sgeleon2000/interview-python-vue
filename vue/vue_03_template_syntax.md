# Vue 3 — Синтаксис шаблонов — конспект и вопросы

## О чём раздел

Vue использует HTML-подобный синтаксис шаблонов, который декларативно связывает
отрендеренный DOM с данными компонента. Все шаблоны Vue — это валидный HTML,
который умеют парсить браузеры и спецификация-совместимые парсеры.

Под капотом Vue компилирует шаблоны в высокооптимизированный JavaScript-код
(render-функции). В сочетании с системой реактивности Vue определяет минимальное
число компонентов, которые нужно перерендерить, и применяет минимальные манипуляции
с DOM при изменении состояния приложения.

Если вы знакомы с концепциями Virtual DOM и предпочитаете «голую» силу JavaScript,
можно писать render-функции напрямую (например, с JSX) вместо шаблонов. Но шаблоны
— это рекомендованный и наиболее распространённый подход.

## Ключевые концепции

- **Текстовая интерполяция** `{{ }}` («усы», mustache) — вывод значения в текст.
- **`v-html`** — вывод сырого HTML (риск XSS!).
- **`v-bind`** (сокращение `:`) — привязка атрибутов и пропсов.
- **Булевы атрибуты** — `disabled`, `checked` и т.п. ведут себя по правилам truthy/falsy.
- **Динамические имена атрибутов** — `:[key]="value"`.
- **JavaScript-выражения** — внутри `{{ }}` и привязок можно использовать выражения (но не операторы).
- **Директивы** — специальные атрибуты с префиксом `v-`.
- **Аргументы директив** — часть после двоеточия: `v-bind:href`, `v-on:click`.
- **Динамические аргументы** — `v-bind:[attrName]`, `v-on:[eventName]`.
- **Модификаторы** — суффиксы после точки: `@submit.prevent`, `v-model.trim`.
- **Сокращения** — `:` для `v-bind`, `@` для `v-on`, `#` для `v-slot`.

## Подробный разбор с примерами кода

### Текстовая интерполяция `{{ }}`

Базовый способ связывания данных — «усы» (двойные фигурные скобки):

```vue
<script setup>
import { ref } from 'vue'

const msg = ref('Привет, Vue!')
</script>

<template>
  <!-- Содержимое усов заменяется значением msg.
       Обновляется реактивно при изменении msg -->
  <span>Сообщение: {{ msg }}</span>
</template>
```

Усы интерпретируют содержимое как **простой текст**, а не HTML. Любые теги внутри
строки будут экранированы и выведены как текст. Это защита от XSS по умолчанию.

### `v-html` и риск XSS

Чтобы вывести **сырой HTML**, нужна директива `v-html`:

```vue
<script setup>
import { ref } from 'vue'

const rawHtml = ref('<span style="color:red">Это красный текст</span>')
</script>

<template>
  <p>С интерполяцией: {{ rawHtml }}</p>
  <!-- Выведет текст: <span style="color:red">... -->

  <p>С v-html: <span v-html="rawHtml"></span></p>
  <!-- Отрендерит реальный красный span -->
</template>
```

**ВАЖНО — риск XSS:** Динамический рендеринг произвольного HTML на сайте крайне
опасен и легко приводит к XSS-уязвимостям. Используйте `v-html` **только на
доверенном контенте** и **никогда** — на пользовательском вводе. Содержимое
`v-html` вставляется как обычный HTML — Vue-синтаксис внутри него не компилируется.

### Атрибуты через `v-bind` и сокращение `:`

Усы нельзя использовать внутри HTML-атрибутов. Для атрибутов служит `v-bind`:

```vue
<script setup>
import { ref } from 'vue'

const dynamicId = ref('user-42')
const objectOfAttrs = ref({
  id: 'container',
  class: 'wrapper',
  style: 'background-color: green'
})
</script>

<template>
  <!-- Полная форма -->
  <div v-bind:id="dynamicId"></div>

  <!-- Сокращение : (используется почти всегда) -->
  <div :id="dynamicId"></div>

  <!-- Same-name сокращение (Vue 3.4+): :id вместо :id="id" -->
  <div :id></div>

  <!-- Привязка целого объекта атрибутов без аргумента -->
  <div v-bind="objectOfAttrs"></div>
</template>
```

Если значение привязки равно `null` или `undefined`, атрибут **удаляется** из
отрендеренного элемента.

### Булевы атрибуты

Булевы атрибуты (например, `disabled`) присутствуют/отсутствуют в зависимости от
истинности значения:

```vue
<script setup>
import { ref } from 'vue'

const isButtonDisabled = ref(true)
</script>

<template>
  <!-- Атрибут disabled будет добавлен при truthy isButtonDisabled
       и убран при false/null/undefined -->
  <button :disabled="isButtonDisabled">Кнопка</button>

  <!-- При isButtonDisabled === '' (пустая строка) атрибут ВКЛЮЧЁН,
       т.к. для булевых атрибутов '' трактуется как true -->
</template>
```

Особенность: для булевых атрибутов пустая строка `''` считается «истинной»
(атрибут включается), хотя как обычное JS-значение `''` — falsy.

### Динамические имена атрибутов (динамические аргументы)

Имя самого атрибута можно сделать динамическим через квадратные скобки:

```vue
<script setup>
import { ref } from 'vue'

const attributeName = ref('href')
const eventName = ref('click')
const url = ref('https://vuejs.org')
</script>

<template>
  <!-- Если attributeName === 'href', эквивалентно :href -->
  <a :[attributeName]="url">Ссылка</a>

  <!-- Динамическое имя события: @click при eventName === 'click' -->
  <button @[eventName]="onAction">Действие</button>
</template>
```

Ограничения динамических аргументов:
- Значение должно резолвиться в **строку** или `null` (`null` снимает привязку).
- Нельзя использовать пробелы и кавычки — они невалидны в именах атрибутов HTML.
- В **DOM-шаблонах** (написанных прямо в HTML, а не в `.vue`) браузер
  автоматически приводит имена атрибутов к нижнему регистру, поэтому
  `:[someAttr]` станет `:[someattr]`. В однофайловых компонентах этой проблемы нет.

```vue
<template>
  <!-- НЕВАЛИДНО: пробел в выражении вызовет ошибку компилятора -->
  <!-- <a :['foo' + bar]="value"></a> -->

  <!-- Решение: вынести вычисление в computed-свойство -->
</template>
```

### Использование JavaScript-выражений и их ограничения

Vue поддерживает полную мощь JS-выражений внутри привязок:

```vue
<template>
  {{ number + 1 }}
  {{ ok ? 'ДА' : 'НЕТ' }}
  {{ message.split('').reverse().join('') }}
  <div :id="`list-${id}`"></div>
</template>
```

**Только одно выражение.** Выражение — это то, что можно поставить после `return`.
Поэтому следующее **НЕ** работает:

```vue
<template>
  <!-- ОШИБКА: это объявление, а не выражение -->
  {{ const x = 1 }}

  <!-- ОШИБКА: оператор if не работает; нужен тернарник -->
  {{ if (ok) { return message } }}
</template>
```

Доступ к функциям ограничен «белым списком» глобальных объектов (`Math`, `Date` и
т.п.). Пользовательские глобали недоступны напрямую — их нужно явно добавить через
`app.config.globalProperties`.

Вызов методов допустим, но **функции в привязках должны быть без побочных
эффектов** — они могут вызываться многократно при каждом ре-рендере:

```vue
<template>
  <!-- ПЛОХО: setTimeout/мутация состояния в интерполяции -->
  <time :title="formatDate(date)">{{ formatDate(date) }}</time>
</template>
```

### Директивы

Директива — это атрибут с префиксом `v-`. Её значение — JS-выражение
(кроме `v-for`, `v-on`, `v-slot`). Задача директивы — реактивно применять
side-эффекты к DOM:

```vue
<template>
  <!-- v-if управляет наличием элемента в DOM -->
  <p v-if="seen">Теперь меня видно</p>
</template>
```

Полный синтаксис директивы:

```
v-on:submit.prevent="onSubmit"
│  │   │        │       │
│  │   │        │       └─ Значение (JS-выражение или метод)
│  │   │        └───────── Модификаторы (после точки)
│  │   └────────────────── Аргумент (после двоеточия)
│  └────────────────────── Имя директивы
└───────────────────────── Префикс
```

### Аргументы директив

Некоторые директивы принимают аргумент после двоеточия:

```vue
<template>
  <!-- href — аргумент v-bind -->
  <a v-bind:href="url">Ссылка</a>

  <!-- click — аргумент v-on (имя события) -->
  <button v-on:click="doSomething">Кнопка</button>
</template>
```

### Модификаторы

Модификаторы — специальные постфиксы после точки, меняющие поведение директивы:

```vue
<template>
  <!-- .prevent вызывает event.preventDefault() -->
  <form @submit.prevent="onSubmit">...</form>

  <!-- .stop останавливает всплытие; модификаторы можно цепочкой -->
  <a @click.stop.prevent="onClick">Ссылка</a>

  <!-- .trim, .number, .lazy — модификаторы v-model -->
  <input v-model.trim.number="age" />

  <!-- Клавиша-модификатор: реакция только на Enter -->
  <input @keyup.enter="submit" />
</template>
```

### Сокращения `:` и `@` и `#`

```vue
<template>
  <!-- v-bind:href === :href -->
  <a :href="url">...</a>

  <!-- v-on:click === @click -->
  <button @click="onClick">...</button>

  <!-- v-slot:header === #header -->
  <template #header>Заголовок</template>

  <!-- Сокращения работают и с динамическими аргументами/модификаторами -->
  <a :[key]="value" @[event].prevent="handler">...</a>
</template>
```

Сокращения — не «хак», а часть официального синтаксиса. Они валидны во всех
поддерживаемых браузерах, потому что Vue убирает их при компиляции шаблона.

## Полный рабочий пример

```vue
<script setup>
import { ref, computed } from 'vue'

// Реактивное состояние
const title = ref('Карточка пользователя')
const user = ref({ name: 'Анна', role: 'admin' })
const isDisabled = ref(false)
const eventName = ref('click')
const attrName = ref('data-role')

// Доверенный HTML (например, из вашего CMS) — НЕ пользовательский ввод!
const safeHtml = ref('<strong>Жирный заголовок</strong>')

// Вычисляемый класс
const cardClass = computed(() => (user.value.role === 'admin' ? 'card--admin' : 'card'))

function onAction() {
  // Действие по динамическому событию
  isDisabled.value = !isDisabled.value
}
</script>

<template>
  <!-- Текстовая интерполяция + JS-выражение -->
  <h2>{{ title }} — {{ user.name.toUpperCase() }}</h2>

  <!-- v-html для ДОВЕРЕННОГО HTML -->
  <div v-html="safeHtml"></div>

  <!-- v-bind: статический класс через computed + динамический атрибут -->
  <article :class="cardClass" :[attrName]="user.role">
    <!-- Булев атрибут -->
    <button
      :disabled="isDisabled"
      @[eventName].prevent="onAction"
    >
      {{ isDisabled ? 'Заблокировано' : 'Переключить' }}
    </button>
  </article>

  <!-- Условный рендер через директиву -->
  <p v-if="user.role === 'admin'">Доступ администратора</p>
</template>

<style scoped>
.card--admin { border: 2px solid gold; }
.card { border: 1px solid #ddd; }
</style>
```

## Частые вопросы на собеседовании (Q/A)

**1. В чём разница между `{{ }}` и `v-html`?**
`{{ }}` интерпретирует значение как простой текст и экранирует HTML (защита от XSS).
`v-html` вставляет сырой HTML без экранирования. `v-html` опасен на недоверенных
данных.

**2. Почему `v-html` опасен и когда его можно использовать?**
Он рендерит произвольный HTML, что открывает дорогу XSS-атакам (внедрение скриптов,
обработчиков событий). Использовать только на доверенном контенте, никогда — на
пользовательском вводе. Внутри `v-html` Vue-синтаксис не компилируется.

**3. Можно ли использовать усы внутри атрибута, например `<a href="{{ url }}">`?**
Нет. Внутри атрибутов усы не работают — нужен `v-bind` (или сокращение `:`):
`<a :href="url">`.

**4. Что произойдёт, если `v-bind` получит значение `null` или `undefined`?**
Атрибут будет удалён из отрендеренного элемента (не будет `attr=""`).

**5. Как работают булевы атрибуты в Vue?**
Атрибут добавляется при truthy-значении и удаляется при `false`/`null`/`undefined`.
Особенность: пустая строка `''` для булевых атрибутов трактуется как `true`.

**6. Какие ограничения у JS-выражений в шаблонах?**
Допускается только **одно выражение** (то, что можно вернуть из `return`).
Объявления (`const x = 1`), `if/else`, циклы — недоступны (используйте тернарник
или computed). Доступ только к whitelisted-глобалям (`Math`, `Date`...).

**7. Что такое директива и из каких частей она состоит?**
Атрибут с префиксом `v-`, который реактивно применяет side-эффекты к DOM.
Состоит из имени, аргумента (после `:`), модификаторов (после `.`) и значения.

**8. Что такое динамический аргумент директивы и какие у него ограничения?**
Это `v-bind:[attr]` / `v-on:[event]`. Значение должно быть строкой или `null`
(`null` снимает привязку). Запрещены пробелы и кавычки. В DOM-шаблонах имена
приводятся к нижнему регистру.

**9. Зачем нужны модификаторы? Приведите примеры.**
Модификаторы декларативно меняют поведение директивы: `.prevent`
(`preventDefault`), `.stop` (`stopPropagation`), `.trim`/`.number`/`.lazy` для
`v-model`, клавиши `.enter`/`.esc`. Их можно цепочкой.

**10. Какие сокращения есть в Vue и валидны ли они для браузеров?**
`:` для `v-bind`, `@` для `v-on`, `#` для `v-slot`. Они валидны, потому что Vue
убирает их на этапе компиляции — в браузер уходит обычный JS.

**11. Что такое same-name сокращение `:id` (без значения)?**
Появилось в Vue 3.4. `:id` эквивалентно `:id="id"` — когда имя атрибута совпадает
с именем переменной в области видимости.

**12. Можно ли вызывать методы в интерполяции? Есть ли подводные камни?**
Можно, но метод должен быть **без побочных эффектов**, потому что вызывается при
каждом ре-рендере. Для производных значений лучше использовать `computed`.

**13. Чем `v-bind="objectOfAttrs"` отличается от обычного `v-bind`?**
Без аргумента `v-bind` привязывает целый объект — каждое его свойство становится
отдельным атрибутом/пропсом. Удобно для прокидывания группы атрибутов.

**14. Чем отличается DOM-шаблон от шаблона в `.vue`-файле по части регистра?**
В DOM-шаблонах (in-html) браузер принудительно приводит имена атрибутов к нижнему
регистру до того, как Vue их увидит. В однофайловых компонентах (SFC) шаблон
компилируется напрямую, регистр сохраняется.

## Подводные камни (gotchas)

- **XSS через `v-html`** — самая частая дыра в безопасности. Никогда не подставляйте
  пользовательский ввод.
- **Усы в атрибутах не работают** — `href="{{ url }}"` не сработает, нужен `:href`.
- **Только одно выражение** — попытка написать `if` или объявление в `{{ }}`
  вызовет ошибку компиляции.
- **Динамические аргументы и регистр** — в DOM-шаблонах `:[myAttr]` станет
  `:[myattr]`. В SFC проблемы нет.
- **Пробелы/кавычки в динамическом аргументе запрещены** — выносите вычисление в
  computed.
- **Побочные эффекты в выражениях шаблона** — функция может вызываться многократно,
  не мутируйте состояние внутри интерполяции.
- **`null`/`undefined` в `v-bind`** убирают атрибут — иногда это неожиданно для
  атрибутов, где нужна пустая строка.

## Лучшие практики

- Всегда используйте сокращения `:` и `@` — это стандарт в современных кодовых базах.
- Сложную логику выносите из шаблона в `computed`/методы — шаблон должен оставаться
  декларативным и читаемым.
- Избегайте `v-html`; если без него никак — санитизируйте HTML (например, DOMPurify)
  и убедитесь, что источник доверенный.
- Для производных значений предпочитайте `computed` вызовам функций в шаблоне
  (кэширование + отсутствие лишних вызовов на ре-рендере).
- Используйте same-name сокращение `:id` (3.4+) для краткости, когда имена совпадают.
- Группируйте множество атрибутов через `v-bind="obj"`, когда это уместно.

## Шпаргалка

```vue
<template>
  {{ value }}                      <!-- интерполяция (экранируется) -->
  <span v-html="trusted"></span>   <!-- сырой HTML (XSS-риск!) -->

  <a :href="url"></a>              <!-- v-bind, сокращение : -->
  <a v-bind:href="url"></a>        <!-- полная форма -->
  <div :id></div>                  <!-- same-name (3.4+) -->
  <div v-bind="attrsObject"></div> <!-- весь объект атрибутов -->

  <button :disabled="flag"></button>      <!-- булев атрибут -->
  <a :[attrName]="val"></a>               <!-- динамический аргумент -->

  <button @click="fn"></button>           <!-- v-on, сокращение @ -->
  <button v-on:click="fn"></button>       <!-- полная форма -->
  <form @submit.prevent="fn"></form>      <!-- модификатор -->
  <input @keyup.enter="fn" />             <!-- модификатор-клавиша -->
  <button @[event]="fn"></button>         <!-- динамическое событие -->

  <template #header></template>           <!-- v-slot, сокращение # -->

  {{ ok ? 'да' : 'нет' }}                 <!-- тернарник (одно выражение) -->
</template>
```

| Конструкция | Назначение |
|---|---|
| `{{ }}` | Текстовая интерполяция (экранируется) |
| `v-html` | Сырой HTML (опасно) |
| `v-bind` / `:` | Привязка атрибутов/пропсов |
| `v-on` / `@` | Обработчики событий |
| `v-slot` / `#` | Слоты |
| `:[arg]` / `@[arg]` | Динамические аргументы |
| `.prevent`, `.stop`, `.trim`... | Модификаторы |
```
