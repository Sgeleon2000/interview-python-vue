# Vue 3 — Передаваемые (fallthrough) атрибуты — конспект и вопросы

## О чём раздел

Fallthrough-атрибуты («сквозные», «передаваемые») — это атрибуты или слушатели `v-on`, переданные компоненту, но **не** объявленные как пропсы или эмиты. По умолчанию Vue автоматически «пробрасывает» их на корневой DOM-элемент компонента. Это удобно: можно навесить `class`, `style`, `id`, `@click`, `data-*` на компонент-обёртку, и они окажутся на нужном элементе внутри.

В разделе разбираем:
- `$attrs` и что в него попадает;
- автоматическое наследование на корневой элемент (включая слияние `class`/`style` и `v-on`);
- `inheritAttrs: false` и ручную привязку `v-bind="$attrs"`;
- компоненты с несколькими корневыми элементами и явное распределение атрибутов;
- доступ к `$attrs` из `<script setup>` через `useAttrs`.

## Ключевые концепции

- **Fallthrough-атрибут** — атрибут/`v-on`-слушатель, переданный компоненту, но не входящий в его объявленные `props` или `emits`.
- **`$attrs`** — объект, содержащий все такие атрибуты: `class`, `style`, `id`, `data-*`, `onClick` и т. д.
- **Автонаследование**: при одном корневом элементе `$attrs` автоматически вешается на него.
- **`class`/`style` сливаются**: корневой элемент компонента и переданный извне `class` объединяются, а не перезаписываются.
- **`v-on` сливается**: переданный `@click` добавляется к собственным обработчикам корня (оба сработают).
- **`inheritAttrs: false`** отключает автонаследование — нужно для ручного распределения.
- **`v-bind="$attrs"`** — ручная привязка всех fallthrough-атрибутов к выбранному элементу.

## Подробный разбор с примерами кода

### 1. Автоматическое наследование атрибутов

```vue
<!-- MyButton.vue -->
<template>
  <button>Нажми меня</button>
</template>
```

```vue
<!-- Родитель -->
<MyButton class="large" id="submit" data-test="btn" />
```

Результат в DOM — атрибуты «упали» на корневой `<button>`:

```html
<button class="large" id="submit" data-test="btn">Нажми меня</button>
```

Ни `class`, ни `id`, ни `data-test` не объявлены пропсами, поэтому они стали fallthrough-атрибутами и наследовались автоматически.

### 2. Слияние class и style

Если у корневого элемента уже есть `class`/`style`, переданные значения **объединяются**, а не заменяются:

```vue
<!-- MyButton.vue -->
<template>
  <button class="btn">Кнопка</button>
</template>
```

```vue
<MyButton class="large" />
<!-- В DOM: <button class="btn large">Кнопка</button> -->
```

То же для `style`: объекты/строки стилей сливаются.

### 3. Наследование v-on слушателей

Слушатели событий тоже передаются. Переданный `@click` добавляется к собственному обработчику корня — сработают оба:

```vue
<!-- MyButton.vue -->
<script setup>
function onClick() {
  console.log('Внутренний клик компонента')
}
</script>

<template>
  <button @click="onClick">Кнопка</button>
</template>
```

```vue
<MyButton @click="() => console.log('Клик родителя')" />
<!-- При клике выведется И «Внутренний клик», И «Клик родителя» -->
```

Важно: это работает, потому что `click` не объявлен в `defineEmits`. Если объявить — `@click` перестанет быть fallthrough и станет кастомным событием.

### 4. Наследование на вложенный компонент

Если корень компонента — другой компонент, fallthrough-атрибуты передаются дальше «насквозь», на его корень:

```vue
<!-- Toggle.vue -->
<template>
  <!-- BaseButton — компонент; атрибуты пройдут на его корень -->
  <BaseButton />
</template>
```

`<Toggle class="x" />` → `class="x"` дойдёт до корневого DOM-элемента `BaseButton`. При этом уже объявленные как пропсы атрибуты не передаются дальше — они «поглощаются».

### 5. Отключение наследования — `inheritAttrs: false`

Когда нужно повесить атрибуты не на корень, а на другой элемент, отключаем автонаследование. В `<script setup>` для этого есть макрос `defineOptions` (3.3+):

```vue
<!-- MyInput.vue -->
<script setup>
// Отключаем автоматическое наследование на корневой элемент
defineOptions({
  inheritAttrs: false
})
</script>

<template>
  <div class="input-wrapper">
    <!-- Вешаем fallthrough-атрибуты вручную на сам input -->
    <input v-bind="$attrs" />
  </div>
</template>
```

Теперь `<MyInput class="x" placeholder="..." />` повесит `class` и `placeholder` на `<input>`, а не на обёртку `<div>`.

До 3.3 (или в отдельном `<script>`) то же делали так:

```vue
<script>
export default {
  inheritAttrs: false
}
</script>

<script setup>
// логика
</script>
```

### 6. Несколько корневых элементов

Если у компонента **несколько корневых узлов**, автонаследования нет (Vue не знает, на какой элемент вешать), и выводится предупреждение, если `$attrs` не привязан явно. Нужно распределить атрибуты вручную:

```vue
<!-- CustomLayout.vue -->
<template>
  <header>...</header>
  <!-- Явно указываем, куда направить fallthrough-атрибуты -->
  <main v-bind="$attrs">...</main>
  <footer>...</footer>
</template>
```

### 7. Доступ к $attrs в script — `useAttrs`

В `<script setup>` к атрибутам обращаются через `useAttrs()`:

```vue
<script setup>
import { useAttrs } from 'vue'

const attrs = useAttrs()

// attrs.class, attrs.style, attrs.onClick, attrs['data-id'] ...
console.log(attrs)
</script>
```

Важно: `attrs` **не реактивен** в смысле деструктуризации — нельзя `const { class: cls } = useAttrs()` и ожидать реактивности. Используйте `attrs.class` напрямую или оборачивайте в `computed`. Имена слушателей в `$attrs` представлены как `onClick`, `onUpdateFoo` (camelCase с префиксом `on`).

В шаблоне `$attrs` доступен напрямую без импорта.

### 8. Опции `$attrs` и регистр

- Все атрибуты, включая `class`, `style`, `id`, `data-*`, `aria-*`, и слушатели `onXxx`, попадают в `$attrs`.
- В `$attrs` имена сохраняют исходный регистр для DOM-атрибутов (`data-test`), а события представлены как `onClick`, `onUpdate:modelValue`.

## Полный рабочий пример

```vue
<!-- LabeledInput.vue — обёртка с лейблом, атрибуты на input -->
<script setup>
import { useAttrs, computed } from 'vue'

// Отключаем автонаследование, чтобы атрибуты не падали на корневой <div>
defineOptions({ inheritAttrs: false })

defineProps({
  label: { type: String, default: '' }
})

const attrs = useAttrs()

// Пример доступа к атрибутам из логики (например, для id связки label/input)
const inputId = computed(() => attrs.id || `input-${Math.random().toString(36).slice(2)}`)
</script>

<template>
  <div class="field">
    <label :for="inputId">{{ label }}</label>
    <!-- v-bind="$attrs" пробрасывает class, placeholder, type, @input и пр. на input -->
    <input :id="inputId" v-bind="$attrs" />
  </div>
</template>
```

```vue
<!-- Form.vue — родитель -->
<script setup>
import { ref } from 'vue'
import LabeledInput from './LabeledInput.vue'

const email = ref('')
</script>

<template>
  <!-- class, placeholder, type, @input — всё уйдёт на внутренний <input>, -->
  <!-- а label обработается как объявленный проп -->
  <LabeledInput
    label="Email"
    class="input-lg"
    type="email"
    placeholder="you@example.com"
    @input="email = $event.target.value"
  />
  <p>Введено: {{ email }}</p>
</template>
```

В DOM атрибуты `class="input-lg"`, `type`, `placeholder` и слушатель `input` окажутся на `<input>`, а не на обёртке `<div class="field">`.

## Частые вопросы на собеседовании (Q/A)

**1. Что такое fallthrough-атрибуты?**
Это атрибуты и `v-on`-слушатели, переданные компоненту, но не объявленные в его `props`/`emits`. По умолчанию Vue автоматически вешает их на корневой DOM-элемент компонента.

**2. Что попадает в `$attrs`?**
Всё, что не объявлено как проп/эмит: `class`, `style`, `id`, `data-*`, `aria-*`, и слушатели вида `onClick`, `onUpdate:foo`.

**3. Как Vue обрабатывает `class` и `style` при наследовании?**
Сливает: переданные снаружи `class`/`style` объединяются с теми, что уже есть на корневом элементе, а не перезаписывают их.

**4. Что происходит со слушателями событий?**
Переданный `@click` добавляется к собственному обработчику корня — сработают оба. Но только если событие не объявлено в `defineEmits`; иначе оно станет кастомным.

**5. Зачем нужен `inheritAttrs: false`?**
Чтобы отключить автонаследование на корень и распределить атрибуты вручную через `v-bind="$attrs"` — например, повесить их на вложенный `<input>`, а не на обёртку.

**6. Как объявить `inheritAttrs: false` в `<script setup>`?**
Через `defineOptions({ inheritAttrs: false })` (Vue 3.3+) или в отдельном блоке `<script> export default { inheritAttrs: false } </script>`.

**7. Что будет, если у компонента несколько корневых элементов и не привязать `$attrs`?**
Автонаследования не произойдёт (некуда), и Vue выдаст предупреждение в dev-режиме. Нужно явно `v-bind="$attrs"` на нужный узел.

**8. Как получить доступ к атрибутам из JS?**
В `<script setup>` — `const attrs = useAttrs()`. В шаблоне — `$attrs` напрямую.

**9. Реактивен ли результат `useAttrs()`?**
Объект `attrs` обновляется, но деструктуризация теряет реактивность. Обращайтесь как `attrs.class` или оборачивайте в `computed`.

**10. Как fallthrough работает с вложенными компонентами?**
Если корень компонента — другой компонент, атрибуты «проходят насквозь» на его корневой DOM-элемент. Атрибуты, поглощённые как пропсы, дальше не идут.

**11. Можно ли пробросить `$attrs`, но исключить часть атрибутов?**
Да — отключить `inheritAttrs`, и в шаблоне привязать вычисленный объект: например `v-bind="filteredAttrs"`, где через `computed` убрать ненужные ключи из `useAttrs()`.

**12. Как `$attrs` соотносится с пропсами при пересечении имён?**
Если атрибут объявлен как проп, он попадает в `props`, а не в `$attrs`. Объявленные эмиты также исключаются из `$attrs` (их слушатели не наследуются как нативные).

## Подводные камни (gotchas)

- **Несколько корневых узлов** ломают автонаследование — будет warning, пока не привяжете `$attrs` вручную.
- **Объявление события убирает его из `$attrs`**: объявив `click` в `defineEmits`, вы лишаете родительский `@click` нативного наследования.
- **Деструктуризация `useAttrs()` не реактивна** — храните ссылку на объект и читайте поля, либо `computed`.
- **`inheritAttrs: false` не очищает `$attrs`** — он лишь отключает автоматическую привязку к корню; сами атрибуты остаются доступны для ручного распределения.
- **Двойное наследование**: если забыть `inheritAttrs: false` и при этом сделать `v-bind="$attrs"` на дочернем элементе, атрибуты могут оказаться и на корне, и на дочернем — дублирование.
- **Порядок `v-bind`**: при `v-bind="$attrs"` рядом с собственными атрибутами учитывайте приоритет — позже указанные могут переопределять (кроме `class`/`style`, которые сливаются).
- **Имена событий в `$attrs`** — это `onClick`, а не `click`; легко ошибиться при ручной фильтрации.

## Лучшие практики

- Для компонентов-обёрток (`input`, `button`) применяйте `inheritAttrs: false` + `v-bind="$attrs"` на целевом элементе.
- Старайтесь держать один корневой элемент, если хотите автонаследование «из коробки».
- Объявляйте только те события, которые действительно кастомные; нативные оставляйте fallthrough, чтобы они наследовались.
- Для `id` и связки `label/for` читайте `attrs.id` через `useAttrs`.
- Не деструктурируйте `useAttrs()` — теряете реактивность.
- Документируйте, какие атрибуты компонент пробрасывает (контракт для пользователей компонента).

## Шпаргалка

```vue
<!-- Автонаследование (один корень) -->
<template>
  <button>...</button> <!-- class/style/id/@click упадут сюда -->
</template>
```

```vue
<!-- Ручное распределение -->
<script setup>
import { useAttrs } from 'vue'
defineOptions({ inheritAttrs: false })   // отключаем автонаследование
const attrs = useAttrs()                  // доступ из JS
</script>

<template>
  <div class="wrapper">
    <input v-bind="$attrs" />            <!-- пробрасываем на input -->
  </div>
</template>
```

```vue
<!-- Несколько корней — обязательно явно -->
<template>
  <header />
  <main v-bind="$attrs" />
  <footer />
</template>
```

| class/style при слиянии | объединяются, не перезаписываются |
| Отключить наследование | `defineOptions({ inheritAttrs: false })` |
| Привязать вручную | `v-bind="$attrs"` |
| Доступ из JS | `useAttrs()` (не деструктурировать) |
