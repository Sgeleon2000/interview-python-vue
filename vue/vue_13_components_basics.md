# Vue 3 — Основы компонентов — конспект и вопросы

## О чём раздел

Компоненты — фундамент любого Vue-приложения. Это переиспользуемые, самодостаточные блоки UI со своей разметкой, логикой и стилями. Приложение строится как **дерево компонентов**: один корневой компонент (`App`) содержит дочерние, те — свои, и так далее. Этот раздел даёт обзор того, как компоненты определяются, регистрируются, общаются между собой (props, события, слоты) и используются динамически.

В современном Vue 3 основной способ описать компонент — **однофайловый компонент (SFC, `.vue`-файл)** с `<script setup>` и Composition API. Старый Options API по-прежнему поддерживается и встречается в легаси-коде.

## Ключевые концепции

- **Компонент** — инкапсулированный, переиспользуемый кусок UI (шаблон + логика + стили).
- **SFC (Single-File Component)** — файл `.vue` с блоками `<template>`, `<script setup>`, `<style>`.
- **Дерево компонентов** — иерархия родитель → потомок, отражающая структуру интерфейса.
- **Props** — входные данные, передаются сверху вниз (от родителя к потомку).
- **Events (emit)** — исходящие сообщения, передаются снизу вверх (от потомка к родителю).
- **Слоты (slots)** — механизм передачи фрагментов шаблона внутрь компонента (композиция контента).
- **Динамические компоненты** — `<component :is="...">` для переключения компонента по условию.
- **Регистрация** — глобальная (доступен везде) vs локальная (только в одном компоненте; со `<script setup>` — автоматически по импорту).
- **Именование** — компоненты называют в **PascalCase**, в DOM-шаблонах допустим **kebab-case**.

## Подробный разбор с примерами кода

### Что такое компонент и как он определяется через SFC

Компонент позволяет разбить UI на независимые части. В Vue 3 самый удобный формат — однофайловый компонент:

```vue
<!-- components/ButtonCounter.vue -->
<script setup>
// Composition API + <script setup> — современный стиль
import { ref } from 'vue'

// Реактивное состояние компонента
const count = ref(0)
</script>

<template>
  <!-- Шаблон компонента -->
  <button @click="count++">Кликнули раз: {{ count }}</button>
</template>

<style scoped>
/* scoped — стили применяются только к этому компоненту */
button {
  font-weight: bold;
}
</style>
```

Каждый компонент имеет **изолированное состояние**: если использовать `ButtonCounter` три раза, у каждого экземпляра будет свой `count`. Это ключевое свойство — состояние не шарится между экземплярами.

### Использование компонента

Импортируем и используем в шаблоне родителя. Со `<script setup>` импортированный компонент автоматически доступен в шаблоне — отдельная регистрация не нужна:

```vue
<!-- App.vue -->
<script setup>
import ButtonCounter from './components/ButtonCounter.vue'
</script>

<template>
  <h1>Здесь дочерний компонент</h1>
  <!-- Используем как обычный тег -->
  <ButtonCounter />
  <ButtonCounter />
  <ButtonCounter /> <!-- три независимых счётчика -->
</template>
```

### Дерево компонентов (организация)

Реальное приложение — это вложенные компоненты, образующие дерево:

```
App (корень)
├── Header
│   ├── Logo
│   └── NavMenu
├── Sidebar
│   └── MenuItem (×N)
└── Content
    ├── ArticleList
    │   └── ArticleCard (×N)
    └── Footer
```

Корневой компонент монтируется в DOM при создании приложения:

```js
// main.js
import { createApp } from 'vue'
import App from './App.vue'

createApp(App).mount('#app')
```

### Передача данных через props (обзор)

Props — это входные параметры компонента. Объявляются через `defineProps`, передаются как атрибуты:

```vue
<!-- BlogPost.vue -->
<script setup>
// Объявляем входной параметр title
defineProps(['title'])
</script>

<template>
  <h4>{{ title }}</h4>
</template>
```

```vue
<!-- Родитель -->
<template>
  <BlogPost title="Мой первый пост" />
  <BlogPost title="Vue классный" />
  <!-- Динамическая передача через v-bind -->
  <BlogPost :title="post.title" />
</template>
```

Подробно props разбираются в отдельном разделе (`vue_14_props.md`).

### События emit (обзор)

Чтобы передать данные снизу вверх (от потомка к родителю), компонент **порождает событие** через `defineEmits`:

```vue
<!-- BlogPost.vue -->
<script setup>
defineProps(['title'])
// Объявляем, что компонент может порождать событие 'enlarge-text'
const emit = defineEmits(['enlarge-text'])
</script>

<template>
  <h4>{{ title }}</h4>
  <!-- Порождаем событие при клике -->
  <button @click="emit('enlarge-text')">Увеличить шрифт</button>
</template>
```

```vue
<!-- Родитель слушает событие через @ -->
<template>
  <BlogPost :title="post.title" @enlarge-text="fontSize += 0.1" />
</template>
```

Можно передавать аргументы: `emit('enlarge-text', 0.2)`. Подробно — в разделе про события.

### Слоты (обзор)

Слоты позволяют родителю передать **фрагмент шаблона** внутрь дочернего компонента — как `children` в других фреймворках:

```vue
<!-- AlertBox.vue -->
<template>
  <div class="alert-box">
    <strong>Внимание!</strong>
    <!-- Сюда вставится контент, переданный родителем -->
    <slot />
  </div>
</template>
```

```vue
<!-- Использование -->
<template>
  <AlertBox>
    Произошло что-то плохое. <!-- этот текст попадёт в <slot /> -->
  </AlertBox>
</template>
```

Бывают именованные слоты (`<slot name="header" />`), слоты по умолчанию и слоты с данными (scoped slots) — это отдельная большая тема.

### Динамические компоненты `<component :is>`

Когда нужно переключать компонент по условию (например, табы), используют встроенный элемент `<component>` с атрибутом `:is`:

```vue
<script setup>
import { ref, shallowRef } from 'vue'
import Home from './Home.vue'
import Posts from './Posts.vue'
import Archive from './Archive.vue'

const tabs = { Home, Posts, Archive }
// shallowRef — компонент не нужно делать глубоко реактивным (оптимизация)
const currentTab = shallowRef(Home)
</script>

<template>
  <button
    v-for="(_, name) in tabs"
    :key="name"
    @click="currentTab = tabs[name]"
  >
    {{ name }}
  </button>

  <!-- :is принимает либо имя зарегистрированного компонента (строка),
       либо сам объект компонента -->
  <component :is="currentTab" />
</template>
```

`:is` может принимать:
- строку — имя зарегистрированного компонента или HTML-тега;
- объект/определение компонента, импортированный напрямую.

При переключении компонент по умолчанию **уничтожается и пересоздаётся**. Чтобы сохранять состояние неактивных компонентов, оборачивают в `<KeepAlive>`:

```vue
<template>
  <KeepAlive>
    <component :is="currentTab" />
  </KeepAlive>
</template>
```

### Регистрация: глобальная vs локальная

**Локальная регистрация** (рекомендуется) — компонент доступен только там, где импортирован. Со `<script setup>` это происходит автоматически:

```vue
<script setup>
import ComponentA from './ComponentA.vue' // доступен только в этом компоненте
</script>
```

**Глобальная регистрация** — компонент доступен во всём приложении без импорта. Делается через `app.component`:

```js
// main.js
import { createApp } from 'vue'
import App from './App.vue'
import MyGlobalButton from './components/MyGlobalButton.vue'

const app = createApp(App)

// Регистрируем глобально (имя в PascalCase)
app.component('MyGlobalButton', MyGlobalButton)

// Можно чейнить
app
  .component('ComponentA', ComponentA)
  .component('ComponentB', ComponentB)

app.mount('#app')
```

Минусы глобальной регистрации:
- **tree-shaking не работает** — компонент попадёт в бандл, даже если нигде не используется;
- **неявные зависимости** — в дочернем компоненте не видно, откуда взялся `MyGlobalButton`, что усложняет понимание кода (аналогично глобальным переменным).

Поэтому глобально регистрируют только действительно повсеместные вещи (UI-кит, иконки).

### Соглашения об именах (PascalCase)

- **Имена компонентов** — `PascalCase` (`BlogPost`, `UserCard`). Это позволяет визуально отличать компоненты от нативных HTML-элементов и даёт подсказки в IDE.
- В **SFC-шаблонах** рекомендуется писать теги в `PascalCase`: `<BlogPost />`.
- В **DOM-шаблонах** (если шаблон пишется прямо в HTML-странице) PascalCase невозможен из-за регистронезависимости HTML — там используется `kebab-case`: `<blog-post>`. Vue умеет сопоставлять `kebab-case` в шаблоне с `PascalCase`-компонентом.
- Имя файла обычно совпадает с именем компонента: `BlogPost.vue`.

### Однофайловые компоненты (SFC) — детали

SFC объединяет три блока в одном файле:

```vue
<script setup>
// Логика: импорты, реактивное состояние, методы
</script>

<template>
  <!-- Разметка -->
</template>

<style scoped>
/* Стили (scoped — изолированы для компонента) */
</style>
```

Преимущества:
- всё про компонент в одном месте;
- подсветка синтаксиса, типы, проверки на этапе сборки;
- CSS-инкапсуляция через `scoped`;
- предкомпиляция шаблонов (быстрее в рантайме).

SFC требует шага сборки (Vite/Vue CLI), но это стандарт для production-приложений.

## Полный рабочий пример

Мини-приложение: список постов, у каждого — кнопка увеличения шрифта (демонстрирует props + emit + дерево компонентов).

```vue
<!-- components/BlogPost.vue -->
<script setup>
// Входные параметры
defineProps({
  title: {
    type: String,
    required: true,
  },
})

// Исходящее событие
const emit = defineEmits(['enlarge-text'])
</script>

<template>
  <article class="post">
    <h4>{{ title }}</h4>
    <button @click="emit('enlarge-text')">Увеличить шрифт</button>
  </article>
</template>

<style scoped>
.post {
  margin-bottom: 8px;
}
</style>
```

```vue
<!-- App.vue -->
<script setup>
import { ref } from 'vue'
import BlogPost from './components/BlogPost.vue'

// Размер шрифта в em — состояние родителя
const postFontSize = ref(1)

// Данные списка
const posts = ref([
  { id: 1, title: 'Vue: знакомство' },
  { id: 2, title: 'Работа с компонентами' },
  { id: 3, title: 'Зачем нужен Composition API' },
])
</script>

<template>
  <div :style="{ fontSize: postFontSize + 'em' }">
    <!-- Перебираем посты, передаём title через props,
         слушаем событие enlarge-text и увеличиваем шрифт -->
    <BlogPost
      v-for="post in posts"
      :key="post.id"
      :title="post.title"
      @enlarge-text="postFontSize += 0.1"
    />
  </div>
</template>
```

Здесь видно весь базовый цикл общения: данные текут вниз через props (`:title`), события — вверх через emit (`@enlarge-text`), а состояние (`postFontSize`) живёт у родителя.

### Тот же `BlogPost` в Options API (для сравнения с легаси)

```vue
<script>
export default {
  // props объявляются в опции props
  props: {
    title: { type: String, required: true },
  },
  // события — в опции emits
  emits: ['enlarge-text'],
}
</script>

<template>
  <h4>{{ title }}</h4>
  <button @click="$emit('enlarge-text')">Увеличить шрифт</button>
</template>
```

## Частые вопросы на собеседовании (Q/A)

**1. Что такое компонент во Vue?**
Переиспользуемый, изолированный блок UI со своим состоянием, шаблоном и стилями. Приложение строится как дерево компонентов. Каждый экземпляр имеет независимое состояние.

**2. Что такое SFC и из каких блоков состоит?**
Single-File Component — файл `.vue` с блоками `<template>` (разметка), `<script setup>` или `<script>` (логика) и `<style>` (стили, опционально `scoped`/`module`). Требует шага сборки.

**3. Чем `<script setup>` отличается от обычного `<script>` с `setup()`?**
`<script setup>` — синтаксический сахар: переменные и импорты верхнего уровня автоматически доступны в шаблоне, props/emits объявляются макросами `defineProps`/`defineEmits`. Меньше шаблонного кода, лучше типизация. Обычный `setup()` требует возвращать всё из функции вручную.

**4. Как передать данные от родителя к потомку?**
Через **props**: объявляем `defineProps`, передаём как атрибут (`:title="..."` для динамики, `title="..."` для статики). Поток — однонаправленный, сверху вниз.

**5. Как передать данные/сигнал от потомка к родителю?**
Через **события**: `const emit = defineEmits(['my-event'])`, затем `emit('my-event', payload)`. Родитель слушает через `@my-event="handler"`.

**6. Что такое слоты и зачем они нужны?**
Механизм передачи фрагментов шаблона внутрь компонента. Позволяет компоненту быть «контейнером», в который родитель вставляет контент. Бывают слоты по умолчанию, именованные и scoped (с передачей данных обратно).

**7. Что делает `<component :is>`?**
Рендерит динамический компонент: `:is` принимает строку (имя зарегистрированного компонента) или объект компонента. Используется для табов, мастеров, условного рендера разных компонентов. При смене `:is` старый компонент уничтожается (если не обёрнут в `<KeepAlive>`).

**8. Зачем `<KeepAlive>`?**
Кэширует неактивные динамические компоненты, сохраняя их состояние и не пересоздавая при переключении. Полезно для табов, где не хочется терять введённые данные.

**9. В чём разница глобальной и локальной регистрации? Что лучше?**
Локальная — компонент доступен только где импортирован (со `<script setup>` — автоматически); поддерживает tree-shaking, явные зависимости. Глобальная (`app.component`) — доступен везде без импорта, но ломает tree-shaking и делает зависимости неявными. Предпочтительна локальная; глобально регистрируют только повсеместные компоненты.

**10. Почему компоненты называют в PascalCase?**
Чтобы отличать их от нативных HTML-тегов, получать автодополнение в IDE и единообразие. В DOM-шаблонах (HTML-страница) используется kebab-case из-за регистронезависимости HTML; Vue сопоставляет `<blog-post>` с компонентом `BlogPost`.

**11. Изолировано ли состояние между экземплярами одного компонента?**
Да. Каждый экземпляр получает собственную копию реактивного состояния. Три `<Counter />` — три независимых счётчика. Поэтому `data` в Options API — функция, возвращающая объект.

**12. Что делает атрибут `scoped` у `<style>`?**
Ограничивает действие стилей только текущим компонентом за счёт добавления уникальных data-атрибутов к элементам и селекторам. Предотвращает протекание стилей в другие компоненты.

**13. Что такое `key` при `v-for` по компонентам и зачем он?**
Уникальный идентификатор элемента списка. Помогает Vue корректно отслеживать, переиспользовать и переупорядочивать DOM-узлы/экземпляры компонентов. Без `key` (или с неправильным) возможны баги с состоянием и неэффективный ререндер.

**14. Можно ли использовать компонент сам в себе (рекурсия)?**
Да, компонент может рендерить сам себя по имени файла (для дерева/комментариев). Нужно обязательно условие выхода, иначе бесконечный цикл. Имя берётся из имени файла либо задаётся через `defineOptions({ name })`.

## Подводные камни (gotchas)

- **Забыли `key` в `v-for`** — особенно болезненно с компонентами, имеющими состояние: Vue может переиспользовать «не тот» экземпляр.
- **Глобальная регистрация всего подряд** — раздувает бандл и прячет зависимости. Регистрируйте локально.
- **PascalCase в DOM-шаблонах не работает** — если пишете шаблон прямо в HTML (без сборки), используйте kebab-case.
- **`:is` со строкой требует регистрации компонента** — если компонент не зарегистрирован, передавайте сам объект компонента, а не строку.
- **Динамический компонент теряет состояние при переключении** — оборачивайте в `<KeepAlive>`, если состояние важно.
- **Мутация props в потомке** — нарушает однонаправленный поток (подробно в разделе про props). Props доступны только для чтения.
- **`shallowRef` для динамических компонентов** — делать определение компонента глубоко реактивным через `ref` бессмысленно и вредит производительности; используйте `shallowRef` или `markRaw`.
- **Несколько корневых узлов в шаблоне (fragments)** — во Vue 3 разрешено, но тогда наследование атрибутов (`$attrs`) требует явного указания, к какому узлу их применять (`v-bind="$attrs"`).

## Лучшие практики

- Используйте **SFC + `<script setup>` + Composition API** как стандарт.
- Именуйте компоненты в **PascalCase**, многословно (`UserProfileCard`, а не `Card`).
- Держите компоненты **маленькими и сфокусированными** на одной задаче; разбивайте крупные.
- Предпочитайте **локальную регистрацию**; глобальную — только для UI-кита/иконок.
- Всегда указывайте **`key`** при `v-for`.
- Данные — **вниз через props**, события — **вверх через emit**; не пытайтесь «достучаться» до родителя напрямую.
- Объявляйте **типы и валидацию props** и **сигнатуры событий** в `defineEmits` — это документирует контракт компонента.
- Используйте **`scoped`-стили** по умолчанию, чтобы избежать конфликтов CSS.
- Для динамических компонентов используйте **`shallowRef`** и при необходимости **`<KeepAlive>`**.

## Шпаргалка

```vue
<!-- Определение компонента (SFC) -->
<script setup>
import { ref } from 'vue'
import Child from './Child.vue'      // локальная регистрация (авто)

const props = defineProps(['title']) // входные данные (вниз)
const emit = defineEmits(['done'])   // события (вверх)
const count = ref(0)                  // изолированное состояние
</script>

<template>
  <Child :title="props.title" @done="count++" />  <!-- props + emit -->
  <slot />                                          <!-- слот -->
  <component :is="currentTab" />                     <!-- динамический -->
  <KeepAlive><component :is="currentTab" /></KeepAlive>
</template>

<style scoped>/* изолированные стили */</style>
```

```js
// Глобальная регистрация
app.component('MyButton', MyButton)
```

| Что | Как | Направление |
|-----|-----|-------------|
| Передать данные вниз | `:prop="value"` / `defineProps` | родитель → потомок |
| Сообщить наверх | `@event` / `emit('event')` | потомок → родитель |
| Передать разметку внутрь | `<slot />` | родитель → потомок |
| Переключить компонент | `<component :is>` | — |
| Сохранить состояние | `<KeepAlive>` | — |
