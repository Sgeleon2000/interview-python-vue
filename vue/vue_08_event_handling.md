# Vue 3 — Обработка событий — конспект и вопросы

## О чём раздел

Обработка событий — это то, как компонент Vue реагирует на действия пользователя: клики, ввод с клавиатуры, движения мыши, отправку форм и т.д. В основе лежит директива `v-on`, у которой есть короткая форма `@`. Vue предоставляет богатый набор **модификаторов**, которые позволяют декларативно описать поведение, которое в «голом» JS пришлось бы писать руками (`event.preventDefault()`, `event.stopPropagation()`, проверки кодов клавиш и т.п.).

В этом разделе разбираем:
- директиву `v-on` / `@`;
- инлайновые обработчики и обработчики-методы (и как Vue их различает);
- доступ к нативному объекту события через `$event`;
- модификаторы событий (`.stop`, `.prevent`, `.self`, `.capture`, `.once`, `.passive`);
- модификаторы клавиш (`.enter`, `.esc`, ...) и системные модификаторы (`.ctrl`, `.shift`, `.alt`, `.meta`, `.exact`);
- модификаторы кнопок мыши (`.left`, `.right`, `.middle`);
- важность порядка модификаторов;
- передачу аргументов в обработчики.

## Ключевые концепции

- **`v-on:click="..."`** ≡ **`@click="..."`** — слушаем DOM-событие (или кастомное событие компонента).
- **Обработчик-метод (method handler)**: значение — имя метода или ссылка на функцию (`@click="handleClick"`). Vue автоматически передаёт нативное событие первым аргументом.
- **Инлайновый обработчик (inline handler)**: значение — выражение JS (`@click="count++"` или `@click="say('hi')"`). Выполняется в момент события.
- **`$event`** — специальная переменная в инлайновом обработчике, дающая доступ к нативному объекту события (или к полезной нагрузке кастомного события).
- **Модификаторы** — постфиксы через точку (`@click.stop.prevent="..."`), декларативно меняющие поведение. Они **компилируются** Vue в соответствующий императивный код.
- **Порядок модификаторов важен**: код генерируется в том порядке, в котором они записаны.

## Подробный разбор с примерами кода

### 1. Два вида обработчиков

```vue
<script setup>
import { ref } from 'vue'

const count = ref(0)

// Обработчик-метод: получает нативное событие первым аргументом
function handleClick(event) {
  // event — это нативный объект MouseEvent
  console.log('Клик по элементу:', event.currentTarget.tagName)
  count.value++
}
</script>

<template>
  <!-- Инлайновый обработчик: выражение выполняется при клике -->
  <button @click="count++">Инлайн: {{ count }}</button>

  <!-- Обработчик-метод: передаём ссылку, событие приходит автоматически -->
  <button @click="handleClick">Метод: {{ count }}</button>
</template>
```

Как Vue различает инлайн и метод? Если значение — простой путь к свойству/методу (например, `handleClick`, `obj.method`) или одиночный идентификатор, Vue трактует это как **обработчик-метод** и вызывает его, передавая событие. В остальных случаях (есть `()`, операторы, присваивания) это **инлайновое выражение**.

### 2. Доступ к событию через `$event`

В обработчике-методе событие приходит автоматически. Но если вы пишете инлайн-вызов с аргументами, нативное событие «теряется» — его нужно передать явно через `$event`:

```vue
<script setup>
function submitMessage(text, event) {
  // event может быть undefined, если его не передали
  if (event) event.preventDefault()
  console.log('Отправка:', text)
}
</script>

<template>
  <!-- $event — нативный объект события внутри инлайнового выражения -->
  <button @click="submitMessage('Привет', $event)">Отправить</button>

  <!-- Эквивалент через стрелочную функцию (тоже работает) -->
  <button @click="(e) => submitMessage('Привет', e)">Отправить</button>
</template>
```

Для **кастомных событий компонента** `$event` содержит полезную нагрузку (payload), переданную в `emit`:

```vue
<!-- $event === значение, переданное в emit('update', значение) -->
<MyInput @update="onUpdate($event, 'extra')" />
```

### 3. Модификаторы событий

Чтобы не «засорять» методы вызовами `event.preventDefault()` / `event.stopPropagation()`, Vue даёт модификаторы. Это позволяет методам заниматься только логикой данных, а не деталями DOM-события.

```vue
<template>
  <!-- .stop — остановить всплытие (event.stopPropagation()) -->
  <a @click.stop="doThis">Не всплывёт</a>

  <!-- .prevent — отменить действие по умолчанию (event.preventDefault()) -->
  <form @submit.prevent="onSubmit">...</form>

  <!-- Модификаторы можно сцеплять -->
  <a @click.stop.prevent="doThat">Стоп + предотвратить</a>

  <!-- Можно использовать без обработчика, только ради модификатора -->
  <form @submit.prevent></form>

  <!-- .self — сработает, ТОЛЬКО если event.target === сам элемент -->
  <!-- (клик по дочернему элементу не вызовет обработчик) -->
  <div @click.self="doThat">...</div>

  <!-- .capture — слушать на фазе перехвата (capture), а не всплытия -->
  <div @click.capture="doThis">...</div>

  <!-- .once — сработает максимум один раз, затем слушатель снимается -->
  <button @click.once="doThis">Только раз</button>

  <!-- .passive — { passive: true }, обещаем не вызывать preventDefault -->
  <!-- Полезно для производительности скролла/тача на мобильных -->
  <div @scroll.passive="onScroll">...</div>
</template>
```

Краткая таблица:

| Модификатор | Эквивалент в JS | Назначение |
|---|---|---|
| `.stop` | `event.stopPropagation()` | Останавливает всплытие |
| `.prevent` | `event.preventDefault()` | Отменяет действие по умолчанию |
| `.self` | проверка `event.target === event.currentTarget` | Срабатывает только на самом элементе |
| `.capture` | `addEventListener(..., { capture: true })` | Фаза перехвата |
| `.once` | `addEventListener(..., { once: true })` | Один раз |
| `.passive` | `addEventListener(..., { passive: true })` | Пассивный слушатель |

> Не используйте `.passive` и `.prevent` вместе — `.passive` обещает браузеру, что `preventDefault()` не будет вызван. Браузер проигнорирует `preventDefault` и выдаст предупреждение.

### 4. Порядок модификаторов имеет значение

Модификаторы применяются в порядке записи, потому что Vue генерирует код последовательно.

```vue
<template>
  <!-- Сначала остановить всплытие, потом обработать клик -->
  <a @click.stop.prevent="doThat"></a>

  <!-- @click.prevent.self — отменит ВСЕ клики по умолчанию (и на дочерних) -->
  <div @click.prevent.self="...">...</div>

  <!-- @click.self.prevent — отменит default только при клике по самому элементу -->
  <div @click.self.prevent="...">...</div>
</template>
```

В первом случае `.prevent` срабатывает для любого клика (включая дочерние элементы), затем `.self` фильтрует обработчик. Во втором случае сначала проверяется `.self`, и только если условие выполнено — отменяется действие по умолчанию.

### 5. Модификаторы клавиш

Для событий клавиатуры (`keyup`, `keydown`, `keypress`) Vue позволяет фильтровать по конкретной клавише:

```vue
<template>
  <!-- Вызвать onEnter только при нажатии Enter -->
  <input @keyup.enter="onEnter" />

  <!-- Псевдонимы клавиш -->
  <input @keyup.tab="..." />
  <input @keyup.delete="..." /> <!-- Delete и Backspace -->
  <input @keyup.esc="..." />
  <input @keyup.space="..." />
  <input @keyup.up="..." />
  <input @keyup.down="..." />
  <input @keyup.left="..." />
  <input @keyup.right="..." />

  <!-- Любую другую клавишу можно указать по KeyboardEvent.key в kebab-case -->
  <input @keyup.page-down="onPageDown" />
  <input @keyup.a="onA" />
</template>
```

Псевдонимы клавиш: `.enter`, `.tab`, `.delete` (ловит и Delete, и Backspace), `.esc`, `.space`, `.up`, `.down`, `.left`, `.right`.

### 6. Системные модификаторы и `.exact`

Системные модификаторы срабатывают, только когда зажата соответствующая клавиша-модификатор.

```vue
<template>
  <!-- Ctrl + Click -->
  <div @click.ctrl="doSomething">Ctrl+клик</div>

  <!-- Alt + Enter -->
  <input @keyup.alt.enter="clear" />

  <!-- .meta — Cmd на Mac, Win на Windows -->
  <div @click.meta="onCmdClick"></div>

  <!-- .ctrl без .exact: сработает, даже если зажаты ещё Shift/Alt -->
  <button @click.ctrl="onClick">Ctrl (+ возможно другие)</button>

  <!-- .exact: сработает РОВНО при Ctrl и больше ничего -->
  <button @click.ctrl.exact="onCtrlClick">Только Ctrl</button>

  <!-- .exact без модификаторов: сработает, только если НЕ зажато ни одного системного -->
  <button @click.exact="onClick">Без модификаторов</button>
</template>
```

Системные модификаторы: `.ctrl`, `.alt`, `.shift`, `.meta`. Модификатор `.exact` контролирует точное сочетание зажатых системных клавиш.

> Особенность `keyup.ctrl`: чтобы событие сработало, нужно отпустить другую клавишу, пока Ctrl ещё зажат. Отпустив сам Ctrl, событие `keyup.ctrl` не получите. Это поведение самих событий клавиатуры, а не Vue.

### 7. Модификаторы кнопок мыши

```vue
<template>
  <!-- Только левая кнопка -->
  <button @click.left="onLeft">ЛКМ</button>

  <!-- Только правая кнопка (часто вместе с .prevent, чтобы убрать контекстное меню) -->
  <div @click.right.prevent="onRight">ПКМ</div>
  <div @contextmenu.prevent="onContextMenu">ПКМ через contextmenu</div>

  <!-- Только средняя кнопка (колёсико) -->
  <button @mousedown.middle="onMiddle">СКМ</button>
</template>
```

Модификаторы: `.left`, `.right`, `.middle`. Они ограничивают срабатывание обработчика конкретной кнопкой мыши.

## Полный рабочий пример

```vue
<script setup>
import { ref } from 'vue'

const log = ref([])
const message = ref('')

function addLog(text) {
  log.value.unshift(`${new Date().toLocaleTimeString()} — ${text}`)
}

// Обработчик-метод: получает нативное событие
function onSubmit(event) {
  // .prevent уже отменил отправку формы, здесь только логика
  addLog(`Форма отправлена: "${message.value}"`)
  message.value = ''
}

// Инлайн с аргументами и $event
function pick(item, event) {
  addLog(`Выбрано: ${item} (target: ${event.target.tagName})`)
}

function onCtrlEnter() {
  addLog('Ctrl + Enter — быстрая отправка')
}

function onSelfClick() {
  addLog('Клик именно по контейнеру (.self)')
}
</script>

<template>
  <section>
    <!-- .prevent — не перезагружать страницу; обработчик-метод получит событие -->
    <form @submit.prevent="onSubmit">
      <!--
        .enter + .ctrl — отправка по Ctrl+Enter
        .esc — очистить поле
      -->
      <input
        v-model="message"
        @keyup.ctrl.enter="onCtrlEnter"
        @keyup.esc="message = ''"
        placeholder="Текст (Ctrl+Enter — отправить, Esc — очистить)"
      />
      <button type="submit">Отправить</button>
    </form>

    <!-- .self — сработает только при клике по самому div, не по кнопкам внутри -->
    <div class="box" @click.self="onSelfClick">
      Кликни по фону этого блока
      <!-- $event передаём явно, т.к. вызов с аргументом -->
      <button @click="pick('A', $event)">A</button>
      <!-- .stop — клик не всплывёт до .self-контейнера -->
      <button @click.stop="pick('B', $event)">B (stop)</button>
    </div>

    <!-- .once — сработает один раз -->
    <button @click.once="addLog('Это сообщение появится один раз')">
      Кликни (once)
    </button>

    <!-- ПКМ без контекстного меню -->
    <button @click.right.prevent="addLog('Правый клик')">ПКМ</button>

    <ul>
      <li v-for="(entry, i) in log" :key="i">{{ entry }}</li>
    </ul>
  </section>
</template>

<style scoped>
.box { padding: 20px; border: 1px dashed #888; margin: 12px 0; }
</style>
```

## Частые вопросы на собеседовании (Q/A)

**1. В чём разница между инлайновым обработчиком и обработчиком-методом?**
Метод (`@click="fn"`) — передаём ссылку, Vue сам подставит нативное событие первым аргументом. Инлайн (`@click="fn('arg')"` или `@click="count++"`) — выполняется выражение JS. Vue различает их по тому, является ли значение простым путём к методу/свойству или полноценным выражением.

**2. Как получить доступ к нативному событию в инлайновом обработчике с аргументами?**
Через специальную переменную `$event`: `@click="handle('id', $event)"`. Альтернатива — стрелочная функция: `@click="(e) => handle('id', e)"`.

**3. Чем `@click.prevent` лучше, чем `event.preventDefault()` в методе?**
Декларативность: метод занимается только логикой данных, а детали DOM-события вынесены в шаблон. Код чище, легче читается и переиспользуется. Также можно вешать модификатор без обработчика: `@submit.prevent`.

**4. Что делает модификатор `.self` и чем отличается от `.stop`?**
`.self` вызывает обработчик, только если `event.target` — сам элемент (а не всплыло от дочернего). `.stop` останавливает всплытие события к родителям. `.self` фильтрует «вход» обработчика, `.stop` влияет на дальнейшее распространение события «наружу».

**5. Почему важен порядок модификаторов?**
Vue генерирует код в порядке их записи. `@click.self.prevent` и `@click.prevent.self` ведут себя по-разному: в первом случае `preventDefault` применяется только при клике по самому элементу, во втором — для всех кликов, включая дочерние.

**6. Зачем нужен `.passive` и почему его нельзя совмещать с `.prevent`?**
`.passive` устанавливает `{ passive: true }`, сообщая браузеру, что обработчик не вызовет `preventDefault()`. Это улучшает производительность скролла/тача, т.к. браузер не ждёт завершения обработчика. С `.prevent` он несовместим, потому что они противоречат друг другу — будет предупреждение, а `preventDefault` проигнорируется.

**7. Что такое `.exact` и приведите пример.**
`.exact` требует точного набора зажатых системных клавиш. `@click.ctrl` сработает при Ctrl, даже если зажаты Shift/Alt. `@click.ctrl.exact` — только если зажат ровно Ctrl. `@click.exact` — только если не зажата ни одна системная клавиша.

**8. Как обработать нажатие Enter только когда зажат Ctrl?**
`@keyup.ctrl.enter="handler"`. Сочетают системный модификатор (`.ctrl`) и модификатор клавиши (`.enter`).

**9. Как сделать так, чтобы ПКМ не показывала контекстное меню браузера?**
`@contextmenu.prevent="handler"` или `@click.right.prevent="handler"`. `.prevent` отменит контекстное меню.

**10. Какой модификатор поймает и Delete, и Backspace?**
`.delete` — псевдоним, ловящий обе клавиши: `@keyup.delete="..."`.

**11. Чем `.once` отличается от ручного снятия слушателя?**
`.once` использует нативную опцию `{ once: true }`, и слушатель снимается автоматически после первого срабатывания. Не нужно вручную хранить ссылку и вызывать `removeEventListener`.

**12. Как `$event` работает с кастомными событиями компонента?**
Для компонентов `$event` — это полезная нагрузка, переданная в `emit`. Если внутри было `emit('change', value)`, то в `@change="onChange($event)"` переменная `$event === value`. Это не нативный DOM-объект.

**13. Можно ли вешать несколько обработчиков на одно событие?**
Через объектный синтаксис `v-on` можно повесить несколько разных событий: `v-on="{ click: a, mouseover: b }"`. Для одного события несколько обработчиков напрямую не описать — но в инлайне можно вызвать несколько функций: `@click="a(); b()"`.

**14. Как работает `.capture`?**
Регистрирует слушатель на фазе перехвата (capture), а не всплытия. Событие на capture-слушателе родителя сработает раньше, чем на целевом элементе. Полезно для перехвата событий до их обработки внутренними элементами.

## Подводные камни (gotchas)

- **`$event` доступен только в инлайновых выражениях.** В методе-обработчике событие приходит как обычный аргумент, `$event` там не используется.
- **Несовместимость `.passive` + `.prevent`.** Браузер проигнорирует `preventDefault` и выдаст warning.
- **`.self` ≠ `.stop`.** Частая путаница. `.self` смотрит на `event.target`, `.stop` — про всплытие.
- **Порядок модификаторов.** `.prevent.self` и `.self.prevent` — это разное поведение.
- **`keyup.ctrl` поведение.** Для срабатывания нужно отпустить другую клавишу при зажатом Ctrl; отпускание самого Ctrl не вызывает `keyup.ctrl`.
- **`.exact` легко забыть.** Без него `@click.ctrl` сработает и при Ctrl+Shift+клик.
- **Инлайн с аргументами теряет событие.** Если написать `@click="fn('id')"`, то `fn` не получит событие — нужно `@click="fn('id', $event)"`.
- **Модификаторы кнопок мыши работают на mouse-событиях.** `.middle` имеет смысл на `mousedown`/`mouseup`/`auxclick`, а `click` для средней кнопки в современных браузерах не всегда генерируется.
- **Регистр клавиш.** Имена клавиш в kebab-case по `KeyboardEvent.key`: `@keyup.page-down`, а не `@keyup.pageDown`.

## Лучшие практики

- Используйте **обработчики-методы** для нетривиальной логики, **инлайн** — только для простых выражений (`count++`, `flag = !flag`).
- Выносите детали DOM-события в **модификаторы** (`.prevent`, `.stop`), а в методах оставляйте чистую логику данных.
- Для отправки форм почти всегда `@submit.prevent`.
- Применяйте `.exact`, когда важно точное сочетание клавиш-модификаторов, чтобы избежать ложных срабатываний.
- Для производительности скролла/тача на мобильных используйте `.passive`.
- Для горячих клавиш сочетайте модификаторы клавиш и системные: `@keydown.meta.s.prevent="save"` (Cmd/Win + S).
- В Options API обработчики кладите в `methods`, в Composition API (`<script setup>`) — это обычные функции в области видимости.
- Предпочитайте именованные обработчики анонимным стрелкам, если логику нужно тестировать или переиспользовать.

  @click="fn('a',$event)" инлайн с аргументами и явным событием

Модификаторы событий:
  .stop       event.stopPropagation()
  .prevent    event.preventDefault()

  .capture    фаза перехвата
  .once       один раз (нативный { once: true })
  .passive    { passive: true } (не с .prevent!)

  .up .down .left .right
  .a .page-down ...  (любая по KeyboardEvent.key в kebab-case)

  .ctrl .alt .shift .meta
